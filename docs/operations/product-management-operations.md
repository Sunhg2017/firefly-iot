# 产品接入模块运维说明

## 1. 适用范围

本文档适用于 Firefly IoT 产品接入模块的页面发布、日常巡检、问题排查与回滚操作，覆盖产品列表页、产品新建/编辑抽屉、产品物模型入口与设备接入入口。

## 2. 页面结构

- 页面顶部为 `产品接入` 页头，说明页面用途和视图切换方式。
- `页面总览` 卡片用于集中展示：
  - 当前页产品数
  - 已发布产品数
  - 开发中产品数
  - 当前页关联设备数
  - 当前启用筛选条件
- `筛选条件` 卡片用于按关键词、分类、协议、状态缩小产品范围。
- 产品支持两种浏览方式：
  - 卡片视图：适合目录浏览与快速识别
  - 表格视图：适合批量管理与密集操作

## 3. 运维关注点

### 3.1 卡片信息层级

- 卡片视图已经去掉重复字段，封面区只保留状态角标和分类图标。
- 如果用户反馈“卡片信息太少”，先确认是否仍可通过卡片摘要区看到分类、协议、认证方式，以及通过指标区看到节点类型、数据格式和设备数量。
- 如果用户反馈“看不到统计信息”，先确认是否忽略了顶部 `页面总览` 卡片。

### 3.2 筛选行为

- 搜索关键词支持产品名称、型号、ProductKey。
- 修改分类、协议、状态筛选后，列表会自动回到第一页。
- 清空搜索框时，关键词筛选会自动移除。
- 新建或编辑产品时，协议下拉会跟随分类联动：
  - 摄像头仅显示 `GB28181`、`RTSP`、`RTMP`
  - 其他分类不显示视频协议

### 3.3 权限与操作

- `协议解析` 按钮依赖 `protocol-parser:read` 权限。
- 已发布产品不允许删除。
- 仍处于开发中的产品可执行发布操作。
- 一型一密产品可在开发中或已发布状态下通过 `设备接入` 抽屉完成动态注册。
- 摄像头产品的接入抽屉会收口到视频接入指引，并引导进入 `视频监控` 页面。

## 4. 回归检查

1. 打开产品接入页，确认顶部先展示 `页面总览`，再展示 `筛选条件`。
2. 切换到卡片视图，确认单卡不再重复出现分类、协议、认证方式、设备数等信息块。
3. 在卡片视图中检查以下信息是否可见：
   - 产品名称
   - ProductKey
   - 分类 / 协议 / 认证方式
   - 节点类型 / 数据格式 / 设备数量
4. 切换到表格视图，确认原有批量管理能力不受影响。
5. 执行关键词、分类、协议、状态筛选，确认总览中的筛选标签会同步变化。
6. 新建摄像头产品，确认协议下拉仅显示 `GB28181`、`RTSP`、`RTMP`。
7. 新建非摄像头产品，确认协议下拉不显示视频协议。
8. 打开摄像头产品的接入抽屉，确认页面引导进入 `视频监控`，不再展示动态注册说明。
9. 构建验证：

```bash
cd firefly-web
npm run build
```

## 5. 常见问题排查

### 5.1 页面看起来仍然很旧

- 确认前端静态资源是否已更新到最新版本。
- 检查浏览器是否命中了旧缓存，必要时强制刷新。
- 检查当前环境是否仍在加载旧的 `ProductList` 打包文件。

### 5.2 卡片操作区换行过多

- 先确认是否为超小宽度屏幕；卡片操作区当前允许自动换行以保证按钮不溢出。
- 若仅个别产品操作较多，优先检查该产品是否同时满足“开发中”“支持动态注册”“协议解析可见”等多个条件。

### 5.3 筛选结果与总览统计不一致

- 当前总览展示的是当前页数据口径，不是全库口径。
- 若需要全库统计，应由后端补充独立统计接口，不应直接从当前页数据推导。

### 5.4 摄像头产品仍能保存普通协议

- 先检查 `firefly-web` 是否已部署到包含分类联动的新版本。
- 再检查 `firefly-device` 是否已部署到包含分类/协议组合校验的新版本。
- 如数据库中已存在历史脏数据，应清理为正确的视频协议，不再依赖前端兜底兼容。
