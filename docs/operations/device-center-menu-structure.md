# 设备中心菜单结构优化运维说明

## 适用范围

本文适用于设备中心左侧菜单结构优化后的发布、回归和排查，覆盖：

- 默认路由菜单
- 租户自定义菜单
- 租户空间菜单分配

## 本次变化

- `/product` 在设备中心菜单中统一显示为“产品接入”。
- 设备中心下新增“协议接入”子分组。
- 协议接入分组下收纳：
  - SNMP 接入
  - Modbus 接入
  - WebSocket 接入
  - TCP/UDP 接入
  - LoRaWAN 接入
- 租户历史菜单配置会在前端展示时自动做命名修正与协议菜单归组。

## 发布验证

### 前端构建

```bash
cd firefly-web
npm run build
```

### 页面验证

1. 登录租户业务空间。
2. 打开设备中心。
3. 检查左侧菜单：
   - “产品管理”应显示为“产品接入”
   - 协议菜单不应再平铺，而应出现在“协议接入”子分组下
4. 打开 `SNMP` 或 `LoRaWAN` 页面，确认面包屑包含 `协议接入`
5. 打开“菜单管理”页，确认路由候选项支持展示多级路径
6. 打开租户空间菜单分配抽屉，确认设备中心菜单树包含“协议接入”分组

## 常见问题排查

### 1. 左侧菜单仍显示“产品管理”

排查顺序：

1. 确认前端静态资源是否更新到最新版本。
2. 清理浏览器缓存后重新登录。
3. 检查是否命中了租户自定义菜单配置。
4. 若命中了租户菜单配置，确认前端版本已包含菜单规范化逻辑。

### 2. 协议菜单仍然平铺

排查顺序：

1. 确认当前环境是否使用了旧前端包。
2. 检查 `BasicLayout` 是否已启用递归菜单构建逻辑。
3. 若租户菜单配置是旧结构，确认归组逻辑是否已生效。

### 3. 面包屑没有出现“协议接入”

排查顺序：

1. 确认当前路径命中了设备中心下的协议类菜单。
2. 确认布局层面包屑生成逻辑已按递归菜单树运行。
3. 若使用租户菜单配置，检查是否仍在使用旧的扁平菜单构造逻辑。

### 4. 菜单管理初始化后仍不是新结构

排查顺序：

1. 确认菜单管理页已更新为递归默认菜单初始化逻辑。
2. 重新执行一次“初始化默认菜单”。
3. 刷新页面后确认菜单树已生成“协议接入”分组。

## 回滚说明

若需要回滚：

1. 回退以下前端文件：
   - `firefly-web/src/config/routes.tsx`
   - `firefly-web/src/config/workspaceRoutes.ts`
   - `firefly-web/src/layouts/BasicLayout.tsx`
   - `firefly-web/src/pages/menu-config/MenuConfigPage.tsx`
   - `firefly-web/src/pages/tenant/TenantList.tsx`
2. 同步回退本次文档。
3. 回退后重新构建前端并验证菜单、面包屑、租户空间菜单分配页。

注意：

- 回滚后设备中心菜单会恢复为协议入口平铺结构。
- 租户历史菜单配置不会自动恢复或清理，需要按实际情况重新初始化。
