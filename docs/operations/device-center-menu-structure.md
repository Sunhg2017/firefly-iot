# 设备中心菜单结构优化运维说明

## 适用范围

本文适用于设备中心左侧菜单结构优化后的发布、回归和排查，覆盖：

- 默认路由菜单
- 租户自定义菜单
- 租户空间菜单分配

## 本次变化

- `/product` 在设备中心菜单中统一显示为“产品接入”。
- 设备中心下新增“协议接入”子分组。
- `/video` 从“运维工具”迁到“设备资产”。
- 协议接入分组下收纳：
  - SNMP 接入
  - Modbus 接入
  - WebSocket 接入
  - TCP/UDP 接入
  - LoRaWAN 接入
- Flyway `V34__move_video_menu_to_device_assets.sql` 会同步修正默认菜单目录，以及仍指向旧父级的历史租户菜单自定义记录。

## 发布验证

### 前端构建

```bash
cd firefly-web
npm run build
```

### Flyway 校验

确认 `firefly-system` 已执行到 `V34__move_video_menu_to_device_assets.sql`。

### 页面验证

1. 登录租户业务空间。
2. 打开设备资产。
3. 检查左侧菜单：
   - “产品管理”应显示为“产品接入”
   - 协议菜单不应再平铺，而应出现在“协议接入”子分组下
   - “视频监控”应出现在“设备资产”下
   - “运维工具”下不应再出现“视频监控”
4. 打开 `SNMP` 或 `LoRaWAN` 页面，确认面包屑包含 `协议接入`
5. 打开“视频监控”页面，确认面包屑归属为 `设备资产`
6. 打开“菜单管理”页，确认路由候选项支持展示多级路径
7. 打开租户空间菜单分配抽屉，确认“视频监控”挂在“设备资产”下

## 常见问题排查

### 1. 左侧菜单仍显示“产品管理”

排查顺序：

1. 确认前端静态资源是否更新到最新版本。
2. 清理浏览器缓存后重新登录。
3. 检查是否命中了租户自定义菜单配置。
4. 若命中了租户菜单配置，确认前端版本已包含菜单规范化逻辑。

### 2. 协议菜单仍然平铺

排查顺序：

1. 确认当前环境是否使用了旧前端包。
2. 检查 `BasicLayout` 是否已启用递归菜单构建逻辑。
3. 若历史环境菜单数据异常，确认菜单台账已执行到最新 Flyway 版本。

### 3. 视频监控仍在运维工具下

排查顺序：

1. 确认 `firefly-system` 已执行 `V34__move_video_menu_to_device_assets.sql`。
2. 检查 `workspace_menu_catalog` 中 `TENANT/video` 的 `parent_menu_key` 是否为 `tenant-device-assets`。
3. 检查 `workspace_menu_customizations` 中是否仍残留 `menu_key='video' AND parent_menu_key='tenant-ops-tools'` 的旧记录。
4. 前端刷新并重新登录后再次验证。

### 4. 面包屑没有出现“协议接入”

排查顺序：

1. 确认当前路径命中了设备中心下的协议类菜单。
2. 确认布局层面包屑生成逻辑已按递归菜单树运行。
3. 若使用租户菜单配置，检查对应菜单父级是否已修正到最新结构。

### 5. 菜单管理初始化后仍不是新结构

排查顺序：

1. 确认菜单管理页已更新为递归默认菜单初始化逻辑。
2. 重新执行一次“初始化默认菜单”。
3. 刷新页面后确认菜单树已生成“协议接入”分组，且“视频监控”位于“设备资产”下。

## 回滚说明

若需要回滚：

1. 回退以下前端文件：
   - `firefly-web/src/config/routes.tsx`
2. 手工回退数据库中 `TENANT/video` 的父级菜单以及相关租户菜单自定义数据。
3. 同步回退本次文档。
4. 回退后重新构建前端并验证菜单、面包屑、租户空间菜单分配页。

注意：

- 回滚后“视频监控”会重新回到“运维工具”。
