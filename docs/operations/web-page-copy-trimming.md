# 前端页面说明文案收口运维说明

## 1. 变更内容

本次变更仅涉及前端展示层：

- 多个页面移除了页头说明和解释性提示卡片
- 协议接入页的顶部引导卡片改为精简入口
- 设备影子、设备消息等工作区减少了辅助说明，只保留状态、结果和必要约束

不涉及：

- 后端接口
- 数据库结构
- 权限模型
- 菜单数据

## 2. 部署与验证

前端部署后重点验证以下页面：

- 协议接入：`Modbus`、`SNMP`、`TCP/UDP`、`WebSocket`、`LoRaWAN`
- OpenAPI / AppKey
- 用户、角色、菜单、权限
- 设备影子、设备消息、设备标识
- 工作台、数据分析、通知、产品、租户

验证要点：

1. 页面头部仍能正常渲染操作按钮。
2. 被删除说明后，筛选、编辑、发送、绑定、保存等主流程仍可直接完成。
3. 危险操作确认、校验报错、发送结果、空状态提示仍然可见。
4. 协议接入页顶部仍保留相关入口按钮，端点信息仍可见。

## 3. 排障口径

如果用户反馈“页面说明不见了”，按以下顺序确认：

1. 先确认是否属于本次预期收口。
2. 再确认对应操作是否仍有必要校验、确认弹窗或结果反馈。
3. 若某个页面确实缺少完成当前操作所必需的信息，应补回最小必要提示，不恢复大段背景说明。

## 4. 回滚

如需回滚，仅回滚前端静态资源或对应前端提交即可；无需回滚后端和数据库。
