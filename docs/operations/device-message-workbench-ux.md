# 设备消息工作台体验优化运维说明

## 变更内容

- 前端设备消息页升级为单设备工作台
- 新增设备上下文区、发送结果反馈、属性模板、服务模板和原始消息示例
- 本次仅改动 `firefly-web` 前端页面，无数据库结构、权限点、菜单台账或后端接口变更

## 部署方式

按现有前端发布流程重新构建并部署 `firefly-web` 即可。

## 依赖关系

- 设备列表接口：用于顶部设备选择
- 产品列表接口：用于展示所属产品信息
- 产品物模型接口：用于生成可写属性和服务参数模板
- 设备消息接口：用于发送属性设置、服务调用和原始消息

## 验证建议

1. 进入设备消息页后，确认顶部可以搜索并切换设备
2. 选中设备后，确认右侧展示所属产品、在线状态、设备状态和能力数量
3. 在属性设置页点击“填充全部可写属性”，确认编辑器生成 JSON 模板
4. 在服务调用页切换服务，确认输入参数模板与右侧参数摘要联动更新
5. 在原始消息页切换消息类型，确认示例模板能够跟随变化
6. 任意发送一次后，确认顶部结果反馈区会显示最近一次发送状态

## 常见问题排查

### 顶部没有可写属性或可调服务

- 检查当前设备所属产品是否已经维护物模型
- 检查物模型里属性是否是只读字段
- 检查产品物模型接口是否返回异常

### 切换服务后模板没有变化

- 检查当前服务是否真的定义了 `inputData`
- 如果编辑器里已经是用户手工修改内容，页面会尽量保留草稿，不会强制覆盖

### 页面能选设备但发送失败

- 检查设备消息接口是否可达
- 检查浏览器控制台和网络面板是否有 4xx/5xx
- 检查下发 JSON 是否为对象而不是数组或字符串

## 回滚方式

直接回退本次前端提交并重新发布静态资源即可。
