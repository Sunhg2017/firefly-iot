# 设备注册阶段维护设备标识运维说明

## 1. 变更概述

本次变更让设备在以下入口注册时即可同步维护设备标识：

- 设备管理手动创建
- 一型一密动态注册
- 设备批量导入
- 设备模拟器 HTTP / MQTT 一型一密自动注册

设备标识仍落在既有 `device_locator` 体系中，无新增数据库表、无新增中间件依赖。

## 2. 发布范围

建议按以下模块一起发布：

1. `firefly-api`
2. `firefly-device`
3. `firefly-connector`
4. `firefly-web`
5. `firefly-simulator`

如果只发布后端、不发布前端：

- 接口链路仍可工作
- 但用户无法在新建设备、动态注册调试、模拟器自动注册页面直接维护标识

如果只发布前端、不发布后端：

- 前端会提交 `locators`
- 旧后端无法完整处理该字段，存在功能缺失风险

## 3. 配置与依赖

### 3.1 新增配置

无。

### 3.2 依赖前提

- `device_locator` 相关表结构与既有服务可正常使用
- 异步任务中心、文件服务、MinIO 维持正常，批量导入仍依赖原有链路
- connector 与 device 服务之间的 API DTO 版本保持一致

## 4. 发布后核验

### 4.1 手动创建核验

1. 打开设备管理页
2. 通过“新建设备”抽屉创建一台一机一密设备
3. 同时填写至少一条设备标识
4. 创建成功后进入设备标识弹窗，确认标识已存在

### 4.2 动态注册核验

1. 打开产品列表
2. 进入一型一密产品的“设备接入”
3. 在“动态注册”页填写设备名称和设备标识
4. 注册成功后到设备管理确认：
   - 设备已创建
   - 设备标识已写入

### 4.3 批量导入核验

1. 准备包含以下任一列组合的 Excel/CSV：
   - `locatorType + locatorValue`
   - `imei`
   - `iccid`
   - `mac`
   - `serial` / `sn`
2. 发起异步导入
3. 导入完成后抽样检查设备标识

### 4.4 模拟器核验

1. 在模拟器新增 HTTP 或 MQTT 设备
2. 选择一型一密认证
3. 填写 ProductSecret 和设备标识
4. 连接设备，确认自动动态注册成功
5. 到平台设备详情核对标识

## 5. 监控与日志

重点关注以下日志与现象：

- `DeviceService`
  - 手动创建、批量创建后的 locator 绑定失败
- `DeviceCredentialService`
  - 动态注册完成但 locator 绑定失败
- `DeviceImportService`
  - 某行导入解析失败
  - 标识列缺失或成对字段不完整
- 前端页面
  - 创建设备抽屉提交失败
  - 动态注册失败
- 模拟器
  - 动态注册返回成功但平台侧没有标识

## 6. 常见故障排查

### 6.1 动态注册成功，但没有设备标识

排查顺序：

1. 确认请求体中是否真的带了 `locators`
2. 检查 `firefly-connector` 与 `firefly-device` 是否同时升级
3. 查看 `DeviceCredentialService` 日志是否出现 locator 绑定异常
4. 查看是否触发了标识唯一性冲突

### 6.2 批量导入整行失败

重点检查：

- `locatorType` 与 `locatorValue` 是否只填了一列
- `imei / iccid / mac / serial` 是否带了多余空格
- 同一产品下是否已经存在相同标识值

### 6.3 标识冲突导致注册失败

设备标识在同一产品下要求唯一。

处理方式：

1. 先确认冲突值是否已被其他设备占用
2. 若为历史脏数据，清理旧数据后再重试
3. 不要在代码里保留兼容分支或绕过唯一性校验

### 6.4 模拟器自动注册没有带出标识

检查：

1. 是否选择了 HTTP / MQTT 的一型一密模式
2. 模拟器设备配置里是否填写了设备标识
3. 本地运行的模拟器是否为最新构建版本

## 7. 回滚方案

若需回滚，建议按同一提交整体回滚 `firefly-api / firefly-device / firefly-connector / firefly-web / firefly-simulator`。

注意事项：

- 已经写入 `device_locator` 的新标识数据不会自动删除
- 如果确认本次功能不再保留，需要运维手动清理对应设备标识数据
- 不建议只回滚单个前端或单个后端模块，以免出现字段口径不一致
