# 设备影子卡片全屏查看运维说明
## 1. 适用范围

本说明适用于 `firefly-web` 中设备影子页面四个 JSON 卡片新增全屏按钮后的发布与验收。

## 2. 发布影响

本次仅涉及前端页面交互增强：

- 新增设备影子四张卡片的全屏查看入口
- Desired 全屏态复用原有编辑动作

不涉及：

- 数据库结构变更
- Flyway SQL
- 后端接口发布
- 菜单权限台账补录

## 3. 发布步骤

1. 在前端工程执行构建验证：

```bash
cd firefly-web
npm run build
```

2. 发布最新前端静态资源。

## 4. 发布后验证

### 4.1 基础验收

- 打开“设备影子”页面并查询任意设备。
- 确认 `Desired / Reported / Delta / Metadata` 四张卡片都出现“全屏”按钮。

### 4.2 Desired 验收

- 点击 Desired 的“全屏”按钮。
- 确认弹层中仍可执行编辑、保存、取消、清空。
- 在全屏中修改草稿后关闭弹层，卡片内内容应保持同步。

### 4.3 只读卡片验收

- 分别打开 Reported、Delta、Metadata 的全屏弹层。
- 确认三者只能查看，不能修改。

### 4.4 切换设备验收

- 打开任一卡片全屏后，切换设备或清空当前设备选择。
- 预期结果：全屏弹层自动关闭，避免显示上一台设备的影子内容。

## 5. 问题排查

如果用户反馈看不到全屏按钮，优先检查：

1. 浏览器是否仍在使用旧版静态资源。
2. 当前页面是否成功查询到了影子数据。
3. 前端构建产物是否为本次发布版本。

如果 Desired 全屏态无法保存，按原有影子编辑链路继续排查：

1. Desired JSON 是否格式正确。
2. `/devices/{id}/shadow/desired` 接口是否返回错误。
3. 保存后是否重新刷新了影子数据。

## 6. 回滚说明

本次只涉及前端页面交互。若必须回滚，回滚 `firefly-web` 静态资源即可；后端服务与数据库无需回滚。
