# 设备影子扩展抽屉运维说明
## 1. 适用范围

本说明适用于 `firefly-web` 中设备影子查看能力的发布与验收。本次能力覆盖两个入口：

- 独立设备影子页
- 设备列表中的影子详情抽屉

## 2. 发布影响

本次仅涉及前端交互调整：

- 四张影子卡片都支持展开查看
- 展开形态统一为右侧 `75vw` 宽度抽屉
- 独立影子页与设备列表影子详情共用同一套扩展抽屉组件
- `Desired` 在卡片与扩展抽屉之间共享编辑草稿

不涉及：

- 数据库结构变更
- Flyway SQL
- 后端接口发布
- 菜单权限台账补录

## 3. 发布步骤

1. 在前端工程执行构建验证：

```bash
cd firefly-web
npm run build
```

2. 发布最新前端静态资源。

## 4. 发布后验收

### 4.1 独立设备影子页

- 打开“设备影子”页面并选择任意设备。
- 确认 `Desired / Reported / Delta / Metadata` 四张卡片都出现展开按钮。
- 确认四张卡片在常规视图中保持对齐，说明区和编辑区没有明显错位。
- 点击任意按钮后，确认打开的是右侧 `75vw` 抽屉，而不是弹窗。

### 4.2 设备列表影子详情

- 从设备列表进入任意设备的影子详情抽屉。
- 确认四张卡片同样都有展开按钮。
- 确认四张卡片在两列布局下高度一致，便于直接横向比对。
- 点击后确认出现第二层 `75vw` 扩展抽屉，并能正常关闭返回主影子抽屉。

### 4.3 Desired 验收

- 分别在两个入口中打开 `Desired` 的扩展抽屉。
- 确认仍可执行编辑、保存、取消、清空。
- 在扩展抽屉中修改草稿后关闭，再回到卡片视图，内容应保持同步。

### 4.4 只读卡片验收

- 分别打开 `Reported`、`Delta`、`Metadata` 的扩展抽屉。
- 确认三者仅可查看，不能修改。

### 4.5 状态切换验收

- 在独立设备影子页中打开扩展抽屉后切换设备或清空设备选择。
- 在设备列表中打开扩展抽屉后关闭主影子抽屉或删除影子。
- 预期结果：扩展抽屉自动关闭，不残留上一台设备的数据。

## 5. 问题排查

如果用户反馈看不到展开按钮，优先检查：

1. 浏览器是否仍在使用旧版静态资源。
2. 当前页面是否已成功加载设备影子数据。
3. 前端构建产物是否为本次发布版本。

如果用户反馈展开后不是抽屉或宽度异常，优先检查：

1. `ShadowPanelFullscreenDrawer` 是否已随本次前端产物发布。
2. 是否存在旧缓存导致仍引用历史页面脚本。
3. 是否有本地样式覆盖了 `Drawer` 宽度。

如果 `Desired` 扩展态无法保存，继续按原有影子编辑链路排查：

1. `Desired` JSON 是否格式正确。
2. `/devices/{id}/shadow/desired` 接口是否返回错误。
3. 保存后是否重新刷新了影子数据。

## 6. 回滚说明

本次只涉及前端页面与组件。若必须回滚，回滚 `firefly-web` 静态资源即可；后端服务与数据库无需回滚。
