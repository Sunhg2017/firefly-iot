# 设备影子版本稳定性修复运维说明
> 模块: firefly-device / firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 适用场景

用于排查以下现象：

- 设备已经断开，但设备影子版本仍持续增长。
- 影子页刷新时 `version` 很快跳变，像被定时任务持续写入。

## 2. 修复内容

- 模拟器在设备断开时会自动停止自动上报定时器。
- 服务端对于内容未变化的 `reported` 重复上报不再递增影子版本。

## 3. 验证步骤

### 3.1 模拟器验证

```bash
cd firefly-simulator
npm run build:vite
```

验证要点：

1. 连接设备并开启自动上报。
2. 执行断开操作，或制造 MQTT / WebSocket / TCP 被动断开。
3. 确认模拟器日志出现自动上报停止提示。
4. 观察设备影子页面，`version` 不应继续快速增长。

### 3.2 服务端验证

```bash
cd firefly-device
mvn test
```

验证要点：

1. 对同一设备重复上报相同属性值。
2. 确认影子 `reported` 未变化时，`version` 保持不变。

## 4. 排查建议

- 若版本仍持续增长，优先检查是否存在其他上报源，而不只是模拟器自动上报。
- 若 `reported` 内容每次都不同，例如带时间戳字段，那么版本继续增长是合理行为。
