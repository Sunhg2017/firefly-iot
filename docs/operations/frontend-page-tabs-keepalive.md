# 前端页面页签与保活运维说明
> 模块: `firefly-web`
> 日期: 2026-03-16
> 状态: Done

## 1. 适用范围

用于 Firefly IoT Web 控制台顶部页签与页面保活能力的发布验证、问题排查和回滚说明。

## 2. 本次变更

- 主布局新增顶部页签区，展示当前会话已打开页面。
- 已访问页面在切换后不再立即卸载，返回页签时保留原本输入与本地状态。
- 页签名称会跟随当前菜单树中的菜单自定义名称动态展示。
- 页签条样式已收口为紧凑型标签带，避免顶部右侧出现大块空白承载区。
- 补齐 `ScheduleOutlined` 图标映射，菜单树中的“定时任务”菜单会恢复图标显示。
- 修复新页面首次打开时页签激活态回退到首个页签的问题，当前路由会优先参与页签渲染与激活计算。
- 顶部页签现已改为自绘按钮组，不再依赖 `antd Tabs` 内部状态切换；若再出现页签误跳转，应优先排查路由守卫与权限收口逻辑，而不是页签组件本身。
- 修复重复路径在多空间下的归属误判问题。系统运维空间访问 `/user`、`/role`、`/menu-customization` 等双空间复用页面时，不会再被误判成租户空间路由并重定向回首页。
- 头部右侧用户名模块已改为独立样式类控制；若界面仍出现异常大白块，请确认前端静态资源缓存已刷新到最新版本。

## 3. 构建与发布

前端构建命令：

```bash
cd firefly-web
npm run build
```

本次能力仅涉及前端静态资源发布，不涉及后端接口、数据库结构或 Flyway 变更。

## 4. 验证步骤

1. 登录系统并进入任意支持编辑或筛选的业务页面，例如产品、设备或协议解析页面。
2. 在页面内输入表单内容或调整筛选条件，但先不要提交。
3. 通过左侧菜单切换到另一个业务页面，确认顶部新增页签。
4. 点击顶部原页签返回，确认之前输入内容仍在。
5. 关闭一个非首页页签，确认当前页签关闭后会自动切换到相邻页签。
6. 切换租户空间或更新菜单授权后，确认无权限页面不会继续保留在顶部页签中。

## 5. 常见问题排查

### 5.1 顶部没有出现页签

- 确认当前访问的是业务路由，而不是 `/403` 之类的异常页。
- 确认当前用户对该页面仍具备菜单授权和权限点。
- 确认前端静态资源已更新到包含本次改动的版本。

### 5.2 返回页签后输入没有保留

- 确认用户是通过顶部页签或左侧菜单在当前登录会话内切换，而不是浏览器整页刷新。
- 确认目标页面没有在切换期间被手动关闭页签。
- 若页面内部逻辑主动在 `useEffect` 或初始化逻辑中重置表单，需要继续检查对应业务页面实现。

### 5.3 页签标题与菜单名称不一致

- 先检查 `/workspace-menu-customizations/current/tree` 返回的菜单树是否已经刷新到最新自定义名称。
- 若菜单配置刚刚变更，触发菜单树刷新后重新验证顶部页签标题。
