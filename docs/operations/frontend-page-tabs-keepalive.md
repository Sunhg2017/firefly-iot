# 前端页面标签与保活运维说明
> 模块: `firefly-web`
> 日期: 2026-03-16
> 状态: Done

## 1. 适用范围

用于 Firefly IoT Web 控制台顶部页面标签、页面保活能力以及全局头部动作区样式的发布验证、问题排查与回滚说明。

## 2. 本次变更

- 主布局新增顶部页面标签区，展示当前会话内已打开的业务页面。
- 已访问页面切换后不再立即卸载，返回标签时仍保留未提交输入和本地状态。
- 标签名称跟随当前菜单树中的菜单自定义名称动态显示。
- 顶部标签改为布局层自绘按钮组，修复新页面首次打开时闪回第一个标签的问题。
- 修复共享路由在系统运维空间下被误判为跨空间访问后强制跳回首页的问题。
- 补齐 `ScheduleOutlined` 图标映射，恢复“定时任务”菜单图标显示。
- 全局头部保持单行布局，右侧将通知、异步任务、用户名聚合在同一块区域。
- 租户空间全局头部新增告警入口，待处理告警数会以角标形式提示，并可直接查看最近告警。
- 用户名区域改为轻量行内样式，不再显示胶囊背景。
- 用户头像下拉菜单新增“修改密码”入口，调用 `/api/v1/users/me/password` 支持当前登录用户自助修改密码。

## 3. 构建与发布

前端构建命令：

```bash
cd firefly-web
npm run build
```

本次变更仅涉及前端静态资源发布，不涉及后端接口、数据库结构或 Flyway 变更。

## 4. 发布后验证

1. 登录系统并打开任意支持编辑或筛选的业务页面。
2. 在页面内输入表单内容或调整筛选条件，但先不要提交。
3. 通过左侧菜单继续打开第二个、第三个业务页面，确认顶部新增对应标签。
4. 点击新打开的标签，确认页面直接展示对应内容，不会闪回第一个标签。
5. 切回之前的标签，确认输入内容和局部状态仍然保留。
6. 关闭当前非首页标签，确认系统自动切换到相邻标签。
7. 检查全局头部右侧通知、异步任务、用户名是否保持同一行展示，用户名区域无胶囊背景。
8. 使用租户账号触发一条待处理告警，确认右上角出现告警角标提示，点击后可看到最近告警摘要并跳转到告警处理页。
9. 点击右上角头像下拉菜单，确认可见“修改密码”与“退出登录”两个选项，修改密码提交后可收到成功反馈。
10. 进入系统运维空间，确认“定时任务”菜单图标正常显示，且系统运维头部不显示租户告警入口。

## 5. 常见问题排查

### 5.1 顶部没有出现页面标签

- 确认当前访问的是业务路由，而不是 `/403` 等异常页。
- 确认当前用户仍具备该页面所需菜单授权和权限点。
- 确认前端静态资源已更新到包含本次变更的版本。

### 5.2 切回标签后输入没有保留

- 确认用户是在同一次登录会话内通过顶部标签或左侧菜单切换，而不是浏览器整页刷新。
- 确认目标页面在切换期间没有被手动关闭标签。
- 若页面内部逻辑会在进入时主动重置表单，需要继续排查对应业务页面实现。

### 5.3 打开新标签后又跳回第一个标签

- 优先确认当前部署版本已包含布局层自绘标签组的修复代码。
- 如问题仍存在，重点排查是否有额外路由守卫、权限重定向或页面自身跳转逻辑干扰当前路径。

### 5.4 右上角动作区不在同一行

- 确认 `firefly-web/src/styles/global.css` 已包含 `layout-header-left`、`layout-header-right`、`layout-user-inline` 等样式。
- 确认浏览器加载的是最新构建产物，避免旧缓存导致头部布局仍沿用历史样式。
