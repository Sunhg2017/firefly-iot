# 管理员会话页重构运维说明

## 1. 适用范围

本说明用于处理以下现象：

- `安全管理 -> 管理员会话` 页签默认没有数据
- 页面要求输入 `userId` 才能查会话，运维人员无法直接查看当前在线管理员
- 管理员会话和我的会话中的 IP、登录方式展示不完整

涉及模块：

- `firefly-system`
- `firefly-web`

## 2. 变更内容

- 后端新增管理员会话专用接口 `/api/v1/admin-sessions/*`
- 页面默认展示当前在线管理员会话，不再要求手输 `userId`
- 管理员口径收敛为：
  - 系统运维账号
  - 租户超级管理员
- 前端支持关键字、平台、身份筛选
- 支持单会话下线和按管理员账号全部下线
- 我的会话、管理员会话统一改为读取真实字段 `loginMethod`、`loginIp`

## 3. 部署说明

推荐同时发布：

1. 发布 `firefly-system`
2. 发布 `firefly-web`

说明：

- 只发布前端时，旧接口会导致管理员会话列表无法加载
- 只发布后端时，旧页面仍然会停留在手输 `userId` 的交互

## 4. 数据与 SQL

- 本次不涉及菜单、权限点、所属空间变更
- 本次不涉及 Flyway SQL
- 本次不涉及数据库结构迁移
- 本次不需要清理历史数据

## 5. 验证步骤

### 5.1 后端接口验证

- 使用具备 `user:read` 权限的系统运维账号登录
- 调用 `POST /api/v1/admin-sessions/list`
- 确认响应中可看到当前在线的系统运维账号或租户超级管理员会话

### 5.2 页面验证

- 进入 `系统运维空间 -> 安全审计 -> 安全管理 -> 管理员会话`
- 确认页面打开后默认显示在线管理员会话
- 使用关键字搜索管理员账号、姓名、租户名称或登录 IP，确认能命中对应记录
- 分别测试 `下线会话` 和 `全部下线`

### 5.3 我的会话验证

- 进入 `安全管理 -> 我的会话`
- 确认列表中的 `登录方式`、`IP`、`User-Agent`、`最后活跃时间` 正常显示

## 6. 排障

### 6.1 管理员会话仍为空

优先检查：

- 当前登录账号是否具备 `user:read`
- 是否已经同时发布前后端
- `POST /api/v1/admin-sessions/list` 的响应是否为空数组
- 目标管理员是否真的存在在线会话

### 6.2 只能看到系统运维账号，看不到租户管理员

优先检查：

- 对应租户是否配置了 `admin_user_id`
- 当前在线用户是否确实是该租户的超级管理员账号
- 在线会话的 `status` 是否为 `ACTIVE`

### 6.3 强制下线无效

优先检查：

- 页面请求是否命中：
  - `DELETE /api/v1/admin-sessions/{sessionId}`
  - `POST /api/v1/admin-sessions/users/{username}/kick`
- Redis 中对应 refresh token 是否已被删除
- 被下线账号是否重新刷新了页面或重新发起鉴权请求

## 7. 回滚说明

- 若仅回滚前端，需要同时回退到旧的管理员会话接口版本，否则页面无法调用
- 若仅回滚后端，需要同时回退前端，否则管理员会话页会请求不存在的新接口
- 本次未修改数据库结构，无需数据库回滚
