# 设备模拟器日志跟随方向修复运维说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 适用场景

用于排查“日志最新一条在最上面，但自动刷新总跳到最底部”的问题。

## 2. 修复内容

- 日志面板自动跟随从 `scrollHeight` 底部定位改为顶部定位。
- 当前日志面板采用头插数据模型，因此顶部才是最新日志区域。

## 3. 验证步骤

```bash
cd firefly-simulator
npm run build:vite
```

回归检查：

1. 打开模拟器日志面板并保持“自动滚动”开启。
2. 触发设备连接、上报或断开，连续生成多条日志。
3. 确认视图始终停留在顶部，新日志持续出现在可视区域。
4. 关闭自动滚动后，手动滚到中部或底部，确认不会再被强制拉回顶部。

## 4. 排查建议

- 若仍出现跳到底部，优先检查本地是否仍在使用旧版 renderer 资源。
- 若后续把日志顺序改回“旧日志在上、新日志在下”，需要同步把自动跟随逻辑再切回底部。
