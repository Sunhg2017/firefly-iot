# 设备模拟器日志跟随方向修复运维说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 适用场景

用于排查“日志应该最新在最下面，但界面顺序和自动滚动不一致”的问题。

## 2. 修复内容

- 日志面板展示层改为按时间正序渲染，最新日志位于底部。
- 自动跟随保持 `scrollHeight` 底部定位，与新的展示顺序一致。
- 导出日志同步按界面顺序输出。

## 3. 验证步骤

```bash
cd firefly-simulator
npm run build:vite
```

回归检查：

1. 打开模拟器日志面板并保持“自动滚动”开启。
2. 触发设备连接、上报或断开，连续生成多条日志。
3. 确认旧日志在上、最新日志持续追加在底部。
4. 确认自动滚动时视图始终跟随到底部。
5. 关闭自动滚动后，手动滚到中部，确认不会再被强制拉到底部。

## 4. 排查建议

- 若界面仍显示最新日志在顶部，优先检查本地是否仍在使用旧版 renderer 资源。
- 若后续改动日志存储结构，需要同步检查展示顺序和导出顺序是否仍保持一致。
