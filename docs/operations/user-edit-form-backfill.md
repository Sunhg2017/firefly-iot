# 用户编辑表单回填修复运维说明
> 模块: firefly-web / 用户管理
> 日期: 2026-03-17
> 状态: Done

## 1. 适用场景

用于排查“用户管理页点击编辑后表单为空”“编辑抽屉只显示空白字段”“角色没有回填”的问题。

## 2. 发布说明

- 本次变更仅涉及前端页面 `firefly-web/src/pages/user/UserList.tsx`。
- 发布时需要重新构建并发布前端静态资源。

## 3. 验证步骤

执行：

```bash
cd firefly-web
npm run build
```

回归检查：

1. 进入“用户管理”页面。
2. 选择一个已有用户，点击“编辑”。
3. 确认抽屉打开后，用户名、姓名、手机号、邮箱和角色都已正确回填。
4. 修改一个非用户名字段后保存，确认保存成功且列表刷新正常。
5. 再次进入同一用户编辑页，确认刚刚保存的数据仍可正确回填。

## 4. 常见故障排查

### 4.1 编辑抽屉仍然空白

- 先确认浏览器加载的是否为最新前端构建产物。
- 检查 `GET /api/v1/users/{id}` 是否返回 200，以及返回体中是否包含 `roles`、`realName`、`phone`、`email`。
- 如果接口失败但抽屉仍应看到列表摘要数据，检查前端是否弹出了“已使用列表数据回填”的错误提示。

### 4.2 角色没有回填

- 检查用户详情接口的 `roles` 数组是否返回了 `roleId`。
- 检查角色选项接口是否已正常加载，否则表单中虽有值但无法映射出可选项。

## 5. 回滚说明

如需回滚，回退 `firefly-web/src/pages/user/UserList.tsx` 与本次三份文档后，重新执行前端构建并发布。
