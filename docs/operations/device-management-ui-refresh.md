# 设备管理与设备影子界面优化运维说明
> 模块: `firefly-web`
> 日期: 2026-03-14
> 状态: Done

## 1. 适用范围

用于设备影子页面与设备管理页影子抽屉的发布验证、问题排查和回归测试。

## 2. 本次变更内容

- 修复设备影子页面和抽屉中的中文乱码。
- 统一 `CodeEditorField` 的只读态样式，显式展示“只读 / 查看态”。
- 页面上仅保留 `desired` 的维护能力。
- 删除用户侧 `reported` 更新接口，避免绕过设备链路直接改写影子上报状态。

## 3. 发布检查

### 3.1 前端

```bash
cd firefly-web
npm run build
```

通过标准：

- TypeScript 编译通过。
- Vite 构建成功。
- 设备影子独立页和设备管理页影子抽屉可正常打开。

### 3.2 后端

```bash
mvn -pl firefly-device -am test
```

通过标准：

- `firefly-device` 模块测试通过。
- 设备影子查询、更新 `desired`、获取 `delta`、删除影子能力可正常编译和运行。

## 4. 回归检查

### 4.1 设备影子独立页

1. 打开“设备影子”页面，确认页面中文显示正常，没有乱码。
2. 通过设备名称或别名选择设备，点击“查询影子”。
3. 确认 `Desired` 区域默认显示“查看态”，点击“编辑”后才允许输入。
4. 确认 `Reported`、`Delta`、`Metadata` 三个区域右上角显示只读标签，且无法编辑。
5. 修改 `Desired` 后保存，确认页面刷新并更新影子状态。

### 4.2 设备管理页影子抽屉

1. 从设备管理页打开任意设备的影子抽屉。
2. 确认抽屉内中文显示正常，没有乱码。
3. 确认只有 `Desired` 提供编辑、保存、清空操作。
4. 确认 `Reported`、`Delta`、`Metadata` 只可查看和复制，不能输入。

## 5. 故障排查

### 5.1 页面看起来还能编辑

- 检查是否已经加载最新前端包。
- 检查 `CodeEditorField` 是否显示“只读”或“查看态”标签。
- 如果只读区域没有标签，优先确认浏览器是否仍在使用旧缓存资源。

### 5.2 用户反馈无法手工改 reported

- 这是本次调整后的预期行为。
- `reported` 只允许由设备运行时消息链路更新。
- 如需修改设备当前状态，应让设备重新上报，或通过调整 `desired` 驱动设备收敛。

## 6. 回滚说明

- 前端回滚到上一个可用版本即可恢复旧页面行为。
- 后端如需回滚，恢复 `DeviceShadowController` 中的 `reported` 更新接口即可，但不建议长期保留该入口。
