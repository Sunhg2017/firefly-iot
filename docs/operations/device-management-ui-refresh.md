# 设备管理与设备影子界面优化运维说明
> 模块: firefly-web
> 日期: 2026-03-14
> 状态: Done

## 1. 适用场景

用于设备管理页和设备影子页的前端发布、回归验证和问题排查。

## 2. 本次优化内容

- 设备管理页增加产品业务信息展示与筛选摘要。
- 设备影子页改为可搜索选择设备，不再依赖手输设备 ID。
- 设备影子独立页和抽屉统一升级为结构化影子概览 + Monaco JSON 编辑区。

## 3. 发布验证

```bash
cd firefly-web
npm run build
```

通过标准：

- TypeScript 编译通过。
- Vite 构建成功。
- 设备管理页与设备影子页可正常打开。

## 4. 回归检查

### 4.1 设备管理页

1. 打开设备管理页，确认顶部统计卡、筛选区、设备目录三区结构清晰。
2. 检查设备表格是否新增产品列，并显示产品名称与 `productKey`。
3. 确认产品列位于设备名称之后，便于快速识别设备归属。
4. 选择若干设备后，确认目录卡片右上角会显示已选数量。
5. 改变筛选条件后，确认筛选摘要区同步更新。

### 4.2 设备影子页

1. 打开设备影子页，确认可以通过搜索设备名称或别名选择设备。
2. 查询后确认能看到版本、desired、reported、delta 四项概览。
3. Desired 区编辑 JSON 后保存，确认可成功刷新。
4. Reported、Delta、Metadata 区应为只读展示。

### 4.3 设备影子抽屉

1. 从设备管理页点击“影子”打开抽屉。
2. 确认抽屉顶部展示设备名称、设备编码、所属产品、同步状态和更新时间。
3. 确认 Desired / Reported / Delta / Metadata 四块内容可读性正常。

## 5. 已知事项

- 构建时 Monaco 相关 chunk 较大，当前为预期警告，不影响功能使用。
- 设备影子能力内部仍通过 `deviceId` 调用后端接口，但用户侧入口已改为按设备名称/别名操作，列表抽屉默认不再展示数据库主键。
