# 列表页查询按钮统一运维说明

## 变更内容

- 前端多个列表页的搜索框统一补充“查询”按钮。
- 本次仅涉及 `firefly-web` 前端展示层，无后端接口、数据库、权限点、菜单台账和配置项变更。

## 部署方式

按现有前端发布流程重新构建并部署 `firefly-web` 即可，不需要额外环境变量或数据迁移。

## 依赖与影响

- 依赖现有 Web 前端构建链路。
- 不依赖数据库结构调整。
- 不依赖缓存清理、消息重放或异步任务补偿。

## 验证建议

- 在各列表页输入关键字后，点击“查询”按钮，确认列表能刷新。
- 在不变更筛选条件的情况下再次点击“查询”，确认当前页可重新拉取数据。
- 验证筛选区在常用桌面分辨率下不会出现关键按钮被遮挡的问题。

## 常见问题排查

### 点击查询无响应

- 检查浏览器控制台是否存在前端异常。
- 检查对应列表页接口是否返回 4xx/5xx。
- 检查按钮点击后是否进入原有 `onSearch` 逻辑。

### 筛选区换行异常

- 优先检查页面容器宽度和浏览器缩放比例。
- 对于窄屏场景，确认 `Space wrap` 是否仍生效。

## 回滚方式

若需回滚，直接回退本次前端提交并重新发布前端静态资源即可。
