> 模块: `firefly-device` / `firefly-web`
> 日期: 2026-04-06
> 状态: Done

## 1. 适用范围

用于设备日志查询优化后的发布检查、回归验证、故障定位和回滚处理。

## 2. 发布内容

- 后端设备日志查询接口返回设备名称、别名、产品名称和 `productKey`
- 后端补齐设备日志的租户过滤、级别归一化和上报时间保留
- 前端设备日志页改为设备/产品选择式筛选
- 前端新增日志详情查看

本次没有新增菜单、权限点和 Flyway 菜单迁移。

## 4. 发布后回归

## 4.1 页面入口

1. 登录租户空间
2. 打开“设备日志”
3. 确认页面可以正常加载日志列表

## 4.2 查询回归

重点检查以下场景：

1. 通过设备搜索框按设备名称或别名筛日志
2. 通过产品下拉筛日志
3. 通过级别、关键字和时间范围组合筛选
4. 点击“重置”后，筛选条件恢复默认

预期结果：

- 页面不再要求手工输入 `deviceId`
- 查询后列表可直接展示设备名称、别名、产品名称和 `productKey`

## 4.3 清理回归

1. 点击“清理过期”
2. 确认页面提示已清理记录数
3. 再次刷新列表，确认剩余日志仍可正常查询

重点确认：

- 清理只影响当前租户下 30 天前的日志
- 不应出现跨租户日志被一起清理的现象

## 5. 日志定位

重点关注 `firefly-device` 日志中的以下关键字：

- `Cleaned {} expired device logs`
- `Failed to record device log`

排查建议：

### 5.1 页面列表没有设备名称或产品信息

优先检查：

1. 日志记录关联的 `device_id` 是否仍对应有效设备
2. 对应设备是否已删除
3. `products` 中是否仍存在对应产品

### 5.2 上报时间异常

优先检查：

1. 调用方是否传入 `reportedAt`
2. 服务日志中是否存在设备日志创建异常
3. 数据库 `device_logs.reported_at` 与 `created_at` 是否都被正确写入

### 5.3 最近日志或统计结果异常

优先检查：

1. 当前租户上下文是否正确注入
2. 查询设备是否属于当前租户
3. 日志级别是否被上游写成了非标准值

## 6. 回滚说明

如需回滚本次能力：

1. 回滚 `firefly-web` 到上一可用版本
2. 回滚 `firefly-device` 到上一可用版本
3. 回滚后重新验证设备日志页是否恢复旧行为

本次没有新增数据库结构和菜单迁移，回滚不需要执行额外 Flyway 回退。
