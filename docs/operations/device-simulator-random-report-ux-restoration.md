# 设备模拟器随机上报入口恢复运维说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 现象

用户反馈设备模拟器“原本支持随机模拟数据上传，但现在看不到了”。

根因不是能力删除，而是前端入口弱化：

- 随机模板仍在
- 自动上报仍在
- 但默认没有选中模板，页面只先显示自定义 JSON

## 2. 修复内容

- 数据上报区新增“随机模拟 / 自定义 JSON”显式切换
- 默认优先进入“随机模拟”
- 自动带出当前上报类型的第一条模板
- 增加随机样例预览和“换一组”按钮

## 3. 验证方式

```bash
cd firefly-simulator
npm run build:vite
```

说明：

- `build:vite` 已通过，可验证 TypeScript 和前端构建
- 完整 `npm run build` 在当前机器上会卡在 Electron 原生依赖重编译，原因是缺少 Visual Studio C++ 构建环境

## 4. 回归检查

1. 打开设备模拟器
2. 选择任意支持通用上报的设备
3. 进入 `Data Report`
4. 确认默认出现“随机模拟”模式
5. 确认模板已自动选中
6. 点击“换一组”，确认预览样例变化
7. 点击“Send”，确认发送成功
8. 打开自动上报，确认日志持续产生且 payload 有变化

## 5. 常见问题

### 5.1 为什么某些上报类型还是只能看到自定义 JSON？

因为当前类型没有可用模板。可在模拟器左下角“模板管理”里补一条对应类型模板。

### 5.2 为什么预览的样例和真正发送的数据不完全一样？

因为预览只是一份示例，真正发送时会重新生成随机值，这是预期行为。
