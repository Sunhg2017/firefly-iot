# 租户管理运维说明

## 1. 模块说明

租户管理负责平台侧的租户生命周期、套餐、配额、空间授权和租户级 Webhook 运维入口。

## 2. 本次变更

- 在租户管理列表新增 `Webhook` 入口
- 系统运维可直接为指定租户维护 `WEBHOOK` 渠道
- 平台默认通知渠道页面不再维护 Webhook

## 3. 运维操作

1. 使用系统运维账号进入租户管理列表
2. 在目标租户行点击 `Webhook`
3. 在抽屉内新增、编辑、启停、测试或删除该租户的 Webhook

## 4. 配置要求

- 必填：`url`
- 推荐：`method`、`contentType`、`secret`

## 5. 排查建议

- 若租户通知未能成功回调，先确认该租户是否配置了自己的 Webhook
- 再检查通知记录中的失败原因和目标地址连通性
- Webhook 不参与平台默认渠道回落，未配置时不会自动继承平台 Webhook

## 6. 租户编码复用

- 租户注销采用逻辑删除。
- 自本次修复起，已注销租户的 `code` 不再继续占用创建名额。
- 若需要重新创建相同 `code` 的租户，请先确认数据库已执行 `V26__allow_reuse_deleted_tenant_code.sql`。
- 如果部署后仍提示租户编码已存在，优先检查 Flyway 是否已成功执行到 `V26`，以及旧租户记录的 `deleted_at` 是否已正确写入。
