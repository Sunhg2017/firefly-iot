# 告警管理运维说明

## 1. 模块说明

告警管理包含两个菜单：

- 告警规则
- 告警处理

本次发布主要影响“告警规则”页面交互，规则协议本身不变，仍为 `version = 3` 的结构化规则。

页面层面的主要变化：

1. 新增顶部配置步骤总览
2. 级别块改为卡片化展示
3. 告警级别、触发语义、条件类型改为更直观的选项式控件
4. 条件参数和预览区域进一步分层
5. 创建/编辑入口改为抽屉，减少复杂条件挤压并保留列表上下文

## 2. 发布影响

### 2.1 前端

需要发布 `firefly-web`，影响点包括：

- 告警规则配置弹窗布局变化
- 级别块交互变化
- 文案与引导信息变化

### 2.2 后端

`firefly-rule` 协议校验逻辑不变，但需要与当前前端版本配套使用。

### 2.3 数据库

本次无数据库迁移，无新增字段。

## 3. 发布前检查

### 3.1 编译检查

后端：

```bash
mvn -pl firefly-rule -DskipTests compile
```

前端：

```bash
cd firefly-web
npm run build
```

### 3.2 联调检查

建议至少验证以下场景：

1. 新建规则时顶部步骤卡和统计汇总正常显示
2. 新增多个级别块时卡片层级清晰
3. `AT_LEAST` 模式下满足条数可正常填写
4. 条件预览、级别块预览、规则预览都能实时更新

## 4. 发布步骤

1. 发布 `firefly-rule`
2. 发布 `firefly-web`
3. 使用规则维护账号验证新建和编辑流程
4. 使用值班账号验证“告警处理”页面无回归

## 5. 发布后验收

### 5.1 页面验收

1. 页面中文正常显示，无乱码
2. 告警规则页面应以抽屉打开，而不是居中弹窗
3. 顶部能看到“配置步骤”和实时汇总
4. 级别块卡片中能直观看到级别、触发语义、条件数
5. 条件类型切换后能显示对应说明
6. 产品未选时页面会提示先选产品再选指标
7. 保存和编辑流程正常

### 5.2 接口验收

1. 创建规则成功
2. 更新规则成功
3. 多个级别块保存成功
4. 非法结构仍会被后端拒绝

## 6. 监控与日志

### 6.1 关注现象

1. 页面布局错位
2. 规则预览未更新
3. 条件编辑区切换条件类型后显示异常
4. 编辑旧规则时级别块未正确回填

### 6.2 关注日志

继续关注 `firefly-rule` 中和规则校验相关的 `PARAM_ERROR`：

- 规则为空
- 缺少级别块
- 缺少触发语义
- 满足条数非法
- 条件字段缺失

## 7. 常见故障排查

### 7.1 页面看起来还是旧样式

排查步骤：

1. 确认前端资源是否已更新
2. 清理浏览器缓存后重新打开
3. 确认部署版本是否包含本次提交

### 7.2 预览内容没有变化

排查步骤：

1. 检查表单是否正确回填 `ruleGroups`
2. 检查条件类型是否已选择
3. 检查指标和阈值等关键字段是否填写

### 7.3 指标无法选择

排查步骤：

1. 检查是否已选择产品
2. 检查产品物模型是否存在属性
3. 检查物模型接口返回是否正常

### 7.4 保存时报错

排查步骤：

1. 检查级别块是否完整
2. 检查 `AT_LEAST` 的满足条数是否超过条件数
3. 检查每条条件的必填字段是否填写
4. 检查后端是否仍为当前结构化规则版本

## 8. 回滚方案

若需要回滚，请同时回滚：

1. `firefly-web`
2. `firefly-rule`

注意：

本次虽未修改数据库结构，但若只回滚前端，不回滚配套的用户说明与培训内容，运维侧会出现“页面和文档口径不一致”的问题。
