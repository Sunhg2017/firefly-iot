# 告警管理运维说明

## 1. 模块说明

告警管理由以下模块协同提供：

- `firefly-web`
  - 告警规则维护页面
  - 告警处理页面
- `firefly-rule`
  - 告警规则与告警记录 CRUD 接口
  - 告警规则条件规范化与校验

本次发布重点是“规则维护能力增强”，包括多种触发条件和更友好的维护页面。

## 2. 发布影响

### 2.1 前端

需要发布 `firefly-web`，新增内容包括：

- 结构化规则编辑器
- 项目/产品/设备联动选择
- 产品物模型属性自动装载
- 规则列表的范围和条件摘要展示

### 2.2 后端

需要发布 `firefly-rule`，新增内容包括：

- 结构化 `conditionExpr` 校验
- `conditionExpr` 标准化存储

### 2.3 数据库

本次无数据库迁移，无新增配置项。

## 3. 发布前检查

### 3.1 编译检查

后端：

```bash
mvn -pl firefly-rule -DskipTests compile
```

前端：

```bash
cd firefly-web
npm run build
```

### 3.2 兼容性检查

发布前请确认以下两点：

1. 下游如果直接消费 `conditionExpr`，是否能够接受 JSON 字符串
2. 是否存在依赖“纯文本表达式”的告警执行器或数据同步任务

如果存在强依赖纯文本的下游，需先完成适配后再切换到结构化规则。

## 4. 发布步骤

1. 发布 `firefly-rule`
2. 验证告警规则接口可正常创建、更新、查询
3. 发布 `firefly-web`
4. 使用规则维护账号登录，验证页面可正常打开和保存
5. 使用告警处理账号登录，验证“告警处理”菜单不受影响

## 5. 发布后验收

### 5.1 页面验收

1. “告警规则维护”页中文正常显示，无乱码
2. 创建规则时可以选择项目、产品、设备
3. 选择产品后可以自动加载物模型属性
4. 支持选择阈值触发、同比/环比、连续触发、累计聚合、自定义表达式
5. 列表中能看到范围标签、触发类型和规则摘要

### 5.2 接口验收

1. 新建结构化规则成功
2. 编辑结构化规则成功
3. 编辑历史纯文本规则时自动落到“自定义表达式”
4. 条件缺项时接口返回参数错误

## 6. 监控与日志

### 6.1 关注日志

`firefly-rule` 中重点关注以下错误：

- `告警触发条件不能为空`
- `结构化告警条件缺少指标标识`
- `不支持的告警触发类型`
- 其他 `PARAM_ERROR`

### 6.2 关注现象

1. 页面保存时报参数错误
2. 选择产品后指标列表为空
3. 历史规则打开后触发类型不符合预期

## 7. 常见故障排查

### 7.1 页面打开后没有产品或设备可选

排查步骤：

1. 检查当前租户下是否已有项目、产品、设备数据
2. 检查 `/projects/list`、`/products/list`、`/devices/list` 接口是否正常
3. 检查当前账号是否具备对应读取权限

### 7.2 选择产品后没有指标属性

排查步骤：

1. 检查产品是否已经维护物模型
2. 检查 `/products/{id}/thing-model` 是否返回合法 JSON
3. 检查物模型 `properties` 下是否包含 `identifier`

### 7.3 保存结构化规则失败

排查步骤：

1. 检查触发类型对应字段是否填写完整
2. 检查阈值、窗口、连续次数是否为正数
3. 检查浏览器请求体中的 `conditionExpr` 是否为合法 JSON

### 7.4 下游执行链路不识别规则

这是本次最需要提前确认的运维风险。

原因：

- 当前 `firefly-rule` 只负责规则维护和校验
- 结构化规则最终存储在 `conditionExpr` 中
- 若下游执行器只支持纯文本表达式，需先升级执行器

处理建议：

1. 回看下游服务对 `conditionExpr` 的解析逻辑
2. 对 `mode=STRUCTURED` 的 JSON 增加解析适配
3. 未适配前，可临时使用 `CUSTOM` 纯文本表达式作为过渡

## 8. 回滚方案

### 8.1 代码回滚

若需要回滚，请同时回滚：

1. `firefly-web`
2. `firefly-rule`

### 8.2 注意事项

由于本次未变更数据库结构，代码回滚成本较低；但若生产环境已经创建了新的结构化规则，回滚后旧前端/旧下游可能无法正确理解这些 JSON 条件。

回滚前建议：

1. 先导出新建规则
2. 评估是否需要把关键规则改回纯文本 `CUSTOM` 表达式
