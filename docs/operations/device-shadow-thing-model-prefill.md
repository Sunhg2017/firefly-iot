# 设备影子期望属性物模型预填充运维说明

## 适用范围

适用于 Web 端设备影子页面：

- 设备管理列表中的影子抽屉
- 设备影子独立页面

## 发布内容

- 前端在进入 `desired` 编辑态时，会额外读取产品物模型并生成默认草稿。
- 不新增后端接口，不新增数据库表，不涉及 Flyway 变更。

## 部署方式

1. 发布包含本次前端改动的 `firefly-web` 制品。
2. 确保网关与设备服务已提供现有产品物模型查询接口 `/products/{id}/thing-model`。
3. 清理浏览器缓存或刷新静态资源版本后验证页面。

## 配置与依赖

- 无新增环境变量。
- 依赖产品物模型接口可正常返回字符串或 JSON 内容。

## 验证项

发布后建议按以下方式抽检：

1. 选择一个已维护物模型的产品，并确保存在可写属性。
2. 打开该产品下设备的影子详情。
3. 点击 `desired` 的“编辑”。
4. 确认编辑器中已出现基于物模型生成的初始 JSON。
5. 若影子已有部分 `desired`，确认原值仍保留，缺失字段被补齐。
6. 确认只读属性或系统属性没有被带入草稿。

## 常见故障

### 点击编辑后仍然是空 JSON

排查顺序：

1. 检查产品是否维护了物模型。
2. 检查物模型中的属性是否为可写属性，而不是只读或系统属性。
3. 检查浏览器开发者工具中 `/products/{id}/thing-model` 是否请求失败。

### 草稿中没有补齐结构体子字段

优先检查物模型中结构体字段是否定义在 `fields` 或 `specs.fields` 中，并确认字段包含 `identifier` 与 `dataType`。

## 日志与回滚

- 本次改动仅在前端执行，没有新增后端日志要求。
- 如需回滚，回退前端版本即可恢复为原先“仅回填当前 desired”的行为。
