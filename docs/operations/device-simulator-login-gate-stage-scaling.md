# 设备模拟器登录门禁、入口视觉优化与主舞台缩放运维说明

## 1. 适用范围

本文档用于设备模拟器增加登录门禁、主舞台改为等比例缩放后的发布、验证与排障。

## 2. 发布内容

本次发布仅涉及 `firefly-simulator` 桌面端前端：

- 未登录时进入登录门禁页
- 登录页改为桌面端双区入口布局
- 登录成功后进入主工作台
- 主工作台改为固定舞台等比例缩放
- 主工作台收口为实体面板样式，移除毛玻璃效果

本次不涉及数据库、Flyway、菜单权限和后端接口定义变更。

## 3. 依赖条件

- 当前环境网关可访问
- 登录接口可用
- 本地 Electron/Vite 构建正常

## 4. 验证步骤

执行构建：

```bash
cd firefly-simulator
npm run build:vite
```

手工回归：

1. 启动模拟器，在没有当前环境登录态时，应先进入登录门禁页。
2. 检查登录页左侧概览区与右侧登录卡是否完整展示，主次清晰，无多余说明文案。
3. 在登录门禁页切换环境，确认环境名称、网关地址和用户名预填会同步刷新，密码被清空。
4. 点击“环境管理”，确认直接打开环境管理抽屉；新增、编辑、删除、设为当前环境操作正常。
5. 使用有效账号登录，确认登录成功后才进入主工作台，且回车可直接提交登录。
6. 在主工作台全屏模式下，确认四周空边明显收窄，主框架能够更充分铺满窗口。
7. 检查主工作台壳层和各面板是否为实体底色，确认不再出现发白或毛玻璃模糊观感。
8. 选中设备后检查“设备基本信息”块，确认该区域及其滚动底板也为实体底色，不再局部发虚。
9. 在主工作台缩小、放大窗口，确认左中右三栏始终按整体比例缩放，右侧不再掉到下一行。
10. 在主工作台执行退出登录，确认页面立即回到登录门禁页。
11. 切换到一个未登录环境，确认也会回到登录门禁页。

## 5. 常见故障

### 5.1 启动后仍直接进入主页面

排查项：

- 当前激活环境是否残留旧登录态
- `workspaceStore.sessions` 是否已正确清理
- 是否误用了其他环境的已登录 session

### 5.2 登录成功后未跳入主页面

排查项：

- 登录响应中是否包含 `accessToken`
- `sessions[activeEnvironment.id]` 是否成功写入
- 当前环境是否在登录后被切换走

### 5.3 切换环境后仍显示上一个环境的用户名

排查项：

- `SimulatorAccessGate.tsx` 中针对 `activeEnvironment.id` 的表单刷新逻辑是否生效
- 当前环境是否存在持久化 session 用户信息
- 是否使用了旧版前端资源缓存

### 5.4 缩小窗口后仍出现布局挤压

排查项：

- 是否已经加载新版本前端资源
- `App.tsx` 是否仍保留旧的 breakpoint 三栏折叠逻辑
- 是否存在系统级缩放或 Electron 窗口约束影响

### 5.5 全屏模式下仍存在大面积空边或界面发虚

排查项：

- `App.tsx` 中 `STAGE_PADDING` 与 `MAX_STAGE_SCALE` 是否已更新到新版本
- `PANEL_SHELL_STYLE` 与 `appShellStyle` 是否仍残留半透明背景或 `backdropFilter`
- 是否加载了旧版打包资源

## 6. 回滚说明

如需回滚，需同时回滚以下文件：

- `firefly-simulator/src/App.tsx`
- `firefly-simulator/src/components/SimulatorAccessGate.tsx`
- `firefly-simulator/src/components/index.ts`

回滚后重新执行：

```bash
cd firefly-simulator
npm run build:vite
```

## 7. 运维提醒

- 登录门禁页依赖当前激活环境，如果用户切换环境后看到登录页，属于预期行为。
- 该主舞台缩放方案面向 PC 端，不再针对窄屏窗口做右栏折叠。
