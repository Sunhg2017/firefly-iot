# 安全审计页面图标与登录日志字段对齐运维说明

## 1. 适用范围

本文用于处理以下现象：

- 系统运维空间左侧 `安全审计` 菜单没有图标
- 安全管理页的登录日志明明登录成功，但结果列全部显示为“失败”
- 登录日志中的 IP、User-Agent 列为空

涉及模块：

- `firefly-web`
- `firefly-system`

## 2. 变更内容

- 前端补充 `SecurityScanOutlined` 图标映射
- 安全管理页登录日志表改为读取后端真实字段 `loginMethod`、`loginIp`、`result`、`userAgent`
- 后端登录日志查询接口新增 `keyword` 条件，支持用户名或登录 IP 模糊搜索
- 后端登录日志视图对象补充 `userAgent`

## 3. 部署说明

推荐同时发布：

1. 发布 `firefly-system`
2. 发布 `firefly-web`

说明：

- 仅发布前端时，可以修复菜单图标和结果列渲染问题
- 同时发布后端后，登录日志搜索和 User-Agent 展示也会一起恢复完整

## 4. 数据与 SQL

- 本次不涉及菜单新增、删除、迁移或改名
- 本次不需要新增 Flyway SQL
- 根因不是菜单台账缺数据，而是前端缺少 `SecurityScanOutlined` 映射

## 5. 验证步骤

### 5.1 菜单验证

- 进入系统运维空间
- 检查左侧 `安全审计` 分组是否显示盾牌扫描图标

### 5.2 登录日志验证

- 正常执行一次账号密码登录
- 进入 `安全管理 -> 登录日志`
- 确认最新记录显示为 `成功`
- 确认 `IP`、`User-Agent` 字段有值
- 用用户名或登录 IP 检索，确认能命中该记录

## 6. 排障

### 6.1 安全审计菜单仍无图标

优先检查：

- `firefly-web` 是否已更新到本次版本
- 浏览器是否还在使用旧静态资源缓存
- 菜单接口返回的图标名是否为 `SecurityScanOutlined`

### 6.2 登录日志仍全部显示失败

优先检查：

- 前端是否已发布到包含 `result === 'SUCCESS'` 渲染逻辑的版本
- 后端接口返回的 `result` 是否确实为 `SUCCESS/FAILED`
- 浏览器网络面板里登录日志接口响应是否包含 `loginIp`、`loginMethod`、`userAgent`

## 7. 回滚说明

- 若仅需回滚前端，可回滚 `firefly-web`
- 若需要回滚搜索和 User-Agent 字段能力，可同时回滚 `firefly-system`
- 本次未改数据库结构，无需数据库回滚
