# 逻辑删除一致性运维说明
> 版本: v1.0.0
> 日期: 2026-03-14

## 1. 适用范围

适用于带 `@TableLogic` 的模块，包括设备、设备定位器、用户、租户、API Key。

## 2. 排查方法

当出现“删除成功但列表仍存在”时，按以下顺序排查：

1. 确认服务端删除代码是否走 `deleteById(...)` / `delete(wrapper)`
2. 确认数据库 `deleted_at` 是否已更新
3. 确认列表查询是否仍过滤 `deleted_at IS NULL`
4. 确认当前运行版本是否已包含逻辑删除一致性修复

## 3. 回归建议

- 删除设备后刷新设备列表，记录应消失
- 删除用户后刷新用户列表，记录应消失
- 删除租户后刷新租户列表，记录应消失
- 删除 API Key 后刷新 API Key 列表，记录应消失

## 4. 回滚说明

若需回滚本次治理，需同步回滚对应服务删除实现和本说明文档，避免代码与运维口径不一致。
