# 前端复杂表单容器运维说明

## 1. 变更说明

本次发布新增仓库级前端交互规则：

- 复杂表单必须使用抽屉
- 简单表单才允许使用弹窗

同时已完成一轮现有页面自检，并将明显不符合规则的配置型页面统一切换为抽屉。

## 2. 发布影响

### 2.1 影响范围

影响 `firefly-web` 中以下类型页面：

- 规则配置
- 协议解析
- 任务注册与调试
- 租户和产品复杂配置
- 模板和批量导入类表单

### 2.2 不影响范围

- 后端接口协议
- 数据库结构
- 告警规则结构化 JSON 协议

## 3. 发布前检查

1. 执行前端构建：

```bash
cd firefly-web
npm run build
```

2. 重点检查已切换抽屉的页面是否存在：
   - 打开后宽度不足
   - 底部按钮遮挡
   - 表单提交按钮失效
   - 抽屉关闭后状态未重置
   - 列表页滚动和抽屉滚动冲突

## 4. 发布后验收

1. 长表单打开后应以抽屉形式出现，而不是居中弹窗。
2. 抽屉底部应保留固定的取消、保存或执行按钮。
3. 用户关闭抽屉后，再次打开时表单状态应符合预期。
4. 列表上下文、筛选条件和背景页应保持可感知，不应整页跳转。
5. 页面中文文案应正常显示，无乱码。

## 5. 常见问题排查

### 5.1 页面仍显示为弹窗

排查顺序：

1. 确认前端静态资源是否已刷新。
2. 清理浏览器缓存后重新打开页面。
3. 确认部署版本是否包含本次抽屉改造提交。

### 5.2 抽屉内容显示不完整

排查顺序：

1. 检查抽屉宽度是否被误改过小。
2. 检查 `footer`、`styles.body.paddingBottom` 是否被移除。
3. 检查是否在抽屉内继续嵌套复杂弹窗，导致空间再次拥挤。

### 5.3 表单提交后没有关闭或没有刷新

排查顺序：

1. 检查提交成功后是否正确重置 `open` 状态。
2. 检查提交后是否触发列表刷新。
3. 检查是否有表单校验阻塞导致提交没有真正执行。

## 6. 回滚说明

本次仅涉及前端交互容器和页面布局，不涉及数据库与接口协议。若需要回滚，回滚 `firefly-web` 对应版本即可；但需同步回滚配套说明文档，避免页面行为与文档口径不一致。
