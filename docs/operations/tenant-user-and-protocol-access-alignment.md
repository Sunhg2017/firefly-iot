# 租户用户管理与协议接入协同运维说明

## 适用范围

本说明适用于以下改动后的运维与排查：

- 租户业务空间用户管理入口
- 协议接入页面导航与中文文案
- SNMP / Modbus / WebSocket / TCP/UDP / LoRaWAN 页面协同入口

## 发布说明

本次变更仅涉及前端页面和前端运行时菜单逻辑，不涉及数据库结构变更，也不涉及后端接口协议变更。

发布前建议执行：

```bash
cd firefly-web
npm run build
```

## 运行时行为

## 租户用户入口补位

- 当租户工作空间启用自定义菜单树时，前端会在运行时检查是否存在 `/user`。
- 如果当前登录用户拥有 `user:read` 权限且菜单树中没有 `/user`，前端会自动插入 `组织与用户 -> 用户管理`。
- 该补位只影响页面展示和访问控制判定，不会自动修改数据库中的菜单配置记录。

## 协议接入协同卡片

- `SNMP`、`Modbus`、`WebSocket`、`TCP/UDP`、`LoRaWAN` 页面顶部都会渲染统一的协同引导卡片。
- 卡片依赖前端路由正常可用，不依赖额外后端接口。

## 常见排查

## 租户空间仍然看不到用户管理

按顺序检查：

1. 当前租户用户是否具备 `user:read` 权限。
2. 当前用户是否登录到了租户业务空间，而不是平台空间。
3. 浏览器是否使用了旧缓存，建议刷新页面或重新登录。
4. 若租户配置了菜单授权，确认前端返回的 `/tenant/menu-configs/tree` 结构没有损坏。

说明：

- 即使菜单配置树里没有 `/user`，只要权限存在，前端仍会自动补位显示该入口。
- 如果权限也缺失，则不会显示。

## 用户管理页面可以打开但创建失败

重点检查：

1. 当前请求头中是否正确携带租户上下文。
2. 当前登录账号是否具备 `user:create` 权限。
3. 后端 `UserService#createUser` 是否拿到了 `tenantId`。

后端服务逻辑要求：

- 没有租户上下文时会直接拒绝创建。
- 租户空间默认创建 `TENANT_USER` 类型用户。

## 协议接入页面仍有乱码

重点检查：

1. 前端资源是否重新构建并完成部署。
2. CDN 或反向代理是否还在缓存旧的静态资源。
3. 页面是否加载到了新生成的 `dist/assets/*` 文件。

## 协议页与设备管理仍然看起来割裂

重点检查：

1. 页面顶部是否已经渲染 `ProtocolAccessGuide` 卡片。
2. 产品列表接口 `/products/list` 是否返回 `name`、`productKey`。
3. 设备列表接口 `/devices/list` 是否返回 `deviceName`、`nickname`、`productId`。

如果上下文接口返回字段缺失，协议页仍可打开，但会退化为“未关联”展示。

## 回滚说明

本次改动无数据迁移，可直接回滚前端版本。

回滚影响：

- 租户业务空间可能再次看不到用户管理入口。
- 协议接入页会恢复为旧版文案与旧导航结构。

## 监控建议

- 关注前端静态资源发布后的 404、缓存未更新、白屏问题。
- 关注租户管理员在 `/user` 页面上的 403、401 请求。
- 关注协议接入页面调用的上下文接口：
  - `/products/list`
  - `/devices/list`
  - 各协议连接器接口
