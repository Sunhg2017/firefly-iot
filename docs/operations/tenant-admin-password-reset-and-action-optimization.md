# 租户超管密码重置与列表操作优化运维说明
> 模块: firefly-web / firefly-system / 租户管理
> 日期: 2026-03-17
> 状态: Done

## 1. 适用场景

- 平台管理员需要帮助租户超级管理员找回登录密码。
- 运维人员需要确认租户列表新的操作布局是否符合预期。

## 2. 发布说明

- 本次变更包含前后端：
  - `firefly-system` 新增租户超级管理员密码重置接口
  - `firefly-web` 新增租户列表“更多”菜单与重置密码交互
- 发布时需重新构建前端，并重新编译部署 `firefly-system`。

## 3. 验证步骤

执行：

```bash
cd firefly-web
npm run build

cd ..
mvn -pl firefly-system -am -DskipTests compile
```

回归检查：

1. 进入“租户管理”页面，确认行内只保留“用量 / 编辑 / 启用或暂停 / 更多”这类核心按钮。
2. 打开任一普通租户的“更多”菜单，确认能看到“调整套餐、调整配额、重置超管密码、空间授权、Webhook、注销租户”。
3. 点击“更多 -> 重置超管密码”，确认弹出二次确认框。
4. 确认后弹出结果窗，展示新的随机密码，并可复制。
5. 使用新密码验证该租户超级管理员账号可以重新登录。

## 4. 常见故障排查

### 4.1 看不到“重置超管密码”

- 先确认当前登录账号是否具备 `tenant:manage` 权限。
- 确认前端是否已刷新到最新构建版本。

### 4.2 重置失败

- 检查接口 `POST /api/v1/platform/tenants/{id}/admin-password/reset` 是否返回成功。
- 检查目标租户是否存在有效的 `adminUserId`。
- 检查该超级管理员账号是否已被逻辑删除。

### 4.3 改完密码仍无法登录

- 确认部署代码已包含本次修复，因为本次重置会同时清空 `loginFailCount` 并解除 `lockUntil`。
- 如仍失败，检查认证日志中是否存在其他账号状态异常，如账号被停用。

## 5. 回滚说明

如需回滚，请同时回退前端租户列表改动、后端租户密码重置接口和本次三份说明文档，然后重新构建前端并重新编译部署 `firefly-system`。
