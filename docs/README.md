# Docs Structure

`docs/` 目录用于统一存放模块交付文档，新增文档必须按分类归档。

## 分类目录

- `docs/design/`
  - 存放模块设计方案文档
- `docs/operations/`
  - 存放模块部署、配置、监控、排障、回滚等运维说明
- `docs/user-guide/`
  - 存放页面、接口、功能操作、示例、限制说明等使用文档

## 交付要求

模块完成后，至少应补齐以下三类文档：

1. 设计方案文档
2. 运维说明文档
3. 使用方法说明文档

如果现有文档仍位于 `docs/` 根目录，可后续逐步迁移；但从本规则生效后，新增或重构后的模块文档必须优先放入上述分类目录。
