# Docs Structure

`docs/` 目录用于统一存放模块交付文档，新增文档必须按分类归档。

## 分类目录

- `docs/design/`
  - 存放模块设计方案文档
- `docs/operations/`
  - 存放模块部署、配置、监控、排障、回滚等运维说明
- `docs/user-guide/`
  - 存放页面、接口、功能操作、示例、限制说明等使用文档
- `docs/progress/`
  - 存放阶段进度、交付记录、迁移记录等过程性文档

## 交付要求

模块完成后，至少应补齐以下三类文档：

1. 设计方案文档
2. 运维说明文档
3. 使用方法说明文档

目前 `docs/` 根目录仅保留目录说明文件，模块文档已按分类归档。后续新增或重构后的模块文档必须继续放入上述分类目录。

文档中的页面说明、接口示例、配置示例与操作步骤，默认也应遵循仓库级规则：尽量不直接暴露数据库主键，优先使用 `code`、`productKey`、`tenantCode` 等业务唯一键口径。
