# 设备管理与设备影子界面优化使用说明
> 适用角色: 租户管理员、运维、联调开发
> 日期: 2026-03-14

## 1. 设备影子现在怎么用

设备影子页分为四块数据：

- `Desired`
  - 平台期望设备达到的状态。
  - 可以编辑。
- `Reported`
  - 设备实际上报的状态。
  - 只读。
- `Delta`
  - 平台期望与设备上报之间的差异。
  - 只读。
- `Metadata`
  - 字段来源、更新时间等元数据。
  - 只读。

页面上已经明确标出“查看态”或“只读”，不需要再判断哪些区域能改。

## 2. 独立影子页操作步骤

1. 打开“设备影子”页面。
2. 按设备名称或别名搜索并选择设备。
3. 点击“查询影子”。
4. 如果需要修改期望状态，点击 `Desired` 卡片右上角“编辑”。
5. 修改 JSON 后点击“保存”。
6. 如需清空平台期望值，点击“清空”。

## 3. 设备管理页中的影子抽屉

1. 打开设备管理页。
2. 在目标设备上点击“影子”入口。
3. 在抽屉中查看影子概览、差异和元数据。
4. 如果需要维护期望值，只在 `Desired` 区域操作。

## 4. 使用注意事项

- `Reported` 不能手工修改，这是系统设计要求。
- 如果设备实际状态不正确，应让设备重新上报，而不是直接改 `reported`。
- 如果 `Delta` 不为空，说明设备当前状态还没有追平平台期望。
- `Desired` 必须保持合法 JSON 格式，否则系统会拒绝保存。
