# 租户用户管理与协议接入使用说明

## 适用角色

- 租户管理员
- 设备接入工程师
- 协议调试与实施人员

## 租户空间创建用户

## 入口位置

在租户业务空间左侧菜单中，进入：

`组织与用户 -> 用户管理`

## 操作步骤

1. 打开 `用户管理` 页面。
2. 点击右上角 `新建用户`。
3. 填写用户名，按需填写姓名、手机号、邮箱、初始密码。
4. 点击 `确定` 完成创建。

说明：

- 在租户业务空间中创建的用户会自动归属当前租户。
- 初始密码可以留空，系统会自动生成临时密码。

## 协议接入页面如何使用

所有协议接入页面顶部都新增了协同引导卡片，建议按以下顺序操作：

1. 先到 `产品接入` 定义产品。
2. 如为自定义报文协议，继续到 `协议解析` 定义解析规则。
3. 再回到对应的协议接入页完成调试、采集任务或会话绑定。
4. 最后到 `设备管理`、`设备数据`、`规则告警` 中查看后续效果。

## SNMP 接入

适用场景：

- 网络设备连通性测试
- OID 读取与遍历
- 轮询采集任务维护

建议做法：

- 仅调试时，可只填写目标主机和 OID。
- 如果需要让采集结果进入设备链路，建议在采集任务中补充 `关联产品` 和 `关联设备`。

## Modbus 接入

适用场景：

- PLC、仪表、传感器寄存器调试
- 保持寄存器 / 输入寄存器读取
- 采集任务注册

建议做法：

- 寄存器定义中的 `alias` 尽量与产品物模型字段保持一致。
- 采集任务绑定到设备后，后续规则、告警、数据分析更容易直接复用。

## WebSocket 接入

适用场景：

- 长连接设备在线调试
- 定向消息发送
- 广播消息测试

建议做法：

- 优先确保设备已完成产品认证，这样在线会话会自动带上设备上下文。
- 需要排查单设备问题时，直接从在线会话点击“发送”进入消息下发。

## TCP/UDP 接入

适用场景：

- 原始 Socket 协议接入
- 会话 / 端点与产品、设备的绑定
- 自定义报文解析前的上下文挂接

建议做法：

- 先绑定产品，再按需绑定设备。
- 绑定完成后，再去 `协议解析` 配置自定义协议的消费与解析链路。

## LoRaWAN 接入

适用场景：

- 查看设备入网与上下行状态
- 配置 Webhook 回调
- 发起下行消息

建议做法：

- 网络侧接入确认完成后，回到 `设备管理` 和 `设备数据` 查看统一设备视角。
- 使用下行时优先从设备列表进入，避免手工输入错误 DevEUI。

## 注意事项

- 协议接入页面里的产品和设备选择，优先展示业务标识，如产品名称、`productKey`、设备名称、昵称。
- 如果页面提示“未关联”，说明当前调试或采集任务还没有绑定到具体产品或设备。
- 如果租户空间仍未显示 `用户管理`，通常是当前账号缺少 `user:read` 权限。
