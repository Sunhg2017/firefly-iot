# 设备模拟器新建设备参数保留使用说明
> 适用角色: 联调开发、测试、实施
> 日期: 2026-03-14

## 1. 适用说明

设备模拟器的新建设备抽屉现在会在最终提交时保留前面步骤填写的接入参数，不需要在创建完成后再回到设备详情中补录 `ProductKey`、`DeviceName`、`DeviceSecret`。

## 2. 正确使用方式

1. 第一步填写模拟器名称和协议。
2. 第二步填写协议接入参数，例如 HTTP 的 `ProductKey`、`DeviceName`、`DeviceSecret`。
3. 第三步按需补充高级参数。
4. 点击“创建设备”后，系统会自动校验三步中所有必填字段，并一次性保存完整配置。

## 3. 结果确认

创建设备成功后，可以直接在访问概览中确认：

- 服务地址
- ProductKey
- DeviceName
- DeviceSecret 或动态注册后的密钥状态

如果这些字段完整显示，说明本次创建已经正确保留了前序步骤配置。

## 4. 注意事项

- 如果后续切换协议重新配置，建议重新检查第二步字段，因为不同协议的必填项不同。
- 系统会直接使用模拟器名称作为 `nickname` 提交给服务端。
- 如果看到旧设备仍出现“待补充”，通常是历史版本创建的数据，不代表当前新建流程仍有问题。
- 在第三步“配置摘要”中可以直接确认 `DeviceName / 设备名称`，便于创建前做最终核对。
- “模拟设备名称 / 平台设备昵称”只会显示一行，两者当前使用同一份名称口径。
- Video 设备不需要再手工填写 `DeviceName`：GB28181 会自动使用 `国标设备 ID`，RTSP 代理会自动使用“模拟设备名称”。
