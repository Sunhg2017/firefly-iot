# 产品物模型固有默认项使用说明
> 适用角色: 产品管理员、设备接入开发、测试人员
> 日期: 2026-03-18

## 1. 有什么变化

现在每个产品物模型都会固定带上以下默认项：

- 默认属性：`ip`
- 固有服务：`online`、`offline`、`heartbeat`

其中：

- `ip` 对应“IP地址”，表示设备当前网络地址
- 三个固有服务用于设备上线、离线和心跳生命周期语义

这些默认项属于平台固有能力，不需要手工新建。

## 2. 在哪里查看

进入“产品与物模型”页面后：

1. 打开某个产品的“物模型”抽屉
2. 切换到“属性”页签，可以看到 `ip`
3. 切换到“服务”页签，可以看到 `online`、`offline`、`heartbeat`

## 3. 可以做什么

你可以：

- 查看 `ip` 和三个固有服务的名称、标识符和说明
- 在它们之后继续新增自己的业务属性、事件和服务
- 导入自定义属性清单或维护其他自定义服务

你不可以：

- 编辑 `ip`
- 删除 `ip`
- 复制 `ip`
- 拖拽调整 `ip`
- 编辑、删除、复制、拖拽三个固有服务

## 4. 保存规则

无论你通过哪种方式维护物模型，系统都会自动确保这些默认项存在：

- 新建产品
- 可视化编辑物模型
- 高级 JSON 编辑
- 异步导入物模型

如果你在 JSON 中手工删掉 `ip` 或三个固有服务，保存后系统仍会自动补回。

## 5. 使用建议

- 如果你要上报设备网络地址，直接复用默认属性 `ip`
- 如果你要维护温度、湿度、电量等业务属性，继续新增自定义属性即可
- 不要用自定义同名属性或服务去覆盖平台默认项，系统会自动还原为平台定义

## 6. 注意事项

- 固有默认项是平台契约的一部分，请不要通过数据库脚本直接删除
- 若页面看不到 `ip` 或三个固有服务，先重新打开物模型抽屉或刷新页面
- 若刷新后仍异常，再联系运维检查后端和前端版本是否都已更新
