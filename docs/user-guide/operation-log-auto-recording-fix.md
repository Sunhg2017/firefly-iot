# 操作日志自动记录修复使用说明

## 1. 修复后效果

- 系统管理操作会自动写入操作日志
- 不再要求每个接口手工补注解后才能记录
- `安全审计 -> 操作日志` 页面可以看到真实后台操作记录

## 2. 会记录哪些操作

当前默认记录 `firefly-system` 控制器里的后台请求，例如：

- 用户管理
- 租户管理
- 角色权限
- 系统设置
- AppKey 管理
- 管理员会话管理

日志会自动展示：

- 模块
- 操作类型
- 描述
- 请求路径
- 请求方法
- IP
- 状态
- 耗时

## 3. 哪些接口不会记入这里

- `操作日志`、`审计日志`、`登录日志` 这些日志查询接口本身不会写入操作日志
- `@Hidden` 标记的内部接口不会写入操作日志

## 4. 使用方式

入口：

- `系统运维空间 -> 安全审计 -> 操作日志`

建议先执行一次真实管理动作，再刷新页面查看最新记录。

## 5. 常见说明

- 如果页面仍没有数据，通常是后端 `firefly-system` 还没有更新到修复版本
- 如果某条记录的操作类型不够准确，说明对应接口摘要语义不够明确，后续可以继续细化
