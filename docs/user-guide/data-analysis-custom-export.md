# 数据分析自定义导出使用说明

## 1. 适用角色

- 租户管理员
- 数据分析人员
- 设备运维人员

前提条件：

- 具备 `analysis:read` 和 `analysis:export` 权限。

## 2. 功能入口

- 菜单：数据分析
- 页签：数据导出

## 3. 使用方式

### 3.1 查询属性候选

1. 先选择一个或多个设备。
2. 可选填写时间范围。
3. 系统会自动加载当前筛选条件下可用的属性列表。

说明：

- 没有先选设备时，属性下拉不会开放。
- 如果不选属性，导出时默认导出当前筛选范围内的全部候选属性。

### 3.2 创建导出任务

1. 进入“数据导出”页签。
2. 选择设备。
3. 选择属性，可留空。
4. 选择时间范围，可留空。
5. 点击“创建导出任务”。

提交成功后：

- 页面会提示“导出任务已创建，请到任务中心下载结果”。
- 导出文件不会立即下载。
- 请前往任务中心查看执行状态。
- 如果任务失败但系统生成了错误清单，任务中心同样可以直接下载，不需要重新发起任务。

### 3.3 下载结果

1. 打开任务中心。
2. 找到任务类型为“导出”、业务类型为 `DATA_ANALYSIS_EXPORT` 的任务。
3. 当状态变为“已完成”后点击下载。

## 4. 输入输出说明

### 4.1 输入项

- 设备
  - 必填
  - 支持多选
- 属性
  - 选填
  - 系统自动联动可选
- 时间范围
  - 选填
  - 用于缩小数据范围

### 4.2 输出文件

导出文件格式：

- CSV

导出列：

- 时间
- 产品 Key
- 产品名称
- 设备名称
- 设备别名
- 属性
- 数值
- 字符值

说明：

- 文件中不再出现 `device_id` 这类数据库主键字段。

## 5. 示例

### 5.1 示例场景

目标：

- 导出设备 `boiler-01` 与 `boiler-02` 在一天内的温度、湿度数据。

操作：

1. 选择两个设备。
2. 属性选择 `temperature` 和 `humidity`。
3. 时间范围选择当天 00:00 到 23:59。
4. 创建导出任务。

结果：

- 任务中心生成一条导出任务。
- 完成后可下载 CSV 文件。

## 6. 注意事项

- 导出任务是异步执行的，提交后请以任务中心状态为准。
- 如果任务已取消，系统会尽快停止后台导出；极少数情况下若取消发生在上传临界点，结果文件也会被自动清理。
- 单次导出存在 50000 条保护上限，超限时任务会成功，但会提示结果被截断。
- 如果筛选范围内没有数据，系统仍可能生成仅包含表头的空 CSV。
- 删除任务时，系统会同时清理对应的本地结果文件或 MinIO 文件。

## 7. 常见问题

### 7.1 为什么我看不到属性候选

可能原因：

- 还没有选择设备。
- 当前时间范围内没有数据。
- 当前账号没有读取分析数据权限。

### 7.2 为什么导出后没有立刻下载

原因：

- 当前版本已改为异步任务中心模式，导出文件需要后台生成后再下载。

### 7.3 为什么失败任务也能下载

原因：

- 失败任务在部分场景下会生成错误清单或失败结果文件，便于定位问题。
- 只要任务中心出现下载按钮，就表示当前任务已经生成了可下载结果。

### 7.4 为什么任务显示成功，但备注里提示已截断

原因：

- 导出结果超过单次保护上限 50000 条。

建议：

- 缩小时间范围或分设备分批导出。
