# 设备影子扩展抽屉使用说明
## 1. 适用角色

- 设备运维人员
- 产品调试人员
- 设备接入实施人员

## 2. 新增了什么

现在有两个入口都支持查看设备影子扩展内容：

- “设备影子”独立页面
- “设备列表”中的影子详情抽屉

在这两个入口里，下面四张卡片都新增了展开按钮：

- `Desired / 期望值`
- `Reported / 上报值`
- `Delta / 差异`
- `Metadata / 元数据`

点击后会在右侧打开一个宽度约为页面 `75%` 的扩展抽屉，方便查看长 JSON、做字段比对和排查。
常规视图下四张卡片也会保持对齐，便于直接比较同一行卡片内容。

## 3. 如何使用

### 3.1 在设备影子页中使用

1. 进入“设备影子”页面并选择设备。
   下拉候选项会显示别名和设备名称，选中后输入框按 `display（value）` 单行格式回填；若没有别名，则只显示设备名称。
2. 查询到影子内容后，在任意卡片右上角点击展开按钮。
3. 在右侧 `75%` 宽度抽屉中查看完整内容。
4. 关闭抽屉后回到原页面继续操作。

### 3.2 在设备列表中使用

1. 进入设备列表并打开某台设备的影子详情。
2. 在影子详情中的任意卡片点击展开按钮。
3. 在右侧 `75%` 宽度抽屉中查看该卡片完整内容。
4. 关闭扩展抽屉后，仍可继续在主影子详情中操作。

## 4. Desired 卡片说明

`Desired` 同时支持卡片内编辑和扩展抽屉内编辑。
进入编辑态时，系统会按物模型默认补齐属性，系统固有属性会优先一起带出；例如内置 `ip` 即使是只读口径，也会先补进草稿。

你可以在扩展抽屉中直接执行：

- 编辑
- 保存
- 取消
- 清空

扩展抽屉中的内容和卡片中的内容始终是同一份草稿，不需要担心两边不同步。

## 5. 其他三张卡片说明

`Reported`、`Delta`、`Metadata` 即使进入扩展抽屉，也仍然是只读查看模式，不支持人工修改。

## 6. 常见场景

### 6.1 比对 Desired 与 Reported

当影子 JSON 很长时，可以先展开 `Desired` 查看目标状态，再展开 `Reported` 查看设备实际上报状态，逐段比对字段差异。

### 6.2 查看 Delta

当页面提示设备存在待同步差异时，可以展开 `Delta`，快速确认哪些字段还没有追平。

### 6.3 排查 Metadata

当需要确认字段更新时间或来源时，可以展开 `Metadata` 查看完整元数据内容。
