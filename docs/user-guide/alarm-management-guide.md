# 告警管理使用说明

## 1. 适用角色

### 1.1 告警规则维护

适用人群：

- 平台配置人员
- 规则管理员
- 负责定义告警口径的实施或运维人员

### 1.2 告警处理

适用人群：

- 值班人员
- 告警处置人员
- 项目运维负责人

## 2. 菜单入口

系统已将告警能力拆分为两个菜单：

- `规则告警 / 告警规则维护`
- `规则告警 / 告警处理`

其中：

- “告警规则维护”用于新增、修改、停用规则
- “告警处理”用于确认、处理、关闭已经触发的告警记录

## 3. 告警规则维护

### 3.1 页面说明

规则维护页由三部分组成：

1. 统计卡片：查看规则总数、启用数、停用数
2. 规则列表：查看范围、触发方式、条件摘要和状态
3. 规则弹窗：创建或编辑规则

### 3.2 新建规则步骤

1. 打开“告警规则维护”
2. 点击“新建规则”
3. 填写规则名称和说明
4. 选择适用范围
   - 项目：可选
   - 产品：建议至少选择
   - 设备：仅在要限定到单台设备时选择
5. 在“告警条件列表”中维护一条或多条条件
6. 为每条条件分别选择告警级别
7. 选择触发方式
8. 选择指标或填写指标标识
9. 补充对应参数
10. 查看“规则预览”和“规则主级别”
11. 保存

### 3.3 多条件、多级别怎么维护

一个告警规则现在可以同时维护多条条件，例如：

- 紧急：最新温度 > 85
- 告警：最近 1 小时平均温度环比上涨 20%
- 通知：连续 3 次温度 > 75

维护方式：

1. 点击“新增条件”
2. 为该条件选择告警级别
3. 选择触发方式并填写参数
4. 按需继续新增下一条条件

系统会自动把所有条件中最高的严重级别归纳为“规则主级别”，用于列表筛选和统计。

### 3.4 支持的触发方式

#### 阈值触发

适合场景：

- 温度超过 80
- 电压低于 180

需要填写：

- 指标
- 统计口径
- 比较符
- 触发阈值

示例：

- 最新值 温度 > 80

#### 同比/环比

适合场景：

- 当前小时平均温度比上一小时上涨 20%
- 本日流量比昨日同期下降 100

需要填写：

- 指标
- 统计口径
- 对标方式：同比 / 环比
- 变化口径：变化比例 / 变化值
- 变化方向：上涨 / 下降 / 双向
- 阈值
- 统计窗口

#### 连续触发

适合场景：

- 连续 3 次采样都超过阈值
- 连续 5 次采样都低于阈值

需要填写：

- 指标
- 比较符
- 触发阈值
- 连续次数

#### 累计聚合

适合场景：

- 1 小时累计能耗超过 500
- 1 天内事件计数超过 100 次

需要填写：

- 指标
- 统计口径
- 比较符
- 触发阈值
- 统计窗口
- 时间单位

#### 自定义表达式

适合场景：

- 特殊逻辑无法用标准表单表达

示例：

```text
payload.temperature > 50 && payload.humidity > 80
```

## 4. 范围与指标选择建议

1. 规则最好至少限定到产品，避免“全局规则”难以管理。
2. 如果是单台重点设备，再追加选择设备。
3. 优先使用产品物模型属性作为指标，不建议随意手写不存在的字段。
4. 若选择产品后没有属性可选，说明该产品物模型还未维护或没有属性定义。

## 5. 规则列表怎么看

列表重点关注四块信息：

- 适用范围：项目、产品、设备标签
- 条件级别：当前规则覆盖了哪些告警级别
- 触发方式：当前规则包含了哪些触发类型
- 触发条件：自动生成的多条件摘要说明

这样可以快速判断规则内容，而不需要阅读原始表达式或 JSON。

## 6. 编辑规则

系统统一使用结构化告警规则：

- 打开规则时，会把 `conditionExpr` 反序列化到对应表单
- `CUSTOM` 类型的表达式内容保存在结构化 JSON 的 `customExpr` 字段中
- 多条条件会一起回填到“告警条件列表”

## 7. 告警处理

“告警处理”页面面向值班和处置人员，主要操作：

1. 查看告警记录
2. 确认告警
3. 填写处理备注
4. 关闭告警

建议处理备注中包含：

- 工单号
- 现场处置动作
- 结论或责任归属

## 8. 注意事项

1. 当前版本已经支持多条件、多级别规则配置和保存校验。
2. 如果下游告警执行链路还没有适配结构化 `conditionExpr`，请先与研发确认再上线。
3. 对于无法标准化表达的规则，可以继续使用“自定义表达式”，但保存后仍然会以结构化 JSON 存储。
4. 发现页面指标为空时，先检查产品物模型是否维护完整。
5. 规则主级别是系统自动归纳的最高严重级别，不需要手工维护。
