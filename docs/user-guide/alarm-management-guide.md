# 告警管理使用说明

## 1. 适用角色

### 1.1 告警规则

- 租户管理员
- 规则管理员
- 项目实施与运维人员

### 1.2 告警处理

- 值班人员
- 告警处置人员
- 项目负责人

## 2. 菜单入口

租户业务空间：

1. `规则告警 / 告警规则`
2. `规则告警 / 告警处理`
3. `规则告警 / 告警接收组`

## 3. 告警现在怎么触发

当前告警运行时已经直接接入设备属性上报链路：

1. 设备上报属性
2. 系统按规则范围和条件判断是否命中
3. 命中后生成告警记录
4. 如规则配置了通知方式和接收对象，会同步发送通知

注意：

1. 当前只对属性上报触发告警，不对事件上报直接触发告警。
2. 阈值规则固定按“最新一次上报值”判断。

## 4. 怎么新建告警规则

1. 进入 `规则告警 / 告警规则`
2. 点击“新建规则”
3. 在右侧抽屉中填写规则名称和说明
4. 选择项目、产品、设备范围
5. 新增并维护告警等级块
6. 为每个等级块配置触发语义和触发条件
7. 如需要，配置通知方式、接收组和指定接收人
8. 查看规则预览与通知摘要后保存

## 5. 等级块怎么理解

一个等级块代表一档告警策略，例如：

1. 紧急：全部满足时触发
2. 告警：任意满足时触发
3. 通知：至少 2 条满足时触发

建议按业务语义拆块，不要把完全不同的异常都堆在同一个等级块里。

## 6. 条件怎么选

### 6.1 阈值触发

适合直接比较当前属性值，例如：

- 温度 >= 80
- 电流 < 5

说明：

1. 阈值触发固定按最新值判断
2. 页面不再支持为阈值规则选择平均值、最大值、累计值等统计口径

### 6.2 同环比

适合比较当前窗口与上一窗口或同期窗口的变化幅度。

### 6.3 连续触发

适合过滤抖动，要求最近 N 次样本连续满足阈值。

### 6.4 累计聚合

适合窗口内求和、计数、最大值、最小值、平均值等场景。

### 6.5 自定义表达式

适合标准条件无法表达的场景，但建议只在确有需要时使用。

## 7. 通知怎么配置

### 7.1 通知方式

规则里只选择“方式”，不选择具体通道实例。

可选方式包括：

- 邮件
- 短信
- 电话
- 企业微信
- 钉钉
- Webhook
- 站内信

### 7.2 接收对象

可以同时配置：

1. 告警接收组
2. 指定接收人

系统会自动合并并去重。

### 7.3 模板说明

通知发送时：

1. 优先使用当前租户自己的模板
2. 若租户未维护模板，则自动使用平台默认告警模板

如果没有配置通知方式或接收对象，规则命中后只会保留告警记录，不会主动通知。

## 8. 告警记录会怎么变化

### 8.1 首次命中

创建一条新告警，状态为“待确认”。

### 8.2 条件恢复

系统会自动关闭当前开放态告警。

### 8.3 更高等级再次命中

系统会自动关闭旧记录，并新建更高等级的告警记录。

### 8.4 同等级持续命中

系统不会重复开单，也不会重复发送同等级通知。

## 9. 告警处理页面怎么用

值班人员可以在“告警处理”页面执行：

1. 查看告警记录
2. 确认告警
3. 填写处理备注
4. 手工关闭告警

## 10. 常见问题

### 10.1 为什么规则保存了却没告警

优先检查：

1. 设备是否真的上报了对应属性
2. 指标名是否和规则里的一致
3. 属性值是否为数值
4. 规则范围是否配错

### 10.2 为什么有告警记录但没收到通知

优先检查：

1. 是否配置了通知方式
2. 是否选择了接收组或指定接收人
3. 接收人是否有邮箱、手机号或站内信目标
4. 当前租户或平台是否有可用模板

### 10.3 为什么阈值规则看不到统计口径选择

因为当前阈值规则统一按最新值判断，这是系统当前唯一支持的阈值语义。
