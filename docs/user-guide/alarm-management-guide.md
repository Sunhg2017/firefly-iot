# 告警管理使用说明

## 1. 适用角色

### 1.1 告警规则维护

适用人群：

- 平台配置人员
- 规则管理员
- 具备告警规则维护权限的租户管理员

### 1.2 告警处理

适用人群：

- 值班人员
- 运维处理人员
- 需要确认和关闭告警的项目负责人

## 2. 菜单入口

系统已将告警能力拆分为两个独立菜单：

- `规则告警 / 告警规则维护`
- `规则告警 / 告警处理`

如果你之前使用的是旧地址 `/alarm`，系统会自动跳转到告警处理页。

## 3. 告警规则维护

### 3.1 页面用途

用于新增、编辑、启停和删除告警规则。

### 3.2 页面内容

- 规则统计卡片
- 规则名称搜索
- 告警级别筛选
- 规则列表
- 新建/编辑弹窗

### 3.3 操作步骤

1. 打开“告警规则维护”菜单。
2. 点击“新建规则”。
3. 填写规则名称、规则说明、告警级别、触发条件。
4. 保存后在列表中检查规则状态。
5. 如需调整规则，点击“编辑”。

### 3.4 输入示例

触发条件示例：

```text
payload.temperature > 50
```

## 4. 告警处理

### 4.1 页面用途

用于查看已触发的告警记录，并进行确认、处理和关闭。

### 4.2 页面内容

- 待确认、已确认、已处理、已关闭统计卡片
- 告警标题搜索
- 告警级别筛选
- 处理状态筛选
- 告警记录列表

### 4.3 操作步骤

#### 确认告警

1. 打开“告警处理”菜单。
2. 找到状态为“待确认”的告警。
3. 点击“确认”。

#### 处理告警

1. 在待确认或已确认的告警行点击“处理”。
2. 填写处理备注，例如故障原因、工单号或值班记录。
3. 提交后，告警状态更新为“已处理”。

#### 关闭告警

1. 在未关闭的告警行点击“关闭”。
2. 系统将该告警标记为“已关闭”。

## 5. 权限说明

### 5.1 规则维护相关

- 查看规则：`alarm:read`
- 新建规则：`alarm:create`
- 编辑规则：`alarm:update`
- 删除规则：`alarm:delete`

### 5.2 告警处理相关

- 查看告警记录：`alarm:read` 或 `alarm:confirm` 或 `alarm:process`
- 确认告警：`alarm:confirm`
- 处理/关闭告警：`alarm:process`

## 6. 注意事项

1. “告警规则维护”和“告警处理”服务于不同岗位，菜单分开后请按职责授予权限。
2. 没有对应权限时，页面按钮不会显示，即使手工访问接口也会被后端拦截。
3. 处理备注建议填写工单号、处置动作和处理结论，方便后续追溯。

## 7. 常见问题

### 7.1 为什么我只能看到告警处理，看不到告警规则维护？

因为你的角色可能只有告警处理权限，没有 `alarm:read` 对应的规则查看权限。

### 7.2 为什么我能看见告警处理，但不能编辑规则？

告警处理和规则维护已经拆分，处理角色默认不具备规则维护权限。

### 7.3 为什么旧收藏地址还能打开？

系统保留了旧地址兼容跳转，会自动跳到新的告警处理页面。
