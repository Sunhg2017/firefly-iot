# 告警管理使用说明

## 1. 适用角色

### 1.1 告警规则维护

适用人群：

- 平台配置人员
- 规则管理员
- 负责定义告警口径的实施或运维人员

### 1.2 告警处理

适用人群：

- 值班人员
- 告警处置人员
- 项目运维负责人

## 2. 菜单入口

系统已拆分两个菜单：

- `规则告警 / 告警规则维护`
- `规则告警 / 告警处理`

## 3. 告警规则配置页面怎么用

### 3.1 页面现在的思路

新的配置页面不再是一长串字段直接堆在一起，而是按步骤引导：

1. 先看顶部“配置步骤”
2. 再填基本信息
3. 再选适用范围
4. 最后配置多个级别块

页面顶部还会实时显示：

- 已配置级别块数
- 已配置条件数
- 当前规则主级别

### 3.2 级别块怎么理解

一个“级别块”代表某一档告警策略，例如：

- 紧急：全部条件满足才触发
- 告警：任意条件满足就触发
- 通知：至少 2 条条件满足才触发

也就是说，你不是先给每条条件定级别，而是先按告警档位拆块，再在块里维护条件。

### 3.3 新建规则步骤

1. 点击“新建规则”
2. 填写规则名称和说明
3. 选择项目、产品、设备范围
4. 点击“新增级别块”
5. 在级别块中依次完成：
   - 选择告警级别
   - 选择触发语义
   - 如需要，填写满足条数
   - 新增并维护条件
6. 查看级别块预览和规则预览
7. 保存

## 4. 触发语义怎么选

### 4.1 全部满足

适合：

- 多个条件同时成立才算真正异常

示例：

- 温度过高并且电流过高时才触发紧急告警

### 4.2 任意满足

适合：

- 任何一个关键条件异常都应该立即告警

示例：

- 温度超阈值或湿度超阈值，任一满足就告警

### 4.3 至少 N 条满足

适合：

- 希望降低误报，要求多个异常同时出现

示例：

- 5 个健康指标里至少 3 个异常才触发

## 5. 条件怎么配

页面中每条条件现在都有三层辅助信息：

1. 条件类型选择
2. 条件类型说明
3. 条件预览

你可以按下面思路选择：

- 阈值触发：适合“温度 > 80”
- 同比/环比：适合趋势变化
- 连续触发：适合过滤抖动
- 累计聚合：适合时间窗口求和或计数
- 自定义表达式：适合标准条件无法表达的场景

## 6. 为什么建议先选产品

如果先选产品，页面可以自动加载物模型属性，后续选指标时会更直观。

如果不选产品：

- 页面仍然允许手工补指标标识
- 但更容易输错
- 也更难确认指标是否真实存在

因此建议优先：

1. 先选产品
2. 再选指标
3. 最后填条件参数

## 7. 页面里几个预览怎么看

### 7.1 条件预览

用于确认单条条件写得是否符合预期。

### 7.2 级别块预览

用于确认这一档告警的触发语义和条件组合是否正确。

### 7.3 规则预览

用于确认整条规则最终表达的告警策略。

## 8. 常见使用建议

1. 先按告警档位拆块，不要把所有条件塞进同一层。
2. 尽量让同一个级别块里的条件服务同一业务语义。
3. 如果条件很多，优先看顶部统计和底部预览，确认复杂度是否超出预期。
4. 对于单台设备规则，再选择设备；否则优先停留在产品级。

## 9. 告警处理页面

“告警处理”页面面向值班和处置人员，主要操作包括：

1. 查看告警记录
2. 确认告警
3. 填写处理备注
4. 关闭告警

## 10. 注意事项

1. 当前规则统一保存为结构化 JSON。
2. 一个规则支持多个级别块。
3. `AT_LEAST` 模式下，满足条数不能大于该级别块的条件数。
4. 规则主级别由系统自动推导，不需要单独填写。
