# 告警管理使用说明

## 1. 适用角色

### 1.1 告警规则维护

适用人群：

- 平台配置人员
- 规则管理员
- 负责定义告警口径的实施或运维人员

### 1.2 告警处理

适用人群：

- 值班人员
- 告警处置人员
- 项目运维负责人

## 2. 菜单入口

系统已将告警能力拆分为两个菜单：

- `规则告警 / 告警规则维护`
- `规则告警 / 告警处理`

其中：

- “告警规则维护”用于新增、修改、启停规则
- “告警处理”用于确认、处理、关闭已触发的告警记录

## 3. 告警规则维护

### 3.1 页面结构

规则维护页主要由三部分组成：

1. 统计卡片
2. 规则列表
3. 创建/编辑弹窗

弹窗内部再分为：

1. 基本信息
2. 适用范围
3. 规则主级别提示
4. 告警级别块列表
5. 规则预览

### 3.2 这次规则怎么配

新的配置方式不是“先写很多条件，再给每条条件选级别”，而是：

1. 先新增一个告警级别块
2. 为这个级别块选择告警级别
3. 为这个级别块选择触发语义
4. 再在这个级别块下维护多条条件

也就是说，一个规则里可以有多个级别块，例如：

- 紧急级别块：全部条件满足才触发
- 告警级别块：任意条件满足就触发
- 通知级别块：至少 2 条条件满足才触发

### 3.3 新建规则步骤

1. 打开“告警规则维护”
2. 点击“新建规则”
3. 填写规则名称和说明
4. 选择适用范围
   - 项目：可选
   - 产品：建议至少选择
   - 设备：仅在要限定到单台设备时选择
5. 点击“新增级别块”
6. 在级别块中选择：
   - 告警级别
   - 触发语义
7. 如果选择“至少 N 条满足”，再填写满足条数
8. 在该级别块下点击“新增条件”
9. 为每条条件选择触发方式并填写参数
10. 按需继续新增其他级别块
11. 查看“规则预览”和“规则主级别”
12. 保存

### 3.4 触发语义怎么理解

#### 全部满足

含义：

级别块下所有条件都满足时，才触发该级别告警。

适合场景：

- 温度高并且电流高时才告警
- 压力超阈值并且设备在线时才升级为紧急

#### 任意满足

含义：

级别块下任意一条条件满足时，就触发该级别告警。

适合场景：

- 温度超阈值或湿度超阈值，任一满足就告警

#### 至少 N 条满足

含义：

级别块下命中条件数达到设定条数时触发。

适合场景：

- 5 条健康指标中至少 3 条异常才触发告警

### 3.5 支持的触发方式

#### 阈值触发

适合场景：

- 温度超过 80
- 电压低于 180

需要填写：

- 指标
- 统计口径
- 比较符
- 触发阈值

#### 同比/环比

适合场景：

- 当前小时平均温度比上一小时上涨 20%
- 当日流量比昨日同期下降 100

需要填写：

- 指标
- 统计口径
- 对标方式
- 变化口径
- 变化方向
- 阈值
- 统计窗口

#### 连续触发

适合场景：

- 连续 3 次采样都超过阈值
- 连续 5 次采样都低于阈值

需要填写：

- 指标
- 比较符
- 触发阈值
- 连续次数

#### 累计聚合

适合场景：

- 1 小时累计能耗超过 500
- 1 天内事件计数超过 100 次

需要填写：

- 指标
- 统计口径
- 比较符
- 触发阈值
- 统计窗口
- 时间单位

#### 自定义表达式

适合场景：

- 特殊逻辑无法通过标准条件组合表达

示例：

```text
payload.temperature > 50 && payload.humidity > 80
```

## 4. 推荐配置方式

### 4.1 多级别示例

例如温度监控规则可这样维护：

1. 紧急级别块
   - 触发语义：全部满足
   - 条件 1：最新温度 > 85
   - 条件 2：连续 3 次温度 > 82
2. 告警级别块
   - 触发语义：任意满足
   - 条件 1：最新温度 > 80
   - 条件 2：1 小时平均温度环比上涨 20%

这样更符合现场值班和升级策略。

### 4.2 范围选择建议

1. 建议至少限定到产品，避免出现难以管理的全局规则。
2. 如果是单台重点设备，再选择设备。
3. 优先使用物模型属性作为指标，减少手工输入。
4. 若产品没有属性可选，先检查物模型是否已维护完整。

## 5. 规则列表怎么看

列表重点看四块内容：

1. 适用范围：项目、产品、设备标签
2. 级别：当前规则覆盖了哪些告警级别
3. 触发方式：当前规则包含哪些条件类型
4. 触发条件：系统自动生成的规则摘要

其中“规则主级别”是系统自动归纳出的最高严重级别，用于筛选和统计。

## 6. 编辑规则

编辑已有规则时，系统会自动：

1. 读取 `conditionExpr`
2. 反序列化为多个级别块
3. 回填每个级别块的级别、触发语义和条件

如果页面打开后没有正确回填，通常说明该规则不是当前支持的结构化协议。

## 7. 告警处理

“告警处理”页面面向值班和处置人员，主要操作有：

1. 查看告警记录
2. 确认告警
3. 填写处理备注
4. 关闭告警

建议处理备注中包含：

- 工单号
- 现场处置动作
- 处理结论

## 8. 注意事项

1. 当前版本不兼容旧文本规则，所有规则都会按结构化 JSON 保存。
2. 一个规则可以维护多个级别块，不需要把所有条件硬塞成同一层。
3. `AT_LEAST` 模式下，满足条数不能大于该级别块条件数。
4. 规则主级别由系统自动归纳，不需要单独填写。
5. 如果下游执行链路还没有适配新的结构化协议，请先与研发确认后再投产。
