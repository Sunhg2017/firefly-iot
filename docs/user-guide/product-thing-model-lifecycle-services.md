# 产品物模型固有生命周期服务使用说明

> 适用角色: 产品管理员、设备接入开发、测试人员
> 日期: 2026-03-14

## 1. 有什么变化

现在每个产品物模型都会固定带上三个生命周期服务：

- 上线
- 离线
- 心跳

对应的标识符分别是：

- `online`
- `offline`
- `heartbeat`

这三个服务属于系统固有能力，不需要手工新建。

## 2. 在哪里查看

进入“产品”页面后：

1. 打开某个产品的“物模型”抽屉。
2. 切换到“服务”页签。
3. 可以看到三个带有“固有服务”标记的服务项。

## 3. 可以做什么

你可以：

- 查看这三个固有服务的名称和说明。
- 在它们之后继续新增你自己的业务服务。
- 导入属性清单或维护其他自定义服务。

你不可以：

- 编辑固有服务
- 删除固有服务
- 复制固有服务
- 拖拽调整固有服务

## 4. 保存规则

无论你通过哪种方式维护物模型，系统都会自动确保这三个服务存在：

- 新建产品
- 可视化编辑物模型
- 高级 JSON 编辑
- 异步导入物模型

如果你在 JSON 中手工删掉它们，保存后系统仍会自动补回。

## 5. 设备服务调用如何使用

设备消息页现在支持按产品物模型选择服务：

1. 进入“设备消息”页面。
2. 选择目标设备。
3. 系统会自动加载该设备所属产品的服务列表。
4. 在“服务名称”下拉中选择需要调用的服务。
5. 按物模型定义填写参数 JSON 并发送。

如果产品物模型暂时加载失败，页面至少会回退显示：

- 上线
- 离线
- 心跳

## 6. 注意事项

- 固有服务是平台契约的一部分，请不要尝试通过数据库脚本直接删除。
- 若你的产品还需要重启、同步时间、抓图等业务能力，请继续新增自定义服务，不要复用这三个固有服务。
- 若页面看不到这三个服务，先尝试重新打开物模型抽屉或刷新页面；若仍异常，再联系运维检查后端版本。
