# 第三方登录与绑定使用说明

## 1. 适用对象

- 系统运维：配置第三方登录能力
- 已有平台用户：绑定第三方账号并使用第三方登录
- 客户端接入方：调用微信小程序、支付宝、Apple 登录接口

## 2. 开始前准备

使用第三方登录前，需要先满足两个条件：

1. 系统运维已经在系统设置里启用了对应提供商并配置好凭据。
2. 你的本地平台账号已经和第三方账号完成绑定，或第三方返回的邮箱/手机号能唯一匹配到现有本地账号。

登录页展示哪些第三方按钮，取决于系统级配置，不区分当前登录租户。

## 3. Web 控制台使用方式

### 3.1 微信或钉钉直接登录

1. 打开登录页。
2. 在“其他登录方式”里点击微信或钉钉。
3. 按第三方页面提示完成扫码或授权。
4. 授权成功后，系统会自动回到控制台并完成登录。

### 3.2 先绑定再登录

如果第一次使用第三方登录，建议先完成绑定：

1. 先用账号密码或短信验证码登录控制台。
2. 进入“安全管理”。
3. 打开“第三方绑定”标签页。
4. 点击微信或钉钉对应的“去绑定”。
5. 完成授权后返回控制台。
6. 之后退出登录，就可以直接使用对应第三方方式登录。

### 3.3 解绑

1. 进入“安全管理”。
2. 打开“第三方绑定”标签页。
3. 在已绑定的提供商行点击“解绑”。

解绑后，该第三方账号将不能再直接登录当前平台账号。

## 4. 客户端接口使用方式

以下提供商当前走“客户端先授权，再调用后端登录接口”的方式：

- 微信小程序：`POST /api/v1/auth/wechat-mini`
- 支付宝：`POST /api/v1/auth/alipay`
- Apple：`POST /api/v1/auth/apple`

使用要点：

1. 客户端先从各自 SDK 或开放平台拿到 `code`、`authCode` 或 `identityToken`。
2. 调用对应后端接口。
3. 若该第三方账号已绑定，接口会直接返回平台 Token。
4. 若未绑定但能唯一匹配本地邮箱或手机号，系统会自动补绑并返回 Token。
5. 若无法匹配，接口会返回失败，需要先使用现有方式登录并完成绑定。

## 5. 常见提示说明

### 5.1 第三方账号未绑定

含义：

- 该第三方账号还没有和平台账号建立绑定
- 或第三方返回的信息不能唯一匹配本地用户

处理方式：

1. 先用账号密码或短信登录。
2. 在“安全管理 -> 第三方绑定”完成绑定。

### 5.2 当前控制台暂不支持该提供商网页回调登录

含义：

- 当前 Web 控制台只直接承接微信、钉钉的网页登录回调。

处理方式：

- 支付宝、Apple、微信小程序请改走客户端授权后调用接口。

## 6. 注意事项

- 当前版本不会自动创建新的租户用户。
- 同一个第三方账号不能同时绑定到多个平台用户。
- 若你更换了第三方开放平台应用，系统会优先尝试按 `UnionID` 或唯一邮箱/手机号关联已有账号。
