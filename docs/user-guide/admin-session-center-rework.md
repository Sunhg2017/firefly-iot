# 管理员会话页使用说明

## 1. 适用角色

- 系统运维管理员
- 具备 `user:read`、`user:update` 权限的运维人员

## 2. 页面变化

修复后，`管理员会话` 页签不再要求输入 `userId`。

页面会默认展示当前在线的管理员会话，包括：

- 系统运维账号
- 各租户超级管理员

## 3. 入口

- `系统运维空间 -> 安全审计 -> 安全管理 -> 管理员会话`

## 4. 使用方式

页面支持以下操作：

- 搜索管理员账号
- 搜索管理员姓名
- 搜索所属租户
- 搜索登录 IP
- 按平台筛选
- 按身份筛选：
  - `系统运维`
  - `租户管理员`
- 下线单个会话
- 下线某个管理员的全部会话

## 5. 字段说明

- `管理员`：优先显示姓名，同时附带账号
- `所属租户`：管理员归属租户名称
- `身份`：系统运维或租户管理员
- `平台`：WEB、APP、小程序等登录平台
- `登录方式`：密码、短信等方式
- `IP`：该会话登录 IP
- `User-Agent`：客户端标识
- `最后活跃时间`：最近一次活跃时间
- `登录时间`：会话创建时间

## 6. 注意事项

- `全部下线` 是按管理员账号执行，会让该管理员所有在线会话一起失效
- 页面不再展示和要求输入数据库主键，运维人员只需要按账号、姓名、租户等业务信息操作
- 如果列表为空，先确认目标管理员当前是否真的在线
