# 安全审计页面修复使用说明

## 1. 适用角色

- 系统运维管理员
- 平台管理员

## 2. 修复后效果

### 2.1 左侧菜单

- `安全审计` 分组会恢复图标显示

### 2.2 登录日志

- 登录成功记录会显示为 `成功`
- 登录失败记录会显示为 `失败`
- 列表会展示登录方式、平台、IP、失败原因、User-Agent、时间

## 3. 登录日志使用方式

入口：

- `系统运维空间 -> 安全审计 -> 安全管理 -> 登录日志`

支持操作：

- 按用户名搜索
- 按登录 IP 搜索
- 按登录结果筛选成功或失败
- 刷新查看最新登录记录

结果说明：

- `成功`：后端登录日志 `result = SUCCESS`
- `失败`：后端登录日志 `result = FAILED`

## 4. 常见说明

- 如果看到成功登录却显示失败，通常是前端版本过旧
- 如果 `User-Agent` 仍为空，通常是后端版本未同步更新
- 本次修复不涉及菜单重新授权，也不需要重新维护菜单台账
