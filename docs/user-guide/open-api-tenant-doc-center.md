# 租户 OpenAPI 文档中心使用说明

## 1. 适用角色

- 租户管理员
- 具备 `AppKey 管理` 查看权限的租户用户

## 2. 打开方式

在租户空间页面右上角点击“接口文档”按钮，系统会跳转到：

```text
/app-key?tab=docs
```

也可以先进入“AppKey 管理”页面，再切换到“接口文档”标签页。

## 3. 页面内容

文档页分为三部分：

### 3.1 调用说明

这里会展示：

- 固定鉴权请求头
- 签名原文模板
- 网关地址模板

调用 OpenAPI 时，请使用 AppKey 创建时拿到的：

- `Access Key`
- `Secret Key`

其中：

- `Access Key` 放在请求头 `X-App-Key`
- `Secret Key` 不直接传输，只用于本地计算 `X-Signature`

### 3.2 筛选区

支持两种筛选方式：

- 关键字搜索：按接口编码、名称、地址、说明搜索
- 服务筛选：只查看某一个服务下的接口

### 3.3 服务分组列表

接口会按服务分组展示，例如：

- 系统服务
- 设备服务
- 数据服务

每一行都可以展开查看完整文档。

## 4. 查看接口详情

展开某个接口后，可以看到：

- 接口编码
- HTTP 方法
- 网关调用地址
- 成功响应状态码
- 请求内容类型
- 响应内容类型
- 路径参数 / 查询参数 / 业务请求头
- 请求体字段
- 响应字段
- 请求示例
- 响应示例
- curl 调用示例

## 5. 字段说明怎么看

字段表中的几列含义如下：

- `字段`：字段路径，嵌套对象会以 `a.b.c` 形式展示，数组会以 `items[].name` 展示
- `位置`：字段属于路径参数、查询参数、请求体还是响应体
- `类型`：字段类型
- `必填`：是否必须传
- `说明`：字段用途说明
- `示例`：字段样例值

## 6. curl 示例怎么用

页面里的 curl 示例使用了占位符，请在实际调用时替换：

- `{{gatewayBaseUrl}}`
- `{{accessKey}}`
- `{{timestamp}}`
- `{{nonce}}`
- `{{signature}}`

如果接口存在路径参数或查询参数，还需要替换：

- `{{id}}`
- `{{deviceName}}`
- 其他业务占位符

## 7. 常见问题

### 7.1 为什么看不到某个接口

可能原因：

- 当前租户没有订阅该 OpenAPI
- 平台已停用该 OpenAPI
- 当前账号没有 `AppKey 管理` 查看权限

### 7.2 为什么有地址但没有完整字段说明

说明系统已经识别到该接口目录，但目标微服务的 OpenAPI 明细暂时没有成功解析。

遇到这种情况，请联系运维或研发检查：

- 对应服务是否在线
- `/v3/api-docs` 是否可访问
- 接口 Swagger 注解是否完整

### 7.3 页面里的 Secret Key 在哪看

Secret Key 只会在创建 AppKey 成功时展示一次。若已经关闭弹窗，需要重新创建新的 AppKey。
