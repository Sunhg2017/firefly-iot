# 设备影子上报日志刷屏处理说明
> 适用角色: 运维、联调开发、测试
> 日期: 2026-03-14

## 1. 问题现象

如果日志中持续出现：

`Device shadow reported updated: deviceId=..., version=...`

这通常不是业务错误，而是设备正常高频上报触发的成功日志过多。

## 2. 当前处理

- 系统已将该类成功日志降为 `DEBUG`。
- 在默认 `INFO` 日志级别下，这类日志不会再刷屏。
- 如需排查设备影子写入细节，可以临时开启 `DEBUG` 查看。

## 3. 如何确认生效

1. 重启 `firefly-device` 服务。
2. 保持设备持续上报。
3. 观察服务日志，确认不再反复出现该条 `INFO` 日志。
