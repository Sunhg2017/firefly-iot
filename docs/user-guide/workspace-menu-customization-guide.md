# 工作空间菜单配置使用说明

## 1. 适用角色

- 系统运维空间：系统超级管理员
- 租户空间：租户超级管理员

## 2. 功能入口

页面路径：

- 系统运维空间：`/menu-customization`
- 租户空间：`/menu-customization`

菜单名称默认均为“菜单配置”。

## 3. 能做什么

你可以对当前空间内已开放的菜单做三类调整：

- 修改菜单显示名称
- 调整菜单挂载到哪个目录下
- 调整同级菜单排序

不会改变：

- 菜单对应页面功能
- 菜单对应权限点
- 当前租户是否拥有该菜单

## 4. 操作步骤

1. 打开“菜单配置”页面。
2. 在左侧树中选择要调整的菜单或目录。
3. 点击右上角“编辑菜单”。
4. 在抽屉中修改：
   - 菜单名称
   - 父级目录
   - 排序
5. 点击“保存”。
6. 保存成功后，左侧导航和顶部面包屑会立即按新配置刷新。

## 5. 恢复默认

如果想取消当前菜单的个性化配置：

1. 在右侧详情区域选择“恢复默认”。
2. 确认后，菜单会恢复成系统基础菜单的默认名称、层级和排序。

## 6. 使用示例

### 示例一：租户按业务术语改名

- 原菜单：`产品与物模型`
- 修改后：`设备产品`

效果：

- 左侧导航显示“设备产品”
- 面包屑同步显示“设备产品”

### 示例二：把页面挂到更符合使用习惯的目录

- 原层级：`组织与权限 / 菜单配置`
- 调整后：仍在当前空间内，但可移动到其他可选目录下

前提：

- 目标父级必须是目录节点
- 不能挂到页面节点下面

## 7. 注意事项

- 租户空间只能调整本租户已授权的菜单，不能配置未授权菜单。
- 系统运维空间只影响系统运维租户自己的运维菜单显示。
- 菜单只是显示方式变化，不代表新增了权限。
- 如果某个菜单在后续版本中被基础菜单重构替换，原来的个性化配置可能需要重新设置。
