# 设备模拟器环境与登录使用说明

> 适用角色：联调开发、测试、实施人员
> 更新时间：2026-03-22

## 1. 怎么切换环境

设备列表左上区域新增了环境选择框。

常用操作：

1. 直接切换下拉框，切到目标环境。
2. 点击 `环境` 打开环境管理。
3. 在环境管理里新增、编辑、删除环境。

每个环境会保存：

- 平台网关地址
- 协议服务地址
- 媒体服务地址
- MQTT Broker 地址

切换环境后，新建设备抽屉会自动带出当前环境的地址默认值。

## 2. 怎么登录当前环境

1. 在左上区域确认当前环境。
2. 点击 `登录`。
3. 输入用户名和密码。
4. 需要下次自动保留当前环境登录态时，勾选 `记住我`。
5. 登录成功后，顶部会显示当前租户和当前用户。

点击 `退出` 只会退出当前环境，不影响其他环境已保存的登录态。

`记住我` 的规则：

- 勾选后：当前环境登录态和用户名会被保留，重新打开模拟器后可直接进入主页面。
- 不勾选：当前环境登录态只在本次运行有效，关闭模拟器后需要重新登录。
- 退出当前环境后：会清除当前 token，但仍保留已记住的用户名，方便再次登录。

## 3. 新建设备时如何直接选产品

登录成功后，新建设备时：

- HTTP
- MQTT
- CoAP

这三类协议会优先显示“所属产品”下拉，而不是手工输入 `ProductKey`。

选择产品后会自动回填：

- `ProductKey`
- HTTP 或 MQTT 认证方式
- `ProductSecret`（一型一密产品）

如果当前环境未登录、登录已失效、产品接口异常，页面会自动回退为手工填写 `ProductKey`。

## 4. 一型一密产品怎么创建模拟设备

### 4.1 HTTP / MQTT

选择一型一密产品后：

- 认证方式会自动切换到 `一型一密`
- `ProductSecret` 会自动带出
- 你只需要继续填写 `DeviceName` 等必要字段

### 4.2 CoAP

CoAP 目前仍沿用 `deviceSecret` 录入方式。
产品选择主要用于快速定位当前租户产品，避免再手工查 `ProductKey`。

## 5. 环境默认值会影响哪些地方

当前环境会影响：

- 新建设备时的 HTTP / CoAP / MQTT / SNMP / Modbus / WebSocket / LoRaWAN 地址默认值
- OpenAPI 网关默认值
- 导入设备时缺失地址字段的默认回填

已经创建好的设备不会因为切换环境而被自动改写。

## 6. 常见提示怎么理解

- `未登录时请手工填写 ProductKey`
  说明当前环境还没有登录。

- `登录已失效，请重新登录`
  说明当前环境保存的 token 已不可用，需要重新登录。

- `当前协议下未查询到产品，可手工填写`
  说明当前租户下没有查到对应协议产品，仍可继续手工创建模拟设备。
