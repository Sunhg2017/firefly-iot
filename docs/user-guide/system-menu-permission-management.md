# 系统菜单权限管理使用说明

## 适用角色

1. 系统运维管理员
2. 平台权限管理员

## 功能入口

登录系统运维空间后，进入：

1. `用户与权限`
2. `系统菜单权限`

## 页面说明

页面分为三块：

1. 顶部页签
   - 切换 `系统运维空间` 和 `租户业务空间`
2. 左侧菜单树
   - 查看当前空间的基础菜单层级
   - 选择某个节点后查看详情
3. 右侧详情区
   - 查看菜单属性
   - 查看当前菜单绑定的权限集合

## 新建菜单

点击右上角 `新建菜单`，系统会以抽屉步骤表单完成配置。

### 第一步：菜单信息

需要配置：

1. 所属空间
2. 父级目录
3. 菜单唯一键
4. 菜单名称
5. 菜单图标
6. 前端路由
7. 排序
8. 是否显示
9. 是否进入角色权限目录

说明：

1. 如果不填写前端路由，系统会把该节点视为目录节点。
2. 目录节点只用于组织菜单结构，不能直接绑定权限。
3. 菜单唯一键是后续空间授权和角色目录的统一标识，创建后不支持修改。
4. 菜单图标为必选项，系统会以下拉方式展示当前已注册的图标列表，不能再手工输入未注册图标名。

### 第二步：权限集合

1. 页面节点可以选择当前菜单可分配的权限编码。
2. 目录节点会直接提示“无需配置权限”。
3. 角色授权页会基于这里绑定的权限集合自动生成权限分组。

## 编辑菜单

在右侧详情区点击 `编辑`：

1. 可调整名称、图标、排序、显示状态和权限集合。
2. 可修改父级目录，但不能把节点移动到自己的子节点下面。
3. 如果某节点已经有子菜单，就不能再改成页面节点。
4. 如果历史菜单缺少图标，编辑时必须先补齐图标后才能保存。

## 删除菜单

在右侧详情区点击 `删除`：

1. 会删除当前节点及全部下级菜单。
2. 会同步清理菜单权限绑定。
3. 如果是租户空间菜单，还会同步清理所有租户对这些菜单的授权记录。

## 和其他功能的关系

### 租户空间授权

系统运维人员在租户管理里为租户分配空间菜单时：

1. 授权树来源于这里的租户业务空间菜单树。
2. 只有页面节点会被真正授权给租户。

### 角色授权

平台角色和租户角色在选择权限时：

1. 权限目录来源于这里维护的菜单权限集合。
2. 只有 `角色目录展示` 为开启且已绑定权限的页面节点，才会进入角色权限目录。

### 用户实际访问

用户登录后实际能否访问页面，取决于两个条件：

1. 角色是否拥有页面所需权限。
2. 当前空间是否授权了该菜单。

只满足其中一个条件都不行。

## 使用建议

1. 先规划好菜单层级，再批量维护权限集合。
2. 目录节点尽量只做分组，不要塞权限语义。
3. 同一个页面的所有权限尽量集中绑定到同一个菜单节点，避免角色目录分散。
4. 变更租户空间菜单后，建议让租户管理员重新登录验证最新效果。

## 注意事项

1. 本功能按新模型直接运行，不兼容旧菜单快照。
2. 如果历史环境中已存在旧数据，建议先清理数据库后再启用。
3. 不要再通过改前端常量或手工改数据库来补菜单，正式入口就是系统菜单权限管理和对应 Flyway 迁移。
4. 新增菜单时，Flyway 菜单基础数据、前端菜单配置和前端图标映射必须一起补齐，否则菜单可能显示名称但不显示图标。
