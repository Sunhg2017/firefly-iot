# 设备消息工作台使用说明

## 适用角色

- 设备联调人员
- 协议接入人员
- 平台运维人员

## 使用流程

1. 在页面顶部搜索并选择目标设备
2. 查看右侧设备上下文，确认当前产品、在线状态和能力数量
3. 根据场景切换到“属性设置”“服务调用”或“原始消息”
4. 优先使用页面提供的模板生成 JSON，再按实际业务调整字段值
5. 点击发送后，在顶部查看最近一次发送结果

## 三个工作区说明

### 属性设置

- 适合标准属性下发
- 可以从物模型中选择部分属性生成模板
- 若想快速生成一整份骨架，可直接点击“填充全部可写属性”

### 服务调用

- 只展示产品自定义服务
- 切换服务后会尽量自动带入输入参数模板
- 右侧会显示服务说明和参数摘要

### 原始消息

- 适合高级调试或私有协议场景
- 可按消息类型重新载入示例模板
- 页面只校验 JSON 是否可解析，不会替你校验业务字段

## 注意事项

- 页面顶部设备一旦切换，三个工作区都会共享新的目标设备
- 如果当前产品没有配置可写属性或自定义服务，对应工作区会显示空态提示
- 最近一次发送结果只保留在当前页面会话内，刷新页面后会重新开始
