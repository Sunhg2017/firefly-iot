# Java 全限定名调用清理使用说明
> 适用角色: 开发、评审、维护人员
> 日期: 2026-03-14

## 1. 后续开发怎么写

- 没有类名冲突时，直接 `import` 后使用简单类名
- 不要在业务代码里写 `new com.xxx...`、`java.util.Map.of(...)` 这类全限定调用
- 如果确实有同名冲突，再显式限定冲突的那个类

## 2. 哪些情况可以保留

- AOP pointcut 表达式里的注解类名
- OpenAPI `@RequestBody` 与 Spring `@RequestBody` 同名冲突
- 其他已经确认存在简单类名冲突的极少数场景

## 3. 这次清理的意义

这不是功能改造，而是统一代码可读性和维护规范。后续看到类似写法，默认应视为需要修正的代码味道，而不是个人风格选择。
