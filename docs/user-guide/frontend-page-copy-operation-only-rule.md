# 前端页面说明文案使用说明

## 适用对象

- 前端开发人员
- 产品设计人员
- 评审与验收人员

## 使用规则

页面中的说明文案只保留以下内容：

- 当前页面的操作步骤
- 执行前必须满足的前置条件
- 本次操作会带来的结果影响
- 当前页面的限制条件
- 失败后的处理提示

页面中禁止继续出现以下内容：

- 设计思路
- 视觉风格说明
- 模块背景介绍
- 能力宣传文案
- 与当前操作无关的补充说明

## 编写方式

推荐写法：

- `先选择设备，再切换到属性设置页签发送消息。`
- `这里只调整当前空间已授权菜单的名称和排序，不会新增权限。`
- `先按服务筛选，再查看接口地址、参数和示例请求。`

不推荐写法：

- `页面统一采用浅色卡片风格，提升信息层级。`
- `该模块用于统一管理多类能力。`
- `这里集中收口平台多个功能入口。`

## 自检清单

提交前请逐项检查：

1. 页头描述是否能直接指导下一步操作。
2. 提示卡片是否只保留限制、结果和错误处理信息。
3. 页面里是否出现“设计、风格、体验、统一收口、面向、用于、建议”等介绍式表达。
4. 需要说明设计取舍时，是否已经写入 `docs/` 而不是放在页面里。
