# 设备模拟器使用说明

> 适用角色: 联调开发、测试人员、实施人员
> 更新时间: 2026-03-19

## 1. 新建设备怎么用

点击左侧设备列表右上角 `新建` 后，会打开抽屉式分步表单。
抽屉底部的操作栏已经和整体深色主题保持一致，如果看到明显白色块，通常说明本地还是旧资源。
协议说明区和配置摘要区也已经统一为深色信息面板，不会再出现大块浅色底卡片。

### 1.1 第一步：基本信息

填写：

- 模拟设备名称
- 接入协议

选中协议后，页面会展示当前协议的用途说明。

### 1.2 第二步：接入参数

系统只显示当前协议真正需要的最小必填项。

常见示例：

- HTTP / CoAP：`productKey`、`deviceName`、`deviceSecret`
- MQTT：`productKey`、`deviceName`、认证方式、Broker 地址
- Video：先选产品或填写 `productKey`，再补媒体服务地址、视频模式、国标设备 ID 或 RTSP 地址

创建完成后，右侧设备控制区会先展示“接入概览”，把地址、标识和密钥状态整理成卡片，不需要再从长表单里回忆参数。

### 1.3 第三步：扩展配置

这一阶段用于补充低频或高级参数，例如：

- MQTT 的 clientId、遗嘱消息
- GB28181 的 SIP 参数和通道列表
- WebSocket 的连接识别参数

如果前一步已经配错，可以直接点 `上一步` 返回修改。

## 2. 各协议使用建议

### 2.1 HTTP / CoAP

适合快速验证平台鉴权和属性上报链路。

如果 HTTP 产品是一机一密，直接填写 `deviceSecret`。

如果 HTTP 产品是一型一密，选择 `一型一密` 后填写：

- `ProductKey`
- `DeviceName`
- `动态注册服务地址`
- `ProductSecret`

连接时系统会先动态注册，自动换取 `deviceSecret`，再继续完成 HTTP 鉴权。

### 2.2 MQTT

如果产品是一机一密，直接填写 `deviceSecret`。

如果产品是一型一密，选择对应认证方式后填写：

- 动态注册服务地址
- `productSecret`

### 2.3 视频

如果你是联调 GB28181：

1. 第二步先选择 GB28181 产品，或手工填写对应 `ProductKey`
2. 再填媒体服务地址和国标设备 ID
3. `DeviceName` 会自动跟随国标设备 ID 生成
4. 第三步再补 SIP 配置和通道

如果你只想验证 RTSP 代理，直接选 `RTSP 代理` 并填写 RTSP 地址即可。
RTSP 代理模式下，先选择 RTSP 产品或填写对应 `ProductKey`，`DeviceName` 会自动复用第一步填写的模拟设备名称。
如果中途切换了视频模式，原先选择的产品会自动清空，需要重新选择与当前模式一致的产品。

### 2.4 WebSocket

WebSocket 当前还需要连接识别参数，所以它们放在第三步。

如果你只是做地址连通性验证，可以先只填 `wsEndpoint`，后续再补高级参数。

## 3. 设备列表怎么用

左侧设备列表支持：

- 顶部总览卡片先看总量、在线、离线和异常
- 搜索名称 / ProductKey / DeviceName
- 按协议筛选
- 按状态筛选
- 复制设备
- 删除设备
- 导入 JSON / CSV
- 导出当前设备配置
- 批量连接
- 批量断开

现在每台设备会以卡片形式展示，卡片上能直接看到：

- 当前协议
- 在线 / 离线 / 异常状态
- 关键标识，例如 `ProductKey / DeviceName`
- 如果接入参数不完整，会直接显示“待补充 ...”
- 已发送次数

## 4. 常见操作

### 4.1 快速复制一个已配置设备

点击设备右侧复制按钮，系统会生成一个“副本”，适合批量造数。

### 4.2 批量连接

点击 `全部连接`，系统会批量尝试连接离线的 HTTP / CoAP / MQTT 设备。
如果某台设备缺少接入参数，系统会直接记录缺项并跳过，不会把空认证参数继续发给服务端。

### 4.3 批量断开

点击 `全部断开`，系统会断开当前在线设备。

## 5. 注意事项

- WebSocket 当前仍可能需要内部识别参数，这是由现有 connector 接入方式决定的。
- MQTT 一型一密场景依赖动态注册接口可用。
- GB28181 配置较多时，建议先创建基础设备，再逐步补 SIP 和通道。
- 如果新建设备抽屉底部出现白色块，优先重启本地模拟器并确认已加载最新前端资源。
- 如果 HTTP 认证失败，先看“接入概览”和 HTTP 请求历史，确认认证参数是否齐全、请求是否已经真正发出。
## 6. 2026-03-14 使用补充

### 6.1 一型一密设备如何填写昵称

1. 新建设备时在基础信息步骤填写模拟器名称。
2. “模拟设备名称”会自动同步为平台设备昵称，不需要再额外填写 `Nickname`。
3. 如需调整平台侧昵称，直接修改“模拟设备名称”即可。

### 6.2 为什么第二次连接不再重复注册

- 一型一密设备首次连接后会拿到平台返回的 `deviceSecret`。
- 后续再连接时，模拟器会直接使用已有 `deviceSecret` 完成认证，不会再次发起动态注册。

### 6.3 删除设备的行为

- 普通本地模拟设备：只删除模拟器本地配置。
- 动态注册创建的 HTTP / MQTT 一型一密设备：会先删除平台侧设备，再删除本地配置。
- 如果平台删除失败，模拟器会保留本地记录并提示失败，方便你排查后重试。


## 7. 2026-03-19 物模型联动与生命周期事件使用补充

### 7.1 如何按产品物模型模拟数据

1. 先切换到目标环境，并登录当前环境。
2. 在设备基础信息中填写正确的 `ProductKey`。
3. 右侧“平台物模型”卡片会自动同步当前租户下的产品物模型，无需再配置 AK/SK。
4. 进入右侧 `数据上报` 面板，将数据源切换到“物模型模拟”。
5. 根据当前上报类型选择要发送的内容。
   - 属性上报：可选择“全部属性”或单个属性。
   - 事件上报：选择具体事件。
   - OTA：继续使用自定义 JSON。
6. 参考面板中的随机样例预览，确认 payload 结构后再点击发送。

### 7.2 连接和断开时会发生什么

- 设备连接成功后，模拟器会自动发送一次“上线”事件。
- 点击 `断开` 主动断开时，模拟器会先发送一次“下线”事件。
- 设备保持在线时，模拟器会定期自动发送“心跳”事件。
- 如果是一型一密设备，且服务端对应设备已被删除，模拟器会在鉴权失败后自动重走一次动态注册并刷新本地 `DeviceSecret`。

### 7.3 如何调整心跳频率

- 连接成功后，`数据上报` 面板会出现 `心跳` 配置。
- 当前支持 `15 / 30 / 60 / 120 / 300` 秒几个档位。
- 修改后会立即重建本地心跳定时器，并按新周期继续发送。

### 7.4 什么时候切回自定义 JSON

- 产品还没有配置对应属性或事件时。
- 需要构造特殊字段或边界值时。
- 调试 CoAP OTA 上报时。

切回后可以继续手工编辑 JSON 并发送自定义 payload。
## 8. 2026-03-19 字段规则与影子同步使用补充

### 8.1 如何配置物模型字段规则

1. 先连接 HTTP、CoAP 或 MQTT 模拟设备，并进入右侧 `数据上报` 面板。
2. 将数据源切到“物模型模拟”，再选择一个属性、全部属性，或一个事件。
3. 在 `字段模拟规则` 区域为字段配置规则：
   - `随机`：继续按物模型默认定义生成。
   - `区间`：适用于数值和时间戳字段。
   - `固定`：适用于固定值，数组和结构体请填写固定 JSON。
4. 规则保存后，预览、发送和自动上报都会立即使用新规则。

### 8.2 字段规则支持范围

- 属性支持根字段以及结构体、数组里的嵌套字段。
- 事件支持 `outputData` 中的字段。
- 字符串字段支持随机字符串和 IP 地址两种随机策略。
- 数组字段支持最小长度和最大长度。

### 8.3 什么时候用固定 JSON

- 需要稳定复现某个复杂属性结构时。
- 需要让数组长度、字段顺序、嵌套内容完全固定时。
- 需要和后端 bug、规则引擎或告警条件做精确联调时。

### 8.4 影子现在会怎么变化

- 属性上报成功后，平台设备影子的 `reported` 会跟着刷新。
- 事件上报不会修改影子，只会进入事件链路。
- 即使属性上报报文用了 `params` 或 `properties` 包裹层，平台也会提取真实属性来更新影子。

### 8.5 配置是否会丢失

- 不会。字段规则会跟随设备配置一起持久化。
- 克隆设备时会复制规则。
- 导出再导入设备配置时也会保留规则。

### 8.6 页面文案有什么变化

- 设备主操作区已经统一为中文按钮和状态，例如 `连接`、`断开`、`发送`、`离线`、`连接中`、`在线`、`异常`。
- 物模型调试区统一使用 `数据上报`、`自动上报`、`心跳`、`字段模拟规则` 等中文名称。
- MQTT 区的 `Topic`、`Retain`、`PUB / SUB` 等旧文案已经分别调整为 `主题`、`保留`、`发布 / 接收`。
- CoAP 区的设备影子操作提示也已统一为中文，更方便现场调试时快速识别操作入口。
