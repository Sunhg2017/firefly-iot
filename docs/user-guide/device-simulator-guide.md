# 设备模拟器使用说明

## 1. 新建 Video 设备

1. 点击 `新建`
2. 选择 `Video`
3. 选择摄像头产品或填写 `ProductKey`
4. 选择 `GB28181 / RTSP / RTMP`
5. 补充对应参数

## 2. Video 模式怎么连接

点击 `连接` 后，模拟器会先同步平台设备资产，再执行媒体控制。

同步顺序：

1. 通过 `DEVICE` 路由查重
2. 创建或更新视频设备资产
3. 保存 `platformDeviceId`
4. 后续控制都使用这个 `platformDeviceId`

Video 设备顶部状态卡口径：

1. `GB28181`：显示 `SIP 注册` 与 `SIP 心跳`
2. `RTSP / RTMP`：显示 `平台同步` 与 `连接状态`

## 3. GB28181 怎么调

1. 填写 `GB 设备编号`
2. 填写 `GB 域`
3. 填写设备级 `SIP 密码`（必填）
4. 媒体源保持 `本地摄像头`
5. 连接后执行 `SIP 注册`
6. 需要时再执行 `开启心跳`
7. 点击 `开始推流` 后，模拟器会在收到 INVITE 时自动发送本地码流

## 4. RTSP / RTMP 怎么调

1. 选择 `RTSP` 或 `RTMP`
2. 媒体源选择：
   - `本地摄像头`：系统自动生成并回填 `sourceUrl`
   - `外部源地址`：手工填写 `sourceUrl`
3. 点击连接
4. 连接成功后点击 `开始推流`
5. 连接成功后可执行播放、截图、录像

## 5. 常见问题

### 5.1 为什么第二次连接没有再创建新设备

因为模拟器会优先复用同一台设备的 `platformDeviceId`。

### 5.2 为什么通道列表现在不走 MEDIA

因为通道是设备资产主数据，已经改为走 `DEVICE` 路由。

### 5.3 为什么页面里不再显示“平台视频设备”

因为现在绑定的是统一设备资产，所以口径统一改成 `平台设备资产`。

### 5.4 本地摄像头启动失败怎么办

1. 检查系统是否已授权模拟器摄像头权限
2. 如果日志提示 `SIP 提示：检测到摄像头参数不兼容，改用 ... 重试`，可先等待系统根据失败日志切换到支持模式自动重试
3. 如果随后提示 `已切换设备默认采集模式重试`，继续等待第二阶段重试结果
4. 检查当前环境网络是否可达媒体服务
5. 检查日志里是否出现 `本地码流启动失败`，按提示处理摄像头占用或设备不存在问题
