# 设备模拟器使用说明

## 1. 新建 Video 设备

1. 点击 `新建`
2. 选择 `Video`
3. 选择摄像头产品或填写 `ProductKey`
4. 选择 `GB28181 / RTSP / RTMP`
5. 如使用 `本地摄像头`，在高级配置里选择摄像头设备和采集模式
6. 补充对应参数

## 2. 编辑已有设备

1. 在左侧设备卡片点击 `编辑`
2. 在抽屉里修改配置后点击 `保存修改`
3. 如果设备当前已经连接，保存时会先断开旧连接
4. 保存完成后重新点击 `连接`，按新配置重新建立连接
5. 如果是 Video 设备且修改了 `国标设备 ID`、视频模式、媒体源类型或外部源地址，下一次连接会重新绑定平台视频资产
6. 如果只想调整摄像头和采集模式，也可以直接在 Video 详情页打开参数弹窗修改

## 3. Video 模式怎么连接

点击 `连接` 后，模拟器会先同步平台设备资产，再执行媒体控制。

同步顺序：

1. 通过 `DEVICE` 路由查重
2. 创建或更新视频设备资产
3. 保存 `platformDeviceId`
4. 若为 `GB28181`，立即自动发起 SIP 注册，并在注册成功后自动发送心跳
5. 后续控制都使用这个 `platformDeviceId`

Video 设备顶部状态卡口径：

1. `GB28181`：显示 `SIP 注册` 与 `SIP 心跳`
2. `RTSP / RTMP`：显示 `平台同步` 与 `连接状态`

## 4. GB28181 怎么调

1. 填写 `GB 设备编号`
2. 填写 `GB 域`
3. 填写设备级 `SIP 密码`（必填）
4. 媒体源保持 `本地摄像头`
5. 如有多路本机摄像头，在高级配置里选择目标摄像头设备和采集模式
6. 点击 `连接`，模拟器会自动发起 `SIP 注册` 并在注册成功后自动开始心跳
7. 点击 `开始推流` 后，模拟器会在收到 INVITE 时自动发送本地码流
8. 如需手动重试注册、停心跳或抓报文，可在详情页继续使用 `SIP 注册 / 开启心跳 / 停止心跳` 按钮
9. 如需修改摄像头设备或采集模式，可在详情页 `参数` 中直接调整
10. 正常断开或手动注销后，不会再在 32 秒后补打一条 `REGISTER(unregister) transaction timeout`

## 5. RTSP / RTMP 怎么调

1. 选择 `RTSP` 或 `RTMP`
2. 媒体源选择：
   - `本地摄像头`：系统按当前环境的 `ZLM 推流主机 / RTSP 端口 / RTMP 端口` 自动生成并回填 `sourceUrl`
   - `本地摄像头`：系统会同时自动选择模拟器本机 IPv4 作为 `设备 IP`，平台设备列表里的 IP 列展示这个地址，不再显示 ZLM 基础设施 IP
   - `外部源地址`：手工填写 `sourceUrl`
3. 如使用 `本地摄像头`，在高级配置里选择目标摄像头设备和采集模式
4. 点击连接
5. 连接成功后点击 `开始推流`
6. 连接成功后可执行播放、截图、录像
7. 如使用本地摄像头，可在流控制区点击 `采集参数` 调整摄像头和采集模式
8. 如果同一台设备已经有活跃的 `live/{streamId}` 代理流，再次点击开始推流时平台会直接复用，不会再报 `This stream already exists`

## 6. 常见问题

### 6.1 为什么第二次连接没有再创建新设备

因为模拟器会优先复用同一台设备的 `platformDeviceId`。

### 6.2 为什么通道列表现在不走 MEDIA

因为通道是设备资产主数据，已经改为走 `DEVICE` 路由。

### 6.3 为什么页面里不再显示“平台视频设备”

因为现在绑定的是统一设备资产，所以口径统一改成 `平台设备资产`。

### 6.4 本地摄像头启动失败怎么办

1. 检查系统是否已授权模拟器摄像头权限
2. 检查当前环境里的 `ZLM 推流主机 / RTSP 端口 / RTMP 端口` 是否填写为真实可达地址，避免继续推到 `localhost:554`
3. 检查平台设备列表中的 IP 是否已显示为模拟器主机地址；若仍显示 ZLM 地址，重新打开设备编辑抽屉确认 `设备 IP` 已自动回正
4. 如果日志出现 `本地码流提示：检测到摄像头参数不兼容，改用 ... 重试`，可先等待系统按支持模式顺序自动重试
5. 如果日志出现 `本地码流提示：Selected pixel format ...`，继续等待系统按兼容像素格式自动重试
6. 如果随后提示 `已切换设备默认采集模式重试`，继续等待第二阶段重试结果
7. 检查当前环境网络是否可达媒体服务
8. 检查日志里是否出现 `本地码流异常：本地码流启动失败`，按提示处理摄像头占用或设备不存在问题

### 6.5 断开后 32 秒又出现 REGISTER(unregister) transaction timeout 怎么办

如果你仍看到这条日志，通常说明当前运行的还是旧模拟器版本；重新构建或替换安装包后再试。
