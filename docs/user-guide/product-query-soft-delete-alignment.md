# 设备连接报 products.deleted_at 缺失时的使用说明
> 适用角色: 运维、联调开发、测试
> 日期: 2026-03-14

## 1. 问题表现

在设备连接、动态注册或协议调试时，如果看到下面这类报错：

`ERROR: column "deleted_at" does not exist`

且 SQL 指向 `products` 表，说明产品查询口径与数据库表结构不一致。

## 2. 当前结论

- 这不是产品配置填写错误。
- 这是服务端产品查询 SQL 写错了，错误引用了不存在的 `deleted_at` 字段。
- 修复后，按 `productKey` 的产品查询会恢复正常，设备连接链路可继续执行。

## 3. 修复后如何确认

1. 重启 `firefly-device` 服务。
2. 重新发起设备连接或动态注册。
3. 如果不再出现 `products.deleted_at` 报错，且产品能被正确识别，说明修复已经生效。
