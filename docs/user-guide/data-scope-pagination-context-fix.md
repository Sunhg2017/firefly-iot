# 列表分页恢复说明

## 本次修复了什么

这次修复解决的是列表页一个公共层问题，不只影响单个页面：

- 页面能看到记录
- 但总数显示为 0
- 或者明明设置了分页大小，接口却返回了超出分页大小的全量记录

升级后，分页会重新按标准口径执行：

- 先统计总数
- 再按页返回记录
- 对受数据权限控制的页面，统计和记录使用同一套权限范围

## 你会看到的变化

- 设备管理列表总数恢复正常
- 产品列表分页恢复正常
- 其他依赖公共分页能力的列表页，分页器会和记录数一致
- 按关键词筛选后，不会再出现“列表有数据但总数变成 0”

## 建议验证的页面

1. 设备管理
2. 产品管理
3. 系统运维里任一带分页的列表
4. 其他受数据权限控制的列表页

重点看三件事：

- 当前页返回条数不应超过你设置的分页大小
- 表格里有记录时，总数不应再显示 0
- 翻页、筛选后，总数和页码应保持一致

## 如果仍有异常

请把下面信息交给运维或开发：

- 具体页面名称
- 查询条件
- 当前页返回了几条记录
- 页面显示的 `total` 是多少
- 同时附上对应接口响应和服务日志

这样可以更快区分是服务未升级，还是该页面仍有独立问题。
