# 工作空间授权与菜单使用说明

## 1. 适用角色

- 系统运维管理员
- 租户管理员
- 租户业务用户

## 2. 系统运维如何给租户分配租户空间菜单

入口：
`系统运维空间 -> 租户管理 -> 空间授权`

操作步骤：

1. 打开目标租户详情。
2. 进入“空间授权”。
3. 勾选该租户允许使用的租户空间菜单。
4. 保存授权。

保存后预期结果：

- 租户超级管理员重新登录后，应同时拿到对应菜单路径和角色权限。
- 如果新授权已经保存，但登录后仍直接进入 `/403`，就不是正常现象，应继续检查权限同步是否成功。

说明：

- 这里是租户空间菜单的唯一授权入口。
- 新租户创建后，不会再自动获得全部租户空间菜单。
- 如果系统运维暂时没有分配任何菜单，租户登录后就不会看到未授权的租户业务菜单。
- 在完全未授权时，租户侧也不会出现“菜单配置”入口。

## 3. 租户管理员如何理解当前菜单显示结果

- 租户看到哪些菜单，先取决于系统运维是否已经给本租户授权。
- 在系统运维已授权的前提下，还会继续受到当前用户角色权限限制。
- 如果某个菜单没有被系统运维授权，租户管理员无法通过角色配置或菜单自定义把它“配出来”。
- “菜单配置”不是零授权兜底入口，只有租户已经拿到至少一项业务菜单授权后才会出现。

## 4. 常见问题

### 4.1 为什么新租户登录后没有看到业务菜单

通常是因为系统运维还没有给该租户分配租户空间菜单。请先在“租户管理 -> 空间授权”中勾选需要开放的菜单。

### 4.2 为什么新租户连菜单配置也看不到

这是预期行为。菜单配置只能自定义已授权菜单；当租户还没有任何已授权菜单时，系统不会单独放开菜单配置入口。

### 4.3 为什么租户只能看到部分菜单

常见原因有两个：

- 系统运维只给该租户授权了部分租户空间菜单。
- 当前登录用户缺少对应菜单绑定的权限。

### 4.4 如果历史租户菜单显示异常怎么办

如果历史环境里还残留旧的全量授权数据，请先由运维清理旧的 `tenant_menu_configs` 记录，再按当前真实需求重新授权。

### 4.5 为什么已经做了空间授权，租户超级管理员还是直接 403

先看两个接口结果：

- `/api/v1/users/me` 里的 `authorizedMenuPaths` 是否已经有菜单路径
- `/api/v1/users/me/permissions` 是否也拿到了对应权限

如果前者有值、后者为空，说明这次不是“没授权到菜单”，而是“角色权限没有同步成功”。这种情况需要系统运维或运维同学发布修复版本并重新同步该租户权限，单靠租户侧自己刷新菜单或改角色名称无法恢复。
