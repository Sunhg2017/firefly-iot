# 工作空间授权与菜单收口使用说明

## 1. 适用角色

- 系统运维管理员
- 租户管理员
- 租户业务用户

## 2. 系统运维如何给租户授权空间功能

入口：

`系统运维空间 -> 租户管理`

操作步骤：

1. 打开目标租户详情。
2. 进入“空间授权”。
3. 勾选该租户允许使用的租户空间菜单。
4. 保存授权。

说明：

- 这里是租户业务空间菜单的唯一授权入口。
- 租户侧不再维护自己的菜单树。
- 保存后，租户管理员角色会自动同步到当前授权范围。

## 3. 租户管理员如何管理租户用户和角色

入口：

- `租户业务空间 -> 用户管理`
- `租户业务空间 -> 角色管理`

推荐使用顺序：

1. 先在角色管理中创建业务角色。
2. 给角色分配当前租户已授权的权限。
3. 再在用户管理中创建用户。
4. 给用户分配一个或多个角色。

说明：

- 租户管理员拥有租户空间的完整管理视角，但角色授权范围仍受租户空间授权限制。
- 系统运维空间创建的用户和角色，只能用于系统运维空间。
- 租户空间创建的用户和角色，只能用于当前租户业务空间。

## 4. 菜单与权限现在如何生效

- 菜单显示由“当前工作空间 + 当前权限”共同决定。
- 不再存在“租户菜单配置”页面。
- 不再存在“租户管理员默认权限配置”页面。
- 如果平台撤销了某项空间功能，该功能对应菜单和可授权权限都会一起收回。

## 5. 协议解析规则如何维护 `DEVICE_LIST`

入口：

`租户业务空间 -> 协议解析`

使用规则：

1. 选择发布策略 `DEVICE_LIST`。
2. 维护目标设备名称列表 `deviceNames`。
3. 保存并发布。

注意事项：

- 不再支持在规则里维护 `deviceIds`。
- 页面上选择设备时应使用业务标识，不需要记忆数据库主键。

## 6. 常见问答

### 6.1 为什么租户侧看不到某个菜单

通常有两个原因：

- 平台尚未给该租户授权该功能。
- 当前用户没有该菜单对应权限。

### 6.2 为什么角色列表里选不到某些权限

因为当前租户并未被授权对应模块，角色只能分配当前空间允许的权限。

### 6.3 老数据里如果还有旧配置怎么办

当前系统未正式启用，建议直接清理数据库旧数据后重新授权，而不是继续沿用旧配置。
