# 设备模拟器连接持久化使用说明
> 适用角色: 联调开发、测试、运维
> 日期: 2026-03-14

## 1. 现在有什么变化

设备模拟器现在会记住两类信息：

- 设备配置
- 上次需要恢复的连接状态

这意味着你关闭模拟器后，再次打开时：

- 之前创建的设备仍然在
- 对于支持恢复的协议，之前在线的设备会自动尝试重连

## 2. 哪些协议会自动恢复

支持自动恢复：

- HTTP
- CoAP
- MQTT
- SNMP
- Modbus
- WebSocket
- TCP
- UDP
- LoRaWAN

暂不自动恢复：

- Video

## 3. 怎么触发恢复

1. 正常连接设备。
2. 直接关闭模拟器。
3. 重新打开模拟器。
4. 系统会自动恢复上次需要保持连接的设备。

## 4. 哪些情况下不会恢复

- 你手工点了“断开”
- 你批量执行了“全部断开”
- 该设备本来就没有连上
- 该协议不在自动恢复范围内

## 5. 注意事项

- 自动恢复只恢复连接意图，不恢复临时 token、自动上报定时器和运行统计。
- 如果服务端地址不可用，设备会在启动恢复后进入 `error` 状态。
- Video 设备需要手工重新连接，避免在服务端重复创建视频设备资源。
