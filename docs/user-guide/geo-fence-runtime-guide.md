# 地理围栏运行时使用说明

## 1. 适用角色

- 租户管理员
- 设备运维人员
- 定位设备实施人员

## 2. 现在地理围栏怎么生效

地理围栏现在不只是页面配置。

设备只要持续上报定位属性，系统就会自动：

1. 记录位置历史
2. 判断是否发生围栏进入/离开
3. 生成围栏事件

## 3. 设备需要满足什么条件

设备属性上报里至少要有经纬度字段，支持两种命名：

1. `longitude` + `latitude`
2. `lng` + `lat`

如果设备不上报这些字段，围栏不会自动触发。

## 4. 如何配置围栏

1. 进入 `设备资产 / 地理围栏`
2. 新建围栏
3. 选择围栏类型
4. 填写圆形中心点和半径，或填写多边形坐标
5. 选择触发类型
6. 保存并启用

## 5. 触发类型怎么理解

### 5.1 进入触发

设备从围栏外移动到围栏内时触发。

### 5.2 离开触发

设备从围栏内移动到围栏外时触发。

### 5.3 进出均触发

只要发生进入或离开都会触发。

## 6. 为什么第一次定位不上报围栏事件

因为系统需要有上一条位置作为比较基线。

第一次定位只会记录位置，不会直接判断“进入”或“离开”。

从第二次开始，只要位置状态发生变化，才会生成围栏事件。

## 7. 围栏事件会出现在哪里

围栏变化会写成设备事件，事件类型为：

- `geofenceAlarm`

你可以在设备事件列表中看到：

1. 围栏名称
2. 进入或离开
3. 当前位置坐标
4. 触发时间

## 8. 如何配合规则引擎使用

围栏变化已经会自动进入规则引擎输入流。

如果你想继续做联动，可以基于围栏事件配置后续动作，例如：

1. 发送通知
2. 调用 Webhook
3. 触发其它自动化流程

## 9. 常见问题

### 9.1 页面里建了围栏，为什么没有任何反应

优先检查：

1. 设备是否真的在上报经纬度
2. 经纬度字段名是否符合系统支持的命名
3. 围栏是否启用
4. 设备是否已经有两次以上的位置上报

### 9.2 为什么设备一直在围栏内，但只触发了一次

这是正常行为。围栏只在状态变化时触发，不会每次定位上报都重复生成同样的进入事件。

### 9.3 围栏会不会自动变成告警

当前不会直接进入告警规则模块。围栏变化会先生成设备事件，再由规则引擎或后续编排去消费。
