# 设备 Kafka 消费落库修复使用说明
> 适用角色: 运维、联调开发、设备接入开发
> 日期: 2026-03-14

## 1. 修复后有什么变化

设备通过协议接入层上报的属性和事件，在进入 Kafka 后会继续落到设备数据表，不再只更新影子或只转发规则引擎。

## 2. 现在如何判断链路正常

1. 上报属性后，设备影子和设备最新遥测都应该能看到数据
2. 上报事件后，设备事件列表应该能查到对应事件
3. 如果配置了规则引擎，消息仍会继续转发，不受这次修复影响

## 3. 注意事项

- 事件 payload 字段结构支持一定兼容提取，但最好显式带上 `eventType`、`eventName`、`level`
- 如果设备事件只带业务自定义字段，系统仍会入库，但事件类型会回退为 `EVENT_REPORT`
