# 设备拓扑使用说明
> 适用角色: 租户设备管理员、实施、运维
> 日期: 2026-03-22

## 1. 在哪里进入

有两个入口：

1. 租户空间 → 设备资产 → 设备拓扑
2. 设备管理页头 → 设备拓扑

## 2. 这个页面看什么

页面把设备分成三类：

- 拓扑链路：存在父子关系的设备链路
- 独立设备：没有上级也没有下级的设备
- 断链设备：配置了上级，但当前找不到上级设备的设备

筛选后，页面会保留上下游链路，不会把真实关系直接裁断。

## 3. 怎么筛选

可用筛选项：

- 关键字
- 所属产品
- 所属分组
- 设备状态
- 在线状态

推荐顺序：

1. 先选产品
2. 再选分组或状态
3. 最后用关键字定位具体设备

如果页面提示设备范围过大，请继续缩小筛选条件。

## 4. 怎么看拓扑链路

1. 打开“拓扑链路”页签
2. 展开树节点查看父子关系
3. 蓝色边框表示当前筛选直接命中的节点
4. 点击任一节点，在右侧查看详情和完整路径

右侧会显示：

- 设备名称
- 产品名称与 `productKey`
- 当前链路位置
- 在线状态与设备状态
- 下级数量
- 固件版本、IP、最后在线时间

## 5. 独立设备怎么处理

如果设备出现在“独立设备”里，说明它当前没有参与父子链路。

常见处理方式：

1. 确认这类设备本来就不需要挂载到网关
2. 如果应该挂在某个网关下，回到设备管理或接入侧补齐挂载关系

## 6. 断链设备怎么处理

如果设备出现在“断链设备”里，优先检查：

1. 上级网关是否已经被删除
2. 上级网关是否不在当前筛选范围
3. 上级关系是否写错

处理完成后刷新页面，确认设备已回到正常链路中。

## 7. 使用注意事项

- 页面优先展示设备名称、别名和 `productKey`，不要求手工输入数据库主键。
- 节点详情里的“路径”用于快速确认设备位于哪条链路。
- 如果筛选结果为空，先放宽关键字，再检查产品和分组是否选错。
