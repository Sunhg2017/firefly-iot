# 设备事件路由修复使用说明
> 适用角色: 运维、联调开发、规则引擎维护人员
> 日期: 2026-03-14

## 1. 这次修复解决了什么

设备事件上报后，服务端现在会正确把消息转发到规则引擎输入 topic，避免消息再次回写到设备事件 topic 形成循环消费。

## 2. 怎么判断修复已生效

1. 设备触发一次事件上报。
2. 在 `firefly-device` 日志中按该次 `messageId` 搜索。
3. 正常情况下，同一条事件只会出现一次消费日志，然后进入规则引擎后续处理。

## 3. 联调时注意什么

- 设备上报链路仍按消息类型自动路由，这部分行为没有改变。
- 只有服务端内部“转发给规则引擎”的环节改成了显式 topic 投递。
- 如果要继续排查事件未触发告警，应转向规则引擎规则、过滤条件和通知链路，而不是设备事件 Kafka 自循环问题。
