# OTA 与固件工作台使用说明

## 入口

- 租户空间进入 `OTA 与固件`
- 页面分为 `固件库`、`设备版本`、`升级任务` 三个页签

## 固件库

### 上传固件

1. 打开 `固件库`
2. 点击 `上传固件`
3. 选择所属产品
4. 输入版本号
5. 选择固件文件
6. 提交后系统自动保存文件地址、大小和 MD5

### 固件流转

- 草稿：可编辑、可删除、可验证
- 已验证：可发布
- 已发布：可用于创建升级任务

## 设备版本

### 查看设备版本

- 可以按产品、固件、在线状态、升级状态筛选
- 未登记过固件的设备也会出现在列表中

如果打开 `设备版本` 页签时直接报错，让运维先确认 `firefly-device` 已升级到修复版；旧版本会因为设备版本总览 SQL 错误引用 `p.deleted_at` 而返回 500。

### 登记单台设备版本

1. 在设备行点击 `登记固件`
2. 选择固件版本
3. 保存

### 批量登记

1. 勾选多台设备
2. 点击 `批量登记`
3. 选择固件版本
4. 保存

限制：

- 只能批量登记同一产品下的设备

## 升级任务

### 创建任务

1. 打开 `升级任务`
2. 点击 `新建任务`
3. 选择产品
4. 选择目标固件
5. 选择任务类型
6. 如需灰度升级，再填写灰度比例
7. 提交

说明：

- `目标版本` 会根据目标固件自动带出
- 不需要手工输入版本号

### 取消任务

- 当任务状态为 `待执行` 或 `执行中` 时，可点击 `取消`

## 变化说明

- 不再单独使用旧的 `固件管理` 页面
- 不再手填 `文件 URL / 文件大小 / MD5`
- 不再手填设备固件绑定的版本号
