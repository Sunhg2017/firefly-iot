# 设备注册阶段维护设备标识使用说明

## 1. 适用角色

- 租户管理员
- 设备管理员
- 需要联调一型一密接入的研发或测试人员

## 2. 支持的入口

现在可以在以下场景注册设备时直接维护设备标识：

- 设备管理 → 新建设备
- 设备管理 → 批量导入
- 产品列表 → 设备接入 → 动态注册
- 设备模拟器 → HTTP / MQTT 一型一密自动注册

支持的常见标识类型：

- `IMEI`
- `ICCID`
- `MAC`
- `SERIAL`

## 3. 手动新建设备

### 操作步骤

1. 进入“设备管理”
2. 点击“新建设备”
3. 选择所属产品并填写设备名称
4. 在“设备标识”区域点击“新增设备标识”
5. 选择标识类型，填写标识值
6. 如需指定主标识，打开“主标识”开关
7. 保存并创建

### 使用提示

- 如果只填写了一条标识且没有手动指定主标识，系统会默认将第一条标识作为主标识。
- 设备名称与设备标识不是同一个字段，建议分别填写真实值。

## 4. 批量导入设备

### 4.1 支持的文件列

必填基础列：

- `deviceName`

标识列支持两种写法。

通用写法：

- `locatorType`
- `locatorValue`
- `primaryLocator`

快捷写法：

- `imei`
- `iccid`
- `mac`
- `serial`
- `sn`

### 4.2 导入步骤

1. 进入“设备管理”
2. 点击“批量导入”
3. 选择一机一密产品
4. 上传 Excel 或 CSV 文件
5. 等待异步任务完成
6. 到设备列表抽样检查导入结果

### 4.3 示例

#### 示例一：通用列

```csv
deviceName,nickname,locatorType,locatorValue,primaryLocator
dev-001,仓库温湿度计,IMEI,860001234567890,true
dev-002,冷库探头,ICCID,8986001234567890123,true
```

#### 示例二：快捷列

```csv
deviceName,nickname,imei,mac,serial
dev-003,网关A,860001234567891,AA:BB:CC:DD:EE:FF,SN-0003
```

### 4.4 注意事项

- `locatorType` 和 `locatorValue` 必须成对出现，不能只填一列。
- 同一产品下，相同标识值不能重复导入到多台设备。

## 5. 动态注册时维护标识

### 操作步骤

1. 进入“产品列表”
2. 找到一型一密产品，点击“设备接入”
3. 切到“动态注册”
4. 填写设备名称
5. 按需新增设备标识
6. 点击“执行动态注册”
7. 使用返回的 `deviceSecret` 继续接入设备

### 请求体示例

```json
{
  "productKey": "demo_product",
  "productSecret": "ps_xxx",
  "deviceName": "SN20240301001",
  "locators": [
    {
      "locatorType": "IMEI",
      "locatorValue": "860001234567890",
      "primaryLocator": true
    }
  ]
}
```

## 6. 模拟器自动注册时维护标识

### 操作步骤

1. 打开设备模拟器
2. 新增 HTTP 或 MQTT 模拟设备
3. 认证方式选择“一型一密”
4. 填写 ProductSecret
5. 在“设备标识”区域新增标识
6. 保存后连接设备

连接时，模拟器会先自动发起动态注册，并把设备标识一起提交给平台。

## 7. 常见问题

### 7.1 为什么注册成功了，但标识没有保存？

先确认：

- 本次入口是否真的填写了设备标识
- 标识值是否与同产品下其他设备冲突
- 是否使用了旧版前端或旧版模拟器

### 7.2 可以不填主标识吗？

可以。

如果本次只维护了一条标识，系统会默认把第一条标识作为主标识；如果维护多条标识，建议手动明确主标识，便于后续识别。
