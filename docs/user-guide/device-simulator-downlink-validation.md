# 设备模拟器下行验证使用说明

> 适用角色: 联调开发、测试人员、实施人员
> 更新时间: 2026-03-21

## 1. 这次改了什么

设备模拟器现在可以更直接地验证平台下发给 MQTT 设备的消息：

- 底部日志区会保留完整 payload，不再只显示前 200 个字符。
- MQTT 设备连接后会自动订阅三类常用下行主题：
  - 属性设置
  - 服务调用
  - 通用下行
- 日志会直接标注消息类型，方便快速判断收到的是哪一种下行。

## 2. 怎么验证平台下行消息

### 2.1 先连接 MQTT 模拟设备

1. 在设备模拟器里选择一台 MQTT 设备。
2. 点击 `连接`。
3. 连接成功后，模拟器会自动准备好默认下行订阅，不需要再手工输入 topic。

### 2.2 在平台侧发送消息

到设备消息工作台分别发送你要验证的消息：

- 属性设置
- 服务调用
- 通用下行

## 3. 去哪里看结果

### 3.1 底部全局日志

适合快速确认：

- 收到时间
- 消息类型
- 完整 topic
- 完整 payload

长 JSON 会自动换行显示，可以直接滚动查看。

### 3.2 MQTT 消息日志弹窗

如果你想按发布/接收方向筛选，或者复制单条 payload，可以打开设备控制区里的 `消息日志`。

## 4. 当前默认订阅的主题

连接 MQTT 设备后，模拟器会自动订阅：

- `/sys/{productKey}/{deviceName}/thing/property/set`
- `/sys/{productKey}/{deviceName}/thing/service/+`
- `/sys/{productKey}/{deviceName}/thing/downstream`

这三类主题分别对应平台当前常用的：

- 属性设置
- 服务调用
- 通用下行消息

## 5. 适合怎么用

- 想确认平台到底有没有把消息发到设备侧：看底部日志。
- 想核对长 JSON 有没有字段缺失：看底部日志或 MQTT 消息日志弹窗。
- 想保留联调证据：直接用日志导出按钮导出原文。

## 6. 注意事项

- 本次优化重点是“收到并看清楚”，不会自动替你回发设备应答。
- 如果你需要验证平台对 `property/set/reply` 或 `service/reply` 的处理，仍然要由设备侧自行发送应答报文。
- 如果连接后仍然收不到属性设置或通用下行，优先确认当前运行的是最新模拟器版本。
