# 设备模拟器统一下行验证使用说明

> 适用角色: 联调开发、测试、实施
> 更新时间: 2026-03-21

## 1. 能做什么

设备模拟器现在可以直接验证以下协议的工作台下行结果：

- MQTT
- WebSocket
- TCP
- UDP
- LoRaWAN

你可以在设备消息工作台发送属性设置、服务调用或通用下行，然后在模拟器里直接查看完整 payload。

## 2. 连接前准备

### 2.1 WebSocket / TCP / UDP

请先在模拟器里补齐：

- `ProductKey`
- `DeviceName`
- 可选 `定位器`

模拟器会自动用这些业务标识完成绑定，不需要手工输入数据库主键。

### 2.2 LoRaWAN

请确认：

- 模拟器里的 `DevEUI` 正确。
- 平台设备已配置可解析到该 `DevEUI` 的定位器。

## 3. 如何验证

### 3.1 WebSocket

1. 在模拟器中连接 WebSocket 设备。
2. 打开设备消息工作台，发送下行。
3. 在 WebSocket 消息面板查看 `接收` 记录。

### 3.2 TCP / UDP

1. 在模拟器中连接 TCP 或 UDP 设备。
2. 连接成功后，模拟器会自动发送绑定报文，不需要额外操作。
3. 在工作台发送下行。
4. 在 TCP/UDP 消息面板查看 `接收` 记录。

### 3.3 LoRaWAN

1. 在模拟器中将 LoRaWAN 设备置为在线。
2. 在工作台发送下行。
3. 在 LoRaWAN 面板查看 `下行` 记录；面板会自动轮询最新结果。

## 4. 去哪里看结果

每个协议面板都提供独立消息列表，特点如下：

- 发送、接收方向分开展示。
- 长 JSON 或长文本不再省略，支持完整换行查看。
- TCP、UDP、WebSocket、LoRaWAN 都能看到完整 payload。

## 5. 典型验证场景

### 场景 1: 验证属性设置

1. 选择设备。
2. 建立在线连接。
3. 在工作台发送属性设置。
4. 在对应协议面板确认收到完整下行内容。

### 场景 2: 验证服务调用

1. 保持设备在线。
2. 在工作台发送服务调用。
3. 在对应协议面板确认消息内容、参数和值都完整。

### 场景 3: 验证 LoRaWAN 下行

1. 确认 `DevEUI` 与平台映射正确。
2. 在工作台发送下行。
3. 等待面板自动刷新出新的 `下行` 记录。

## 6. 注意事项

- 本次能力解决的是“平台是否成功把消息下发到协议侧并被模拟器看到”。
- 模拟器不会自动替你回发 `property/set/reply` 或 `service/reply`。
- 如果 TCP/UDP 连上后仍收不到下行，优先检查 `ProductKey / DeviceName / 定位器` 是否与平台设备一致。
- 如果 LoRaWAN 收不到下行，优先检查平台里的历史定位器数据是否干净、是否仍指向旧设备。
