# 设备分组动态成员使用说明

## 适用角色

- 租户设备管理员
- 运维人员

## 动态分组能做什么

动态分组会根据设备当前状态自动维护成员，不需要人工逐台加设备。  
适合这些场景：

- 某个产品下的全部设备
- 设备名称带固定前缀的一组网关
- 当前在线的设备
- 带某个标签的设备

## 如何创建动态分组

1. 打开“设备分组”
2. 点击“新建分组”
3. 在第一步选择“动态分组”
4. 继续到“匹配规则”
5. 选择触发方式：
   - 全部条件满足
   - 满足任意条件
6. 逐条添加匹配条件
7. 在“预览确认”中检查规则 JSON
8. 保存

## 支持配置哪些条件

- 产品
- 设备名称
- 设备别名
- 设备状态
- 在线状态
- 设备标签

## 使用示例

## 示例 1：所有在线网关

- 条件 1：产品属于 `网关产品`
- 条件 2：在线状态等于 `在线`
- 触发方式：全部条件满足

## 示例 2：华北区域设备

- 条件 1：包含标签 `region: north`
- 触发方式：全部条件满足

## 示例 3：名称前缀为 `gw-` 的设备

- 条件 1：设备名称前缀匹配 `gw-`
- 触发方式：全部条件满足

## 动态分组和静态分组的区别

- 静态分组：手动加入或移除设备
- 动态分组：系统根据规则自动维护成员

注意：

- 动态分组内的设备不能手动移除
- 如果要人工维护成员，请改用静态分组

## 什么时候会自动更新

以下情况会自动重算动态分组：

- 设备标签变化
- 设备被启用或禁用
- 设备上线或离线
- 动态分组规则被修改

## 使用建议

- 先用“产品 + 标签”做大类分组，规则更稳定
- 名称匹配适合过渡期，长期建议还是用标签或产品做规则
- 动态分组适合状态视图，静态分组适合组织结构视图
