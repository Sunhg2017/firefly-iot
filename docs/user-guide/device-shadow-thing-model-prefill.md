# 设备影子期望属性自动填充使用说明

## 适用角色

- 设备运维
- 产品调试人员
- 设备接入实施人员

## 功能说明

在设备影子页面编辑 `desired` 时，系统会先根据设备所属产品的物模型，自动填充一份期望属性初始值草稿，再叠加当前已经保存的 `desired`。

这意味着：

- 新设备首次维护影子时，不需要从空 JSON 手工起草
- 已经保存过的期望值不会被默认值覆盖
- 系统内置属性不会出现在可编辑草稿里
- 历史物模型中即便自定义属性保存为只读口径 `accessMode = r`，也会先带入草稿，方便你从完整模型开始调整

## 操作步骤

1. 进入“设备管理”中的设备影子抽屉，或进入“设备影子”独立页面。
2. 选择目标设备并打开影子详情。
3. 在 `Desired / 期望属性` 卡片中点击“编辑”。
4. 系统会读取该设备所属产品的最新物模型，并自动生成一份 JSON 草稿。
5. 按实际需要调整属性值后点击“保存”。

## 使用示例

如果产品物模型定义了以下可写属性：

```json
{
  "properties": [
    { "identifier": "power", "dataType": { "type": "bool" } },
    { "identifier": "temperature", "dataType": { "type": "int", "min": 0 } }
  ]
}
```

点击编辑后，若当前设备还没有 `desired`，编辑器会先出现类似内容：

```json
{
  "power": false,
  "temperature": 0
}
```

如果当前设备已经保存了：

```json
{
  "power": true
}
```

则编辑时会自动补齐为：

```json
{
  "power": true,
  "temperature": 0
}
```

## 注意事项

- 自动填充只发生在点击“编辑”进入编辑态时。
- 查看态显示的仍然是服务端当前真实保存的影子内容。
- 如果产品没有维护任何自定义物模型属性，编辑器仍会按当前 `desired` 内容打开。
- 将属性值设为 `null` 后保存，可用于删除对应期望属性。
