# 设备模拟器工作台改版使用说明

> 适用角色：联调开发、测试人员、实施人员
> 更新时间：2026-03-22

## 1. 新版界面怎么看

新版模拟器工作台按四个区域划分：

1. 顶部总览：看全局设备数量、在线情况和快捷键提示。
2. 左侧设备列表：找设备、筛选设备、创建或导入导出设备。
3. 中间主工作区：看当前设备摘要、发数据、做协议调试。
4. 右侧辅助区：看运行日志，打开压测、场景编排、数据模板工具。

核心变化是：当前设备的主操作全部保持在中间区域，日志和低频工具不再挤占首屏。

## 2. 设备列表怎么用

左侧设备列表现在重点保留这几类信息：

- 设备名称
- 协议
- 在线 / 离线 / 异常状态
- 关键标识，如 `ProductKey / DeviceName`
- 已发送次数、错误次数、自动上报状态

常用操作：

- 点击 `新建` 创建模拟设备
- 点击 `导入` / `导出` 批量处理配置
- 通过搜索框按名称、`ProductKey`、`DeviceName` 检索
- 通过协议和状态筛选列表
- 点击 `批量连接` / `批量断开` 做统一操作

## 3. 当前设备工作区怎么看

选中设备后，中间区域会先显示摘要头图：

- 设备名称
- 协议
- 当前状态
- 关键标识摘要
- 发送次数、错误次数、自动上报状态、心跳周期或认证状态

摘要头图下面的主要区块有：

- `接入概览`：查看当前连接真正需要的参数
- `物模型 OpenAPI`：配置物模型同步所需的 AppKey 信息
- `数据上报`：这是主工作区，用于属性、事件、OTA、自定义 JSON、物模型模拟
- `协议专属工具`：查看 HTTP 请求历史、MQTT 订阅消息、CoAP 影子等

## 4. 日志和工具放到哪里了

### 4.1 运行日志

运行日志移动到了右侧独立卡片中。

你可以：

- 切换自动跟随
- 切换只看当前设备日志
- 导出日志
- 清空日志

### 4.2 快捷工具

右侧 `快捷工具` 卡片集中放了三个入口：

- `压力测试`
- `场景编排`
- `数据模板`

现在入口带有文字说明，不需要再根据图标猜功能。

## 5. 常见问题

### 5.1 为什么看不到底部状态条了

因为新版已经把原底部状态条收口到顶部总览，避免底部占空间。

### 5.2 为什么日志不在主区底部了

因为日志属于辅助信息。移到右侧后，当前设备的主操作区能保留更完整的可视高度。

### 5.3 为什么工具按钮位置变了

压测、场景编排、数据模板属于低频但重要的能力。新版把它们统一放进右侧工具区，避免继续沉在左侧底部。

### 5.4 为什么黑色输入框不见了

- 新版工作台已经统一收口为浅色主题。
- 输入框、按钮、告警等基础控件现在会跟随整张工作台的浅色风格，不再出现大块黑色控件。
- 如果你本地仍看到旧样式，优先重启模拟器并确认已加载最新前端资源。

### 5.5 为什么界面里的说明变少了

- 界面上的说明现在只保留对当前操作有帮助的提示。
- 例如会提示你先连接设备、补齐物模型签名参数、或进入协议工具查看调试结果。
- 改版背景、布局解释和设计说明已经转移到文档，不再占用主操作区空间。

### 5.6 为什么新建设备抽屉看起来和以前不一样了

- 新建设备抽屉已经跟随整个模拟器切到浅色工作台风格，头部、步骤条、摘要卡片和底部操作区都会更接近主界面的视觉语言。
- 抽屉里的说明现在只保留对操作有帮助的提示，例如先选协议、先选认证方式、填入 AppKey 后可直接使用物模型选择属性或事件。
- 如果你本地仍看到旧的深色抽屉样式，优先重启模拟器并确认已加载最新前端资源。
