# 前端根路由首页跳转修复使用说明
## 1. 适用角色

- 系统运维用户
- 租户管理员
- 租户业务用户

## 2. 现在访问首页会发生什么

访问系统根路径 `/` 时，系统会自动把你带到“当前账号真正有权限进入的首页”：

- 如果你有多个菜单权限，会进入系统按菜单顺序解析出的第一个可访问页面。
- 如果你没有仪表盘权限，但有别的业务页面权限，不会再被错误送到 `/403`。
- 如果你确实没有任何可访问菜单，系统仍会显示 `/403`，这表示当前账号还没有被授予可用页面。

## 3. 常见场景

### 3.1 直接打开系统域名

以前：某些账号会因为默认被送到 `/dashboard`，随后又被拦到 `/403`。

现在：系统会直接进入该账号实际可访问的首页。

### 3.2 登录成功后跳转

登录成功后，页面会和直接访问 `/` 使用同一套首页解析规则，因此两次看到的首页应当一致。

### 3.3 在 403 页面返回首页

如果你访问了一个自己无权进入的页面，可以在 403 页面点击“返回首页”，系统会带你回到自己的可访问首页。

### 3.4 输入了不存在的站内地址

如果你输入的是一个系统里并不存在的站内路径，系统会回到你的可访问首页，不会因为这个未知地址直接显示 403。

## 4. 如果仍然看到 403 怎么办

如果你在访问 `/` 时仍然看到 403，通常表示：

- 当前账号没有任何已授权菜单
- 或者当前账号缺少已授权菜单对应的权限点
- 或者浏览器里仍加载着旧前端资源，尚未更新到这次修复后的版本

这时请联系系统运维检查该账号的菜单授权和角色权限，而不是继续刷新页面。
