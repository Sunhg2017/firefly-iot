# 设备中心菜单使用说明

## 你会看到什么变化

设备中心左侧菜单现在有三处变化：

1. 原来的“产品管理”统一改成“产品接入”。
2. 原来平铺的一串协议入口，统一收进“协议接入”分组。
3. 原来在“运维工具”里的“视频监控”，现在统一放到“设备资产”。

这样做的目的是把“产品定义”和“协议接入能力”区分开，避免菜单又长又难找。

## 新的菜单结构

现在设备相关菜单会按下面这棵树展示：

- 设备接入
  - 产品接入
  - 协议解析
  - 协议接入
    - SNMP 接入
    - Modbus 接入
    - WebSocket 接入
    - TCP/UDP 接入
    - LoRaWAN 接入
- 设备资产
  - 设备管理
  - 设备拓扑
  - 视频监控
  - 设备分组
  - 设备标签
  - 地理围栏
  - 设备影子
  - 设备消息

## 怎么理解“产品接入”

“产品接入”不是单纯的产品台账，而是设备接入的入口定义页。你可以在这里：

- 创建产品
- 维护协议、认证方式、物模型
- 打开设备接入抽屉查看接入参数
- 对一型一密产品执行动态注册

如果你之前习惯找“产品管理”，现在请直接进入“产品接入”。

## 怎么使用“协议接入”

如果你要按某种具体协议做接入调试，请先展开“协议接入”分组，再进入对应页面：

- SNMP：适合做 SNMP 连通性、采集、Trap 监听
- Modbus：适合做 Modbus TCP 寄存器读写和采集
- WebSocket：适合做 WebSocket 连接和会话联调
- TCP/UDP：适合做原始 TCP/UDP 会话调试
- LoRaWAN：适合做 LoRaWAN 网络服务集成和回调

如果你要管理摄像头、预览视频流、做目录查询、截图或云台控制，请直接进入“设备资产 -> 视频监控”。

## 常见问题

### 为什么我登录后左边还是显示“产品管理”？

通常是因为浏览器仍在使用旧前端资源。刷新页面或重新登录后，应该会显示“产品接入”。

### 为什么视频监控还在运维工具里？

通常是因为当前环境还没执行到最新菜单迁移，或者浏览器还在使用旧前端资源。刷新并重新登录后仍不一致时，请联系运维检查菜单迁移是否已执行完成。

### 菜单管理里要怎么恢复新结构？

进入“菜单管理”，执行一次“初始化默认菜单”，系统会按当前最新的菜单结构重新生成，包括把“视频监控”放到“设备资产”下。
