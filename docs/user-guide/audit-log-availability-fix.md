# 审计日志加载失败修复使用说明

## 1. 修复后效果

- `系统运维空间 -> 安全审计 -> 审计日志` 页面可以正常打开
- 页面会正常显示审计记录列表
- 点击“详情”可以查看单条审计日志内容

## 2. 使用入口

入口：

- `系统运维空间 -> 安全审计 -> 审计日志`

页面支持：

- 按用户、描述、请求路径搜索
- 按模块筛选
- 按操作类型筛选
- 按成功/失败状态筛选
- 查看单条审计日志详情

## 3. 常见说明

- 如果页面还能进入但仍提示加载失败，通常是后端 `firefly-system` 还没有更新到修复版本
- 如果页面能打开但没有数据，说明当前时间段内还没有新的审计记录，或者当前账号没有对应可见范围
- 审计日志是否产生，取决于对应接口是否带有 `@Auditable`
