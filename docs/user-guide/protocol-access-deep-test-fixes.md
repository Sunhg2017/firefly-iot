# 协议接入模块深度测试修复使用说明

## 1. 本次影响的协议

- HTTP
- CoAP
- MQTT

TCP/UDP 本次主要做了回归验证，没有修改原有交互方式。

## 2. 你会看到哪些变化

### HTTP / CoAP

- 认证接口如果没有真正签发出 token，会直接返回失败
- 不会再出现“认证成功但 token 为空”的情况

### CoAP

- 使用无效 token 上报属性、事件或 OTA 进度时，会明确返回 `401`

### MQTT

- 外部 Broker 兼容模式下，设备只能访问自己的 `/sys/{productKey}/{deviceName}/#`
- 访问其他设备 Topic 会被拒绝
- 如果外部回调头里带了异常数值字段，平台会尽量继续解析，不会轻易把整条消息打挂

## 3. 使用建议

- 如果设备认证成功后仍无法继续上报，先检查 token 是否实际存在，不要只看返回是否为 200
- CoAP 设备出现 `401` 时，优先重新认证拿新 token
- MQTT 外部 Broker 接入时，确保设备身份与 Topic 身份严格一致

## 4. 常见说明

- 本次不是放宽，而是把原本被隐藏的接入错误显式暴露出来
- 如果新版本上线后看到更多协议侧失败日志，通常说明之前的异常只是被旧实现吞掉了
