# 通知中心使用说明

## 1. 适用角色

- 平台运维：维护平台默认通知渠道
- 租户管理员：查看通知记录、维护消息模板
- 告警与消息维护人员：排查通知发送结果

## 2. 功能入口

主要入口包括：

- 系统运维空间 / 通知渠道
  - 维护平台默认通知渠道
- 租户业务空间 / 通知记录
  - 查看发送记录
- 消息模板
  - 作为唯一模板维护入口

## 3. 通知渠道维护

## 3.1 新建渠道

1. 使用系统运维账号进入“通知渠道”
2. 点击“新建渠道”
3. 在抽屉中填写基础信息
4. 选择渠道类型
5. 根据渠道类型填写对应配置
6. 点击“创建渠道”

说明：

- 邮件渠道填写 SMTP 参数
- 短信/电话填写统一网关地址
- 企业微信/钉钉填写机器人地址
- 站内信只需维护默认来源、级别等参数

## 3.2 编辑渠道

1. 在渠道列表中点击“编辑”
2. 修改配置
3. 点击“保存修改”

## 3.3 测试渠道

1. 在渠道列表中点击“测试”
2. 系统会返回连接校验结果

提示：

- 邮件会测试 SMTP 连通性
- Webhook/短信/电话/机器人会校验地址是否有效
- 站内信会校验本地渠道可用性

## 3.4 启用或停用渠道

可直接通过列表中的开关控制。

停用后：

- 历史记录保留
- 新消息不会再通过该渠道发送

## 4. 发送记录查看

在租户业务空间的“通知记录”页面可以：

- 按渠道筛选
- 按状态筛选
- 查看发送详情
- 查看失败原因

常见状态：

- `PENDING`：待发送
- `SUCCESS`：发送成功
- `FAILED`：发送失败

## 5. 消息模板维护

说明：

- 系统只保留消息模板这一套模板体系
- 所有通知发送统一使用这里维护的消息模板
- 系统设置页不再提供第二套模板入口

## 5.1 新建模板

1. 打开“消息模板”
2. 点击“新建模板”
3. 填写模板编码、模板名称、渠道、模板类型
4. 填写消息主题和消息内容
5. 如有变量，维护变量定义
6. 点击“创建模板”

建议：

- 模板编码使用英文大写和下划线，例如 `ALARM_NOTIFY`
- 消息内容使用 `${变量名}` 占位

## 5.2 预览模板

1. 在模板列表中点击“预览”
2. 系统会使用示例变量渲染内容
3. 维护人员可快速检查模板语义和格式

## 5.3 启用与停用模板

列表开关可直接控制模板启用状态。

停用后：

- 模板不会再被选中发送
- 历史消息不受影响

## 6. 各渠道维护建议

### 邮件

- 主题尽量明确
- 内容适合长文本

### 短信

- 内容要简短
- 避免过长变量

### 电话

- 内容建议保持一句话内
- 接收号码需按网关要求维护

### 企业微信/钉钉

- 优先使用 Markdown 提升可读性
- 适合告警和值班通知

### Webhook

- 适合转发给其他系统
- 内容建议结构化

### 站内信

- 适合平台内待办、提醒、审批类消息
- 接收方应传入用户 ID 列表

## 7. 注意事项

- 平台默认通知渠道由系统运维空间统一维护；消息模板和通知记录仍按租户隔离。
- 如果发送失败，请先查看发送记录中的失败原因。
- 短信和电话渠道依赖外部网关，平台不直接集成某一家供应商 SDK。
- 复杂表单统一使用抽屉维护，如遇配置项较多，请优先在抽屉内完成维护，不建议继续用弹窗。
- 租户发送时会优先使用本租户专属渠道；若未配置对应渠道，系统会回落到平台默认通知渠道。
