# 前端复杂表单使用说明

## 1. 这次变化是什么

平台把“复杂表单”统一改成了抽屉式交互。也就是说，像规则配置、协议调试、批量注册、模板维护这类内容较多的页面，点击新建或编辑后，会从右侧拉出抽屉，而不是弹出居中窗口。

## 2. 为什么改成抽屉

抽屉更适合复杂配置，因为它可以：

1. 给表单提供更宽的可视空间。
2. 保留原列表和上下文，方便对照。
3. 让多分区、多步骤配置更清晰。
4. 避免长表单在弹窗里上下挤压。

## 3. 哪些场景会看到抽屉

通常你会在这些场景看到抽屉：

- 维护告警规则
- 编辑规则引擎
- 编辑协议解析器
- 注册 Modbus 或 SNMP 采集任务
- 批量注册设备
- 维护消息模板
- 编辑定时任务、产品、租户等复杂配置

## 4. 哪些场景仍然会看到弹窗

下面这些轻量场景仍可能保留弹窗：

- 删除前确认
- 简短创建
- 只读详情
- 小范围状态调整
- 预览类窗口

## 5. 使用建议

1. 复杂配置页面请优先在抽屉底部使用“保存”“执行”“取消”等固定按钮。
2. 如果需要对照列表信息，可直接保持抽屉打开，无需反复关闭。
3. 如果发现某个明显很长、很多字段的配置页仍然是弹窗，可以按仓库规则继续提改造需求。
