# 前端表单容器使用说明

## 1. 这次变化是什么

平台继续收敛复杂配置页面的交互方式：

- 简单表单继续使用弹窗。
- 复杂表单使用抽屉。
- 如果抽屉内容已经很长，会继续按模块拆成步骤流，而不是让用户在一个长抽屉里来回滚动。

## 2. 什么时候会看到步骤式抽屉

当页面包含多个配置模块、联动下拉较多、动态条件较多，或者整体内容已经明显过长时，会看到步骤式抽屉。

本次已落地到：

- 告警规则的新建
- 告警规则的编辑

## 3. 如何使用步骤式抽屉

### 3.1 基本操作

1. 先在当前步骤完成配置。
2. 点击“下一步”进入下一个模块。
3. 如需回看或修改，点击“上一步”或顶部已完成步骤返回。
4. 在最后一步确认摘要后再提交。

### 3.2 回退规则

- 已经配置过的步骤可以返回继续修改。
- 返回后不会丢失之前已填写的数据。
- 尚未进入的后续步骤不能直接跳转。

### 3.3 校验规则

- “下一步”只检查当前模块。
- 最终提交会检查整张表单。
- 如果已选择通知方式，则必须至少配置一类告警接收人。

## 4. 告警规则页面示例

告警规则抽屉当前分为以下步骤：

1. 基本信息
2. 适用范围
3. 告警等级块
4. 通知编排
5. 预览与状态

建议按页面默认顺序完成，系统会在页面中持续汇总等级块、条件数量、通知方式数量和规则主级别。

## 5. 使用建议

1. 优先使用下拉、联动选择和预置选项，不要把可以选择的字段改成手工输入。
2. 如果某个复杂配置页面仍然是一个很长的抽屉，可以继续按本规范发起优化。
3. 如果发现中文显示异常或步骤切换后数据丢失，应及时反馈为缺陷。
