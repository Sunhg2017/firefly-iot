# 设备模拟器流程一致性优化使用说明

## 1. 这次改了什么

设备模拟器这次主要收口了几类容易误导使用的行为：

- MQTT 下行消息、断连和异常日志现在统一记录，不会再分散到多个监听入口。
- 批量连接会覆盖更多真实支持的协议设备，不再只偏向 `HTTP / MQTT / CoAP`。
- 场景编排和压力测试的“通用发送”范围收口为 `HTTP / MQTT / CoAP`。
- MQTT Topic 模板里的 `{productKey}`、`{deviceName}` 会按设备业务身份替换。
- 设备发送成功数、失败数在批量并发时会稳定累加。

## 2. 批量连接现在怎么用

- 设备列表里的“批量连接”会自动选中全部离线、可恢复、非视频协议设备。
- 当前支持进入批量连接的协议包括：
  - `HTTP`
  - `MQTT`
  - `CoAP`
  - `SNMP`
  - `Modbus`
  - `WebSocket`
  - `TCP`
  - `UDP`
  - `LoRaWAN`
- `Video` 设备不会进入批量连接，需要单独操作。

## 3. 场景编排和压力测试有什么边界

- 场景编排里的“发送数据”步骤只对 `HTTP / MQTT / CoAP` 生效。
- 压力测试也只统计这三类协议的通用上报链路。
- 如果当前在线设备里混有 `SNMP / Modbus / WebSocket / TCP / UDP / LoRaWAN / Video`，这些设备会被明确跳过，而不是错误地走 HTTP 兜底发送。
- 这些协议仍应在各自专属控制面板里联调。

## 4. MQTT Topic 模板怎么写

推荐继续使用：

```text
/sys/{productKey}/{deviceName}/thing/property/post
```

现在占位符会直接替换为设备的 `productKey` 和 `deviceName`。如果只填写了 `mqttUsername`、`mqttClientId`，而没有补齐业务身份字段，Topic 可能无法按预期生成。

## 5. 日志和统计怎么看

- MQTT 下行消息会同时出现在全局日志和当前设备的 MQTT 消息列表里，但全局日志只保留一条记录。
- MQTT 断连或异常后，设备状态会同步更新，自动上报和心跳会自动停止。
- 批量发送、压力测试、SNMP 调试等场景下，成功数和失败数会稳定累加，适合持续观察设备质量。

## 6. 使用建议

- 做混合协议演练时，先用“批量连接”把可恢复协议设备拉起，再分别进入专属面板联调不共享上报链路的协议。
- 做 MQTT 场景编排或压测前，先确认设备已经补齐 `productKey` 和 `deviceName`，这样 Topic 模板才能正确展开。
