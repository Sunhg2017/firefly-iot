# 告警通知编排与接收组使用说明

## 1. 适用角色

- 告警规则维护人员
- 值班负责人
- 租户管理员

## 2. 告警接收组怎么用

### 2.1 进入页面

租户业务空间：

- `规则告警`
- `告警接收组`

### 2.2 新建接收组

1. 点击“新建接收组”
2. 填写接收组名称
3. 可选填写接收组说明
4. 在“组内成员”中多选用户
5. 点击保存

建议做法：

- 按值班班次建组，例如“白班值守组”
- 按项目或区域建组，例如“华东项目组”
- 接收组名称尽量直接表达职责

## 3. 告警规则中如何配置通知

### 3.1 进入告警规则

租户业务空间：

- `规则告警`
- `告警规则`

### 3.2 新建或编辑规则

在抽屉中按以下顺序完成：

1. 维护规则名称和说明
2. 选择项目、产品、设备范围
3. 按等级块维护触发条件
4. 在“通知编排”区域选择通知方式
5. 选择告警接收组
6. 如有需要，补充指定接收人

### 3.3 通知编排说明

#### 通知方式

- 只选择“方式”，不需要关心底层通道配置
- 邮件、短信、电话、企业微信、钉钉、站内信由系统运维统一维护
- Webhook 自动使用当前租户已启用的 Webhook 配置

#### 接收对象

- `告警接收组`：适合长期稳定的接收对象
- `指定接收人`：适合补充个别人
- 两者可以同时配置，系统会按规则合并使用

## 4. 推荐配置方式

### 4.1 稳定值班场景

- 通知方式：`站内信` + `邮件`
- 接收对象：选择“值班组”

### 4.2 外部系统联动场景

- 通知方式：`Webhook`
- 接收对象：选择负责该业务的接收组

### 4.3 临时专项保障场景

- 通知方式：`站内信` + `企业微信`
- 接收对象：选择接收组，再补充专项负责人

## 5. 页面预览怎么看

规则抽屉底部会显示两类摘要：

- `规则预览`：当前等级块和触发条件摘要
- `通知摘要`：当前通知方式与接收对象摘要

如果通知摘要显示“未配置通知推送”，说明该规则只会保留告警记录，不会主动通知。

## 6. 注意事项

- 如果页面中没有可选通知方式，请联系系统运维检查通知渠道配置。
- 如果页面中没有可选用户，请先确认当前租户已经创建用户。
- 删除接收组不会自动删除告警规则，但相关规则需要重新检查接收对象配置。
