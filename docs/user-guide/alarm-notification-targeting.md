# 告警通知编排与接收组使用说明

## 1. 适用角色

- 告警规则维护人员
- 值班负责人
- 租户管理员

## 2. 告警接收组怎么用

### 2.1 页面入口

租户业务空间：

1. `规则告警`
2. `告警接收组`

### 2.2 新建接收组

1. 点击“新建接收组”
2. 填写接收组名称
3. 按需填写接收组说明
4. 在“组内成员”中选择用户
5. 保存

适用场景：

1. 值班班次
2. 项目组
3. 区域负责人
4. 专项保障组

## 3. 告警规则里如何配置通知

### 3.1 基本流程

1. 进入 `规则告警 / 告警规则`
2. 新建或编辑规则
3. 在“通知编排”区域选择通知方式
4. 选择一个或多个接收组
5. 如需要，再补充指定接收人
6. 查看通知摘要后保存

### 3.2 通知方式说明

规则里只选择“方式”，不需要选择具体渠道实例。

系统会在触发时自动解析真正可用的通道：

1. 邮件
2. 短信
3. 电话
4. 企业微信
5. 钉钉
6. Webhook
7. 站内信

说明：

1. `WEBHOOK` 使用当前租户自己的启用 Webhook
2. 其他方式可使用租户通道，也可能落到平台默认通道

### 3.3 接收对象说明

可以同时配置两类对象：

1. 告警接收组
2. 指定接收人

系统会自动合并并去重。

## 4. 模板是怎么生效的

告警发送时：

1. 优先使用当前租户自己的告警模板
2. 如果租户未维护对应模板，则自动使用平台默认模板

这意味着你通常不需要为每个租户都重复创建一套基础告警模板。

## 5. 推荐配置方式

### 5.1 常规值班

1. 通知方式选择 `站内信 + 邮件`
2. 接收对象选择固定值班组

### 5.2 外部系统联动

1. 通知方式选择 `Webhook`
2. 接收对象选择负责该业务的接收组

### 5.3 临时专项保障

1. 通知方式选择 `站内信 + 企业微信`
2. 接收对象选择接收组
3. 再补充专项负责人

## 6. 常见问题

### 6.1 为什么没有可选通知方式

通常是因为当前租户或平台没有启用可用通道，请联系运维检查通知中心配置。

### 6.2 为什么保存时报接收对象不可用

通常是以下原因之一：

1. 接收组已删除
2. 用户已禁用
3. 邮件通知接收人没有邮箱
4. 短信或电话通知接收人没有手机号

### 6.3 为什么规则触发后只生成告警，不发通知

通常是因为：

1. 没有配置通知方式
2. 没有配置接收对象
3. 当前通知发送失败，可在通知记录中查看失败原因

## 7. 注意事项

1. 告警规则不会保存具体通道 ID，只保存通知方式类型。
2. 删除接收组后，相关告警规则需要重新检查并保存。
3. 如果业务需要独立模板，可以让租户单独维护；否则默认使用平台模板即可。
