# 设备分组使用说明

> 适用角色: 租户管理员、设备运营人员、运维人员
> 更新时间: 2026-03-13

## 1. 页面怎么看

设备分组页现在按三块来使用：

1. 页面总览
   - 快速查看当前租户的分组规模、静态分组数量、动态分组数量和静态成员总量。
2. 分组目录
   - 在左侧树里查找和选择分组。
   - 支持直接搜索分组名称。
3. 工作区
   - 上半部分是“分组画像”，看当前分组的类型、说明、创建时间和父分组。
   - 下半部分是“分组成员”，用于查看成员或维护静态分组成员。

## 2. 什么时候用静态分组，什么时候用动态分组

### 2.1 静态分组

适合这些场景：

- 按区域、楼层、项目、运维责任边界手工归类
- 需要人工决定哪些设备属于该组
- 希望手工加入或移除成员

### 2.2 动态分组

适合这些场景：

- 按规则自动归类设备
- 不希望人工逐台维护成员
- 需要统一按规则解释“为什么命中该组”

注意：动态分组不支持手工加入或移除成员。

## 3. 新建分组

### 3.1 新建顶级分组

1. 点击页面右上角 `新建分组`
2. 填写分组名称
3. 上级分组保持为“顶级分组”
4. 选择分组类型
5. 按需填写描述
6. 如果是动态分组，补充动态规则
7. 点击确定

### 3.2 新建子分组

1. 在左侧分组目录中选中目标父分组
2. 点击目录卡片右上角 `+`，或者在分组画像区域点击 `新建子分组`
3. 系统会自动带入当前父分组
4. 完成填写后提交

## 4. 编辑分组

1. 选择目标分组
2. 点击 `编辑分组`
3. 可修改：
   - 分组名称
   - 上级分组
   - 分组类型
   - 分组描述
   - 动态规则

限制说明：

- 不能把上级分组改成自己
- 不能把上级分组改成自己的子分组或更深层后代
- 静态分组切换为动态分组后，页面会按动态分组规则视角展示

## 5. 维护静态分组成员

1. 先选中一个静态分组
2. 在“分组成员”区域点击设备搜索框
3. 输入设备名称或昵称
4. 从下拉候选中选择设备
5. 点击 `加入设备`

加入后可以在成员表中查看：

- 设备名称
- 昵称
- 所属产品
- `productKey`
- 设备状态
- 在线状态
- 加入时间

如需移除，点击成员行右侧 `移除`。

## 6. 查看动态分组

1. 在左侧选择动态分组
2. 在“分组画像”中查看当前规则内容
3. 在“分组成员”区域查看只读提示

如果你看到“动态分组按规则自动归集设备，不支持手动添加或移除成员”，这是正常行为，不需要处理。

## 7. 使用建议

- 分组名称建议直接表达业务边界，例如“华东一区网关”“二号厂房温湿度设备”。
- 分组描述建议写清楚归类口径，方便多人协作维护。
- 静态分组优先用于人工责任边界，动态分组优先用于规则归类。
- 如果只是想快速定位分组，优先使用左侧搜索，不需要层层展开树目录。

## 8. 常见问题

### 8.1 为什么搜索不到设备

可能是因为：

- 当前不是静态分组
- 输入的关键字不匹配设备名称或昵称
- 设备已经在当前分组中

### 8.2 为什么不能给动态分组加设备

因为动态分组的成员来自规则，不来自手工维护。如果需要人工维护，请改用静态分组。

### 8.3 为什么父分组下拉里找不到某些分组

系统会自动排除当前分组和它的子孙节点，防止形成错误的树结构。
