# 角色数据范围使用说明

## 1. 适用角色

- 系统运维空间管理员
- 租户空间管理员
- 负责创建和维护角色的权限管理员

## 2. 新建角色时如何配置数据范围

进入“角色管理”后，在“基本信息”步骤选择数据范围：

- `全部数据`
  - 该角色可访问当前租户全部数据
- `项目范围`
  - 需要继续选择一个或多个项目
- `设备分组范围`
  - 需要继续选择一个或多个设备分组
- `仅本人`
  - 该角色仅可访问自己创建的数据
- `自定义范围`
  - 可同时选择项目和设备分组
  - 至少选择一个项目或设备分组

## 3. 范围会影响哪些页面

配置完成后，角色的数据范围会自动影响：

- 项目
- 产品
- 设备
- 规则引擎
- 告警规则
- 告警记录
- 固件
- OTA 任务
- 视频设备

## 4. 推荐使用方式

- 如果只希望某角色负责固定几个项目，使用 `项目范围`
- 如果只希望某角色负责某几个设备分组，使用 `设备分组范围`
- 如果既要负责指定项目，又要兼顾某些独立设备分组，使用 `自定义范围`
- 如果只是限制个人创建的数据，使用 `仅本人`

## 5. 注意事项

- `项目范围` 不能为空
- `设备分组范围` 不能为空
- `自定义范围` 不能项目和分组都为空
- 角色页看到的项目、设备分组选项都来自当前系统已有业务数据
- 旧的用户角色项目绑定口径已经停用，应以角色页当前配置为准

## 6. 常见问题

### 6.1 为什么保存角色时报“请选择至少一个项目或设备分组”

因为当前选择的数据范围需要具体边界，系统不再允许创建空范围角色。

### 6.2 为什么选了设备分组后，其他页面的数据也会变少

因为系统会根据分组里的成员设备，自动收口到对应的设备、产品、项目、告警、规则、OTA、视频数据。

### 6.3 修改角色范围后，用户需要重新登录吗

通常不需要；权限缓存会自动刷新。但如果浏览器页面长时间未刷新，建议重新进入相关页面确认最新结果。
