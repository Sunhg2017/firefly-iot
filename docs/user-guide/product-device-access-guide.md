# 产品设备接入使用说明

## 适用角色

- 产品经理
- 设备接入工程师
- 联调测试人员

## 功能入口

进入“产品接入”页面，在目标产品卡片或列表行点击“设备接入”。

## 页面说明

设备接入抽屉分为三个页签：

- 接入指引：告诉你当前产品应该先做什么。
- 协议参数：告诉你按当前协议具体怎么接。
- 动态注册：一型一密产品可直接在这里调试注册。

## 按认证方式使用

### 一机一密

适用场景：每台设备都有独立 `DeviceSecret`。

使用步骤：

1. 打开产品的“设备接入”。
2. 在“接入指引”页查看说明。
3. 点击“设备管理”，先创建或导入设备。
4. 获取设备自己的 `DeviceSecret`。
5. 回到“协议参数”页，按页面给出的 MQTT / HTTP / CoAP 口径接入。

注意事项：

- 一机一密不支持动态注册。
- 不同设备的 `DeviceSecret` 不能混用。

### 一型一密

适用场景：设备出厂时只带产品级凭证，需要运行时动态换取 `DeviceSecret`。

使用步骤：

1. 确认产品已发布。
2. 打开“设备接入”。
3. 在“接入指引”页查看并复制 `ProductSecret`。
4. 切到“动态注册”页，填写设备名称。
5. 执行动态注册，拿到 `DeviceSecret`。
6. 切到“协议参数”页，按页面说明继续接入。

注意事项：

- 产品未发布时，不能执行动态注册。
- 设备名称必须唯一，建议使用 SN、MAC 或设备唯一编号。
- `ProductSecret` 只用于动态注册，真正接入时最终使用的是 `DeviceSecret`。

## 协议说明

### MQTT

页面会给出以下参数：

- `ClientId = productKey.deviceName`
- `Username = deviceName&productKey`
- `Password = DeviceSecret`
- 属性 Topic
- 事件 Topic
- 服务订阅 Topic

### HTTP

页面会给出以下口径：

- 认证接口：`POST /api/v1/protocol/http/auth`
- Token 请求头：`X-Device-Token`
- 属性上报接口
- 事件上报接口
- 心跳接口

### CoAP

页面会给出以下口径：

- 认证接口：`POST /api/v1/protocol/coap/auth`
- Token 参数：`token=<token>`
- 属性上报接口
- 事件上报接口
- OTA 进度接口

### 其他协议

如果页面提示当前协议暂无内置示例，说明需要配合“协议解析”页面或设备模拟器完成联调。

## 动态注册结果怎么看

动态注册成功后，页面会展示：

- `productKey`
- `deviceName`
- `deviceSecret`

你可以直接复制这些值用于后续联调。

页面不会展示数据库主键，避免把内部 ID 当成接入参数误用。

## 常见问题

### 为什么我只能看到 ProductSecret，不能点动态注册？

因为产品还没有发布。一型一密产品只有在发布后才能正式动态注册。

### 为什么提示设备名称已存在？

同一产品下的设备名称必须唯一，请更换为新的设备唯一标识后重试。

### 为什么 HTTP 认证成功后还要带 Token？

因为 HTTP 接入采用“先认证换 Token，再带 Token 上报”的方式，后续属性、事件、心跳请求都必须带 `X-Device-Token`。
