# 工作空间菜单权限目录使用说明

## 1. 适用角色

- 系统运维管理员
- 平台权限管理员

## 2. 入口位置

登录系统运维空间后，进入：

`权限资源 -> 空间权限目录`

## 3. 页面能做什么

该页面用于查看当前系统中：

- 某个功能属于哪个工作空间
- 某个菜单路径对应哪些权限点
- 某个功能是否会进入角色授权目录

适合用于以下场景：

- 排查某菜单为什么在租户空间可见或不可见
- 核对角色页为什么能分配或不能分配某些权限
- 新功能上线后确认台账是否补齐

## 4. 常用查看方法

### 4.1 按空间查看

在“所属空间”中选择：

- `系统运维空间`
- `租户业务空间`

用于区分功能归属。

### 4.2 按关键字查看

可按以下关键字检索：

- 模块名称
- 菜单路径
- 权限编码
- 权限说明

## 5. 结果如何理解

一行记录表示：

- 某个空间中的一个功能模块
- 绑定了一个具体权限点

因此：

- 一个功能模块可能对应多行记录
- 如果一个功能在两个空间都存在，会分别出现两组记录

## 6. 维护规则

该页面是只读台账，不在页面上直接新增或编辑。

后续功能变更时，需要由研发通过 Flyway SQL 维护：

- 功能只属于一个空间，写一条记录
- 功能属于两个空间，写两条记录

## 7. 常见问答

### 7.1 为什么有的记录不出现在角色授权页

因为该记录只用于追踪菜单和空间归属，`role_catalog_visible` 被设为隐藏，不会作为独立模块显示在角色授权目录里。

### 7.2 为什么租户空间查不到某个功能

通常有两种原因：

- 该功能本来就只属于系统运维空间
- 当前租户没有被授权该菜单路径

### 7.3 新功能已经上线，为什么这里没有记录

说明发布缺了对应 SQL，需要补 Flyway 迁移，而不是继续靠代码硬编码兜底。
