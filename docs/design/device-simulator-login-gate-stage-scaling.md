# 设备模拟器登录门禁与主舞台等比例缩放设计

## 1. 背景

设备模拟器当前存在两个交互问题：

- 未登录时也会直接进入主工作台，用户先看到复杂主界面，再去找环境和登录入口，流程割裂。
- 主界面采用响应式三栏挤压，窗口缩小时会优先牺牲中间主工作区，出现“只剩左右两侧”的观感，不符合桌面应用预期。

本次改造将模拟器入口收口为“先登录、后进入工作台”，并将主页面改为固定主舞台的统一缩放模型。

## 2. 目标

- 未登录当前环境时，只展示登录门禁页，不进入主工作台。
- 登录成功后再进入主页面。
- 主页面保持固定舞台比例，窗口放大缩小时整体等比例缩放。
- 在单个 PC 屏幕内始终保持完整的三栏主框架，不再通过响应式折叠右侧区域。

## 3. 范围

涉及文件：

- `firefly-simulator/src/App.tsx`
- `firefly-simulator/src/components/SimulatorAccessGate.tsx`
- `firefly-simulator/src/components/index.ts`

不涉及：

- 平台登录接口变更
- 模拟器设备接入链路变更
- 数据库存储结构变更

## 4. 方案

### 4.1 登录门禁

- `App.tsx` 直接读取当前激活环境对应的 session。
- 当 `accessToken` 不存在时，渲染 `SimulatorAccessGate`。
- `SimulatorAccessGate` 负责：
  - 选择当前环境
  - 管理环境列表
  - 执行用户名密码登录
- 登录成功后写入 `workspaceStore.sessions[environmentId]`，主页面自动切换为工作台。

### 4.2 主工作台缩放

- 主页面不再使用断点式三栏折叠。
- 以固定舞台尺寸作为桌面端基准：
  - 宽度 `1760`
  - 高度 `960`
- 根据当前窗口可用尺寸计算统一 `scale`，使用 `transform: scale(...)` 对整套主舞台等比例缩放。
- 左侧设备栏、中间主工作区、右侧工具与日志栏始终保持固定比例，不再发生“右栏掉到下一行、中间区域被挤没”的情况。

### 4.3 副作用收口

- 主页面相关键盘快捷键、连接恢复、MQTT 事件监听都只在已登录状态下挂载。
- 用户从工作台退出登录或切换到未登录环境后，会立即回到登录门禁页。

## 5. 关键取舍

- 不再依赖响应式折叠来适配缩小窗口，改为桌面应用式固定舞台缩放，更符合用户对 PC 工具的预期。
- 登录门禁页与左侧面板都保留环境管理能力，保证未登录状态下也能切换和维护环境。
- 主工作台只在已登录时挂载，避免未登录状态下提前启动工作区副作用。

## 6. 风险与边界

- 在极小窗口下，主舞台仍会继续缩小以保证完整框架可见，文字会相应变小；该方案面向 PC 场景。
- 切换到未登录环境会立即返回登录页，这是预期行为，不再保留“未登录也能继续停留工作台”的旧交互。
