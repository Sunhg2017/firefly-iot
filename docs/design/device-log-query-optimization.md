# 设备日志查询优化设计说明
## 1. 背景

现有设备日志能力虽然已经具备查询、统计和清理接口，但存在几个直接影响使用和数据边界的问题：

- 前端只能手工输入 `deviceId` 查询，要求用户记内部主键，不符合设备域优先使用业务唯一键和选择型交互的规则。
- 日志列表主视角只有 `deviceId`，无法直接看出设备名称、别名和所属产品。
- 后端最近日志、级别统计和过期清理接口没有统一按当前租户收口，存在跨租户读取和清理风险。
- 设备端上报的 `reportedAt` 没有真正入库，导致日志时间轴可能被服务端时间覆盖。

因此本次对设备日志模块做一次收口优化，直接围绕当前实现补齐可读性、边界和验证能力，不继续保留旧查询口径。

## 2. 目标

- 前端查询条件收口为设备选择、产品选择、日志级别、关键字和时间范围。
- 页面主视角展示设备名称、别名、产品名称和 `productKey`，不再要求用户依赖数据库主键理解日志。
- 后端日志列表、最近日志、级别统计和清理操作统一按当前租户边界执行。
- 设备端上报时间保留原值；如果未传，则回退到服务端当前时间。
- 补齐服务层测试和模块交付文档，确保设备日志能力可回归、可运维。

## 3. 范围

本次改动包含：

- `firefly-device`
  - 设备日志返回模型补齐设备名称、别名、产品名称和 `productKey`
  - 修复 `reportedAt` 丢失
  - 补租户过滤、级别归一化和分页/最近日志可读字段补齐
  - 新增 `DeviceLogServiceTest`
- `firefly-web`
  - 设备日志页改为选择式筛选
  - 新增产品筛选、重置、详情查看
  - 列表展示切换为业务可读口径
- `docs`
  - 新增设计、运维、使用三类文档

本次不包含：

- 新增设备日志菜单、权限点或空间归属变更
- 新增日志导入导出能力
- 改造设备日志存储模型或拆分冷热表

## 4. 后端方案

## 4.1 返回模型补齐

`DeviceLogVO` 新增以下字段：

- `deviceName`
- `nickname`
- `productName`
- `productKey`

`DeviceLogService` 在查询当前页日志后，按 `deviceId` 批量调用 `DeviceMapper.selectBasicByIdsIgnoreTenant(...)` 补齐设备基础信息，再转换成前端可直接展示的 `DeviceLogVO`。

这样可以避免：

- 前端为了显示设备信息逐条查详情
- 页面继续把 `deviceId` 当作主视角

## 4.2 时间与级别口径

- `record(...)` 保留调用方传入的 `reportedAt`
- 未传 `reportedAt` 时，回退到 `LocalDateTime.now()`
- 日志级别统一做大写归一化；空值回退为 `INFO`

## 4.3 租户边界

以下接口统一补 `tenantId` 过滤：

- 分页查询 `listLogs`
- 最近日志 `getRecentLogs`
- 按级别统计 `countByLevel`
- 清理过期日志 `cleanExpiredLogs`

这样可以避免租户空间的设备日志能力误读或误删其他租户数据。

## 4.4 参数收口

- 最近日志 `limit` 小于等于 0 时回退为 100，最大限制 500
- 清理保留天数小于等于 0 时回退为 30
- 创建设备日志时补充 `content` 非空、`level/traceId/ip` 长度约束

## 5. 前端方案

## 5.1 查询区

设备日志页查询区改为：

- 设备选择：远程搜索设备名称/别名，内部仍传 `deviceId`
- 产品选择：按产品名称和 `productKey` 选择
- 日志级别选择
- 日志内容关键字
- 时间范围
- 查询 / 重置

不再保留手工输入 `deviceId` 的交互。

## 5.2 表格展示

列表按当前使用场景展示：

- 设备：别名 + 设备名称
- 产品：产品名称 + `productKey`
- 级别
- 模块
- 内容
- Trace ID
- IP
- 上报时间

## 5.3 详情查看

新增日志详情弹窗，用于查看：

- 设备和产品信息
- 级别、模块、Trace ID
- IP、上报时间、入库时间
- 完整日志内容

## 6. 风险与约束

- 设备基础信息补齐依赖 `devices` 与 `products` 当前可见数据；如果日志关联的设备已被删除，页面会显示“设备信息缺失”。
- 本次没有改动菜单、权限和 Flyway 菜单台账，因为设备日志入口和权限编码未变化。
- 本次只收口租户边界，不新增额外兼容分支；如历史数据存在跨租户脏日志，需要运维侧单独清理。

## 7. 验证点

- `DeviceLogServiceTest` 覆盖：
  - 上报时间保留
  - 日志级别归一化
  - 日志可读字段补齐
  - 最近日志 / 统计 / 清理的租户过滤
- `firefly-web` 执行 `npm run build` 成功
- 页面查询不再要求手填 `deviceId`
- 列表和详情可直接看到设备名称、别名、产品名称和 `productKey`
