# 前端页面标签与保活设计说明
> 模块: `firefly-web`
> 日期: 2026-03-16
> 状态: Done

## 1. 背景

控制台页面在路由切换后会直接卸载原页面组件，导致用户返回页面时，表单输入、筛选条件和局部编辑状态全部丢失。对于设备、产品、协议、菜单配置等需要频繁来回切换的页面，这会明显打断操作流程。

同时，主布局原先缺少顶部“已打开页面”承载区，用户只能依赖左侧菜单重新进入页面，无法直观看到当前工作上下文，也不方便快速返回刚才编辑中的页面。

## 2. 目标

- 在主布局内容区顶部提供已打开页面标签，支持用户快速切换已访问页面。
- 页面切出后继续保留组件实例，返回标签页时恢复之前的输入与本地状态。
- 保持现有菜单权限、空间隔离和面包屑逻辑不变。
- 全局头部左侧信息与右侧动作区保持同一行展示，右侧将通知、异步任务、用户名聚合为一组。
- 租户空间全局头部右侧新增告警入口，显示待处理告警数量，并支持查看最近告警。
- 用户名区域采用轻量行内样式，不再使用额外胶囊背景。
- 用户头像下拉菜单提供“修改密码”入口，登录用户可在全局头部直接修改自己的密码。

## 3. 方案

### 3.1 页面标签管理

- 在 `BasicLayout` 中维护 `openedTabs`，按访问顺序记录当前会话中已打开的业务页面路径。
- 标签仅纳入已注册且允许访问的业务路由，排除 `/403` 等异常页。
- 标签标题优先取当前菜单树中的自定义菜单名称，保证租户空间与系统运维空间修改菜单名称后，顶部标签同步生效。
- 如果菜单树尚未加载完成，则回退到 `routes.tsx` 中维护的默认标题和图标。
- 关闭当前标签时，优先切换到相邻标签；若已无其他标签，则回到当前空间首页。

### 3.2 页面保活

- 在布局层通过 `useOutlet()` 获取当前匹配页面。
- 首次进入可保活页面时，将对应 `ReactNode` 缓存到 `cachedPages[pathname]`。
- 内容区持续渲染所有缓存页面，仅通过 `display: block / none` 切换当前可见页面，避免组件卸载。
- 因页面组件实例未卸载，其内部 `useState`、表单实例、抽屉输入和筛选状态可继续保留。

### 3.3 权限与空间收口

- 只有通过 `isWorkspacePathAllowed(...)` 校验的页面才允许进入标签集合与缓存集合。
- 当用户权限、授权菜单或所在空间发生变化时，自动裁剪无权访问的标签并释放对应缓存页面。
- `workspaceRoutes.ts` 按路径维护可归属空间集合，解决 `/user`、`/role`、`/menu-customization` 等共享路由被误判为跨空间访问的问题。

### 3.4 顶部布局收口

- 主布局头部使用左右双区结构，`justify-content: space-between` 保持页面标题信息与操作区稳定分布。
- 左侧区域承载折叠按钮、当前空间标识和面包屑，并通过 `min-width: 0` 与面包屑截断控制保证始终在同一行内收缩。
- 右侧区域固定为通知、告警、异步任务、用户名四项的同行动作组，避免被拆到第二行或下沉到页面内容区域。
- 用户名触发器改为头像加用户名的纯行内结构，仅保留 hover 反馈，不再渲染胶囊、底色或独立卡片式背景。
- 用户名下拉菜单扩展为“修改密码 + 退出登录”，其中修改密码使用轻量弹窗承载当前密码、新密码、确认新密码三项表单。
- 告警入口仅在租户空间且当前用户具备 `/alarm-records` 访问权限时显示，待处理告警数复用仪表盘 `alarmPending` 指标，打开后展示最近告警摘要。

## 4. 影响范围

- `firefly-web/src/layouts/BasicLayout.tsx`
- `firefly-web/src/config/routes.tsx`
- `firefly-web/src/config/workspaceRoutes.ts`
- `firefly-web/src/config/iconMap.tsx`
- `firefly-web/src/styles/global.css`

## 5. 设计取舍

- 未引入额外 keep-alive 组件库，避免与现有 React Router、权限校验和懒加载链路叠加新的不确定性。
- 保活范围限定为“当前登录会话内、已打开且未关闭的页面”；浏览器整页刷新后，标签和页面实例会重新初始化。
- 页面标签改为布局层自绘按钮组，不依赖 `antd Tabs` 内部激活态切换，规避新页面首次打开时被误切回首个标签的问题。
- 头部动作区保留在全局 Header 中统一承载，不再拆入页面级标题区域，避免不同页面重复实现和样式漂移。
- `ScheduleOutlined` 图标映射已补齐，系统运维空间中的“定时任务”菜单可正常显示图标。
