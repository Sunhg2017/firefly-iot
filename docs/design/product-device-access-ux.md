# 产品设备接入设计

## 1. 目标

- 产品页统一提供 `设备接入` 入口。
- 摄像头产品不再跳独立 `/video` 页面，而是跳到 `/device` 的视频设备视图。
- 产品上下文、协议、视频新增动作在一次跳转内联动完成。

## 2. 协议口径

### 2.1 普通 IoT 产品

- 一机一密：先到设备管理创建设备，再按协议接入
- 一型一密：先动态注册，再使用返回的 `deviceSecret` 接入

### 2.2 摄像头产品

- 页面统一显示为 `视频协议接入`
- 不展示 `ProductSecret`
- 不开放动态注册
- 后续入口统一跳到 `/device?assetType=video`

## 3. 产品到视频设备视图联动

摄像头产品点击 `视频接入` 后，前端带以下上下文跳转：

- `assetType=video`
- `autoCreate=1`
- `productId`
- `productKey`
- `productName`
- `protocol`

视频设备视图收到参数后：

1. 切换到视频设备页签
2. 自动打开新增抽屉
3. 预填产品
4. 锁定协议

## 4. 关键取舍

- 不再保留独立视频页和双轨跳转。
- 不再用页面说明去解释设计背景，只保留必要操作引导。
- 摄像头产品继续走 `产品 -> 协议/物模型 -> 设备资产` 主链路，视频只是设备资产扩展能力。
