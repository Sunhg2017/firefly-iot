# 设备模拟器日志跟随方向修复设计说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 背景

设备模拟器日志数据内部仍采用头插存储，`addLog(...)` 会把新日志插入数组头部。但用户希望日志面板按常规阅读顺序展示，即旧日志在上、最新日志在下，自动滚动也应跟随到底部。

## 2. 问题表现

- 日志列表与用户预期相反，最新日志出现在顶部。
- 自动滚动方向与“最新日志应在底部”的阅读习惯不一致。
- 导出日志时也会沿用倒序，排查时不直观。

## 3. 修复方案

- 保持现有日志存储结构不变，避免影响其他调用链路。
- 日志面板展示层将日志反转为正序，即旧日志在上、最新日志在下。
- 自动跟随逻辑改为滚动到底部，确保新增日志始终落在可视区域。
- 导出日志时同步按展示顺序输出，方便联调和问题排查。

## 4. 影响范围

- `firefly-simulator/src/components/LogPanel.tsx`
