# 设备模拟器日志跟随方向修复设计说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 背景

设备模拟器日志面板采用“最新日志在最上面”的展示方式，`addLog(...)` 会把新日志插入数组头部。但日志面板的自动滚动仍按传统正序日志实现，新增日志时把容器滚动到最底部。

## 2. 问题表现

- 页面最上方显示的是最新日志。
- 自动刷新打开后，视图却总被拉到最下面。
- 用户看不到刚刚产生的新日志，需要手动回滚到顶部。

## 3. 修复方案

- 保持现有“新日志在最上面”的渲染顺序不变。
- 将自动跟随逻辑从“滚动到底部”调整为“固定到顶部”。
- 在代码中补充注释，明确日志列表是头插模型，避免后续再误按尾部跟随实现。

## 4. 影响范围

- `firefly-simulator/src/components/LogPanel.tsx`
