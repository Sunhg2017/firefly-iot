# 前端页面说明文案收口设计

## 背景

近期页面改版过程中，部分页头描述、说明卡片和提示文案混入了设计思路、风格表达和模块介绍。这类内容不会直接帮助用户完成当前操作，反而会抬高页面噪音，影响核心信息识别效率。

## 目标

- 页面说明只保留对当前操作直接有帮助的内容。
- 所有介绍型、宣传型、设计表达型文案从页面中移除。
- 将设计取舍、改版背景、实现思路统一沉淀到 `docs/`，不再放在页面里。

## 适用范围

- `PageHeader` 页头描述
- 页面说明卡片、提示卡片、引导卡片
- 接入页公共引导区
- 登录页、首页等入口型页面的说明文案

## 设计原则

1. 页面文案只回答“现在该怎么做”。
2. 页面文案可以保留前置条件、限制条件、结果影响和失败处理提示。
3. 页面文案不再承担模块介绍、风格解释和改版说明职责。
4. 设计思路、实现背景、能力边界说明统一下沉到文档。

## 本次收口内容

- 登录页改为登录步骤与登录须知，不再展示风格和模块介绍。
- 工作台首页改为处理顺序提示，不再描述视觉组织方式。
- AppKey、OpenAPI、产品、租户、菜单、权限等管理页改为“先做什么，再做什么”的操作描述。
- LoRaWAN、Modbus、SNMP、TCP/UDP、WebSocket 接入页的公共引导区改为操作入口提示，并移除能力宣传标签。
- 告警相关页头描述改为配置、处理、编排步骤说明。
- 仓库级规则新增“页面文案强制规则”，作为后续开发约束。

## 影响说明

- 本次仅调整前端展示文案和公共引导区表达，不改业务接口和权限行为。
- 页面说明减少后，操作入口与主功能区的视觉优先级更清晰。

## 风险与取舍

- 风险：过度删减可能导致必要限制提示丢失。
- 取舍：保留与当前操作强相关的限制和结果提示，删除与操作无关的介绍型描述。
