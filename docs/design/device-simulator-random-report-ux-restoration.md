# 设备模拟器随机上报入口恢复设计说明
> 模块: firefly-simulator
> 日期: 2026-03-14
> 状态: Done

## 1. 背景

设备模拟器一直支持“随机模拟数据上报”：

- `store.ts` 中保留了默认模板
- 模板字段支持 `random_int`、`random_float`、`timestamp`、`enum`
- 自动上报也会基于模板重复生成 payload

但在控制面板改版后，这项能力被弱化成了一个普通的 `Payload Template` 下拉，并且默认不选模板，页面初始只显示自定义 JSON。用户看到的效果就像“随机模拟上报没了”。

## 2. 目标

- 让随机模拟能力重新成为设备模拟器的默认上报方式
- 明确区分“随机模拟”和“自定义 JSON”两种模式
- 让用户能直观看到当前随机模板会生成什么样的样例

## 3. 方案

在 `DeviceControlPanel` 的数据上报区域增加显式交互：

- 增加 `payloadMode`
  - `template`
  - `custom`
- 默认优先使用 `template`
- 根据当前 `reportType` 自动选中第一条可用模板
- 当没有对应模板时，自动回退到 `custom`

同时补充可见性更强的 UI：

- “数据源”切换按钮
  - `随机模拟`
  - `自定义 JSON`
- “随机模板”下拉
- “当前已启用随机模拟数据”提示
- “随机样例预览”区域
- “换一组”按钮，用于刷新一份新的随机样例

## 4. 关键设计点

### 4.1 随机样例预览不参与真实发送

预览仅用于展示模板可能生成的数据，真正点击“发送”或自动上报时，仍会重新调用 `generatePayload(...)`，确保每次发送都是真随机数据。

### 4.2 不破坏原有自定义 JSON 流程

自定义 JSON 仍然保留，适用于：

- 精确回放固定 payload
- 调试边界值
- 联调特定异常报文

### 4.3 模板管理继续复用现有能力

没有新建平台接口，继续复用本地模板管理能力。若某种上报类型没有模板，页面会明确提示去“模板管理”补充。

## 5. 影响范围

- [DeviceControlPanel.tsx](/E:/codeRepo/service/firefly-iot/firefly-simulator/src/components/DeviceControlPanel.tsx)

## 6. 验证

- 切换到 `property/event/ota` 时，若有模板，默认进入“随机模拟”模式
- 点击“换一组”会刷新预览样例
- 点击“发送”时，设备仍正常上报
- 自动上报仍可用，并且每次会生成新的随机 payload
- 切换到“自定义 JSON”后，仍可按旧方式发送固定报文
