# Web 控制台模拟器风格统一改造设计

## 1. 背景

当前 `firefly-web` 控制台与设备模拟器存在明显风格割裂：

- Web 端仍保留深色侧边栏、毛玻璃头部和偏紫色主题，和模拟器当前的浅色实体卡片风格不一致。
- 登录页信息块过多，品牌说明、能力卡和提示卡并列堆叠，登录动作不够聚焦。
- 仪表盘首页存在一排高饱和彩色统计卡，信息密度高但主次不够清晰。
- 各业务页面虽然已经统一使用 `PageHeader`，但头部样式仍偏轻量，缺少明显的页面层级锚点。

本次改造目标是让 Web 控制台与当前模拟器收口到同一套视觉语言，形成统一的产品观感。

## 2. 目标

- Web 控制台整体对齐模拟器的浅色、实体、PC 优先的视觉风格。
- 去除深色侧栏和毛玻璃头部，收口为浅色布局骨架。
- 登录页改为简洁入口页，聚焦登录动作并去掉无关说明文字。
- 仪表盘首页改为“概览区 + 实体指标卡 + 明细面板”结构。
- 通用页面头部统一为带层级锚点的卡片式 `PageHeader`。

## 3. 范围

涉及文件：

- `firefly-web/src/main.tsx`
- `firefly-web/src/styles/global.css`
- `firefly-web/src/layouts/BasicLayout.tsx`
- `firefly-web/src/components/PageHeader.tsx`
- `firefly-web/src/pages/login/index.tsx`
- `firefly-web/src/pages/dashboard/DashboardPage.tsx`

不涉及：

- 菜单、权限、接口、路由定义变更
- 业务表单字段变更
- 数据结构和后端接口变更

## 4. 方案

### 4.1 全局主题令牌

- 主色从偏紫色收口为模拟器同系蓝色。
- 字体栈收口为中文优先的桌面字体组合。
- 全局圆角、边框色、布局底色、弹层阴影统一到浅色实体卡片体系。
- `Button`、`Card`、`Input`、`Select`、`Table`、`Menu`、`Modal`、`Drawer`、`Tabs` 使用统一组件 token。

### 4.2 基础布局

- 侧边栏改为浅色渐变底和浅边框，不再使用深色菜单体系。
- 头部去除 `backdropFilter` 毛玻璃效果，改为实体白色头部。
- 标签页、内容区壳层统一为浅色边框卡片样式。
- 用户信息区、顶部动作按钮、菜单选中态统一为浅蓝提示态。
- 顶部右侧快捷入口取消独立外框，默认只保留图标，悬停或展开时再显示文字并带轻微放大反馈。

### 4.3 页面共享头部

- `PageHeader` 改为卡片式头部容器。
- 标题左侧增加小型蓝色锚点，强化页面层级识别。
- 描述文案保留，但统一为次级浅色说明。
- 所有已引用 `PageHeader` 的业务页面自动继承新样式。

### 4.4 登录页

- 登录页改为简洁双栏入口布局。
- 左侧保留品牌信息，并补充一组圆形轨道式万物互联动效，不再展示能力卡、标签和独立提示卡。
- 右侧聚焦登录表单，取消独立表单卡片外框，仅保留登录方式切换、输入框和提交动作。
- 去掉无实际作用的附加控件，减少视觉噪音和误导性操作。
- 保留现有登录方式和登录成功后的首页跳转逻辑。
- 增加窄屏下的一栏布局兜底，避免固定双栏撑坏页面。

### 4.5 仪表盘

- 首屏改为“概览卡 + 核心指标卡 + 告警明细/分布面板”结构。
- 去掉高饱和渐变大卡，改用实体白底卡片和浅色状态底。
- 重点突出设备总量、在线率、待处理告警和规则启用状态。
- 保留现有数据接口，不引入新的统计逻辑。

## 5. 关键取舍

- 不做逐页单独重构，优先改主题、布局和共享头部，让绝大多数页面自动继承新风格。
- 登录页保留双栏，但左侧只承担品牌识别和圆形轨道式联网动效展示，右侧取消独立表单卡片，仅聚焦登录动作，不再保留重复提示文案。
- 仪表盘不增加更多图表，而是优先压缩视觉噪音，提升信息主次。

## 6. 风险与边界

- 本次更多是视觉层收口，个别业务页内部仍可能保留旧的局部配色，需要后续按页面再继续精修。
- 由于大量页面依赖 `Card` 和 `PageHeader`，全局样式收口后会产生统一变化，这是预期行为。
